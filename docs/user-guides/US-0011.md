# US-0011 — Unified analytics (Grafana embed)

**Purpose:** Use Grafana dashboards inside the financegnome SPA at a single public URL (`https://financegnome.omniflow.cc`). Operators do not need a separate public Grafana host for day-to-day analytics (DEC-0057, DEC-0056).

**Prerequisites:** US-0010 external deploy (or greenfield Compose with `grafana` + `flow-finance-ai`). Traefik `auth` on production edge; optional `AUTH_DEV_BYPASS=true` for local API-only dev.

---

## Single-URL UX (AC-7)

- Open **only** the financegnome app URL (e.g. `https://financegnome.omniflow.cc`).
- Use the **Analytics** sidebar group for all six SQL dashboards.
- Grafana stays on the internal Docker network; the browser talks to Grafana via the backend proxy at `/analytics/grafana/`.
- Public `GRAFANA_TRAEFIK_HOST` is optional and **not** used by default embed paths.

---

## Analytics sidebar (AC-1)

| Nav label | Route |
|-----------|-------|
| Platform Health | `/analytics/platform-health` |
| Cashflow | `/analytics/cashflow` |
| Subscriptions | `/analytics/subscriptions` |
| Budgets | `/analytics/budgets` |
| Portfolio | `/analytics/portfolio` |
| Forecast Horizons | `/analytics/forecast-horizons` |

Each page is a full-viewport kiosk iframe: `{VITE_GRAFANA_EMBED_BASE}/d/{uid}/{slug}?kiosk=tv` (default embed base `/analytics/grafana`).

## Environment variables

| Variable | Default | Role |
|----------|---------|------|
| `GRAFANA_UPSTREAM` | `http://grafana:3000` | Backend proxy target (Docker service name) |
| `VITE_GRAFANA_EMBED_BASE` | `/analytics/grafana` | SPA iframe prefix (build-time) |
| `GRAFANA_TRAEFIK_HOST` | empty | Optional public Grafana — not required for US-0011 |

**Deprecated:** `VITE_GRAFANA_URL` — Wealth and nav no longer open Grafana in a new tab. Use `/analytics/portfolio` instead.

**Dev vs Docker upstream:**

- Docker Compose: `GRAFANA_UPSTREAM=http://grafana:3000`
- Local Grafana on host port 3000: `GRAFANA_UPSTREAM=http://localhost:3000` (allowlisted at backend startup)

Grafana compose flags (anonymous Viewer behind Traefik boundary):

- `GF_AUTH_ANONYMOUS_ENABLED=true`
- `GF_AUTH_ANONYMOUS_ORG_ROLE=Viewer`
- `GF_SECURITY_ALLOW_EMBEDDING=true`

## Canonical UX: ECharts vs Grafana (AC-6)

| Product page | Primary analytics | Grafana (secondary) |
|--------------|-------------------|---------------------|
| `/forecast` | ECharts + API | Optional → `/analytics/forecast-horizons` |
| `/wealth` | ECharts + API | In-app → `/analytics/portfolio` |
| `/planning` | ECharts + API | Optional → `/analytics/budgets` |
| `/subscriptions` | ECharts + API | Optional → `/analytics/subscriptions` |
| `/alerts` | React inbox | — |
| Platform ops | — | `/analytics/platform-health` |

**Future-chart guideline:** New product charts default to **React + API** (ECharts). Add Grafana embed only for SQL-heavy operational views that already exist as provisioned dashboards (exception list). Do not duplicate the same metric in both surfaces without documenting which is authoritative.

## Traefik auth and dev bypass (AC-3)

- Production: Traefik `auth` middleware protects the edge. The iframe is same-origin, so the browser reuses the authenticated session for `/analytics/grafana/*` requests.
- The Grafana proxy is **outside** the `/api/v1` JWT stack; Grafana uses anonymous Viewer behind the network boundary.
- Local dev: `AUTH_DEV_BYPASS=true` skips JWT on API routes; edge auth may still apply if testing through Traefik.

## Troubleshooting

| Symptom | Checks |
|---------|--------|
| Blank iframe | Verify `flow-finance-ai` is up; `GET /analytics/grafana/` returns 200; CSP includes `frame-src 'self'`; proxy strips `X-Frame-Options: deny` |
| Stale / non-refreshing panels | WebSocket path `/analytics/grafana/api/live/` must upgrade through proxy; check browser devtools WS errors |
| 502 from proxy | `GRAFANA_UPSTREAM` reachable from backend container; host allowlist (`grafana`, `localhost`, `127.0.0.1`) |
| Dev iframe works in Docker but not locally | Set `GRAFANA_UPSTREAM=http://localhost:3000` and rebuild if needed |
| Anonymous login / org issues | DEC-0057 decision gate: if anonymous Viewer is insufficient, **stop** and request auth-proxy DEC — do not expose Grafana publicly without a new decision |
