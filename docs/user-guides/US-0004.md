# Purpose

US-0004 adds **financial planning** to Flow Finance AI: named plans with scenario adjustments (delta overlay on forecast baseline), built-in templates, v1/v2/v3 versioning, plan-vs-Ist daily comparison, React `/planning` page, and Grafana Dashboard 3 (`budgets`).

# Prerequisites

- US-0001 platform with synced Firefly transactions (Ist from mirror only)
- US-0002 forecast engine with at least one successful `forecast_computation`
- US-0003 confirmed subscriptions (optional, for Savings mode template suggestions)
- Run **Sync now** on Sync Status before comparing plan vs actual

# Usage steps

## 1. Create a plan from a template

1. Open **Planning** in the sidebar (`/planning`)
2. If no plans exist, enter a name and click **Create from Leasing template**
3. Or select an existing plan and use template cards:
   - **Current (Ist)** — baseline forecast only
   - **Leasing** — +€300/month household outflow
   - **Savings mode** — remove selected confirmed subscriptions; optional −€100/month discretionary cut
   - **House purchase** — +€500/month savings transfer
   - **Custom** — empty; add adjustment lines manually

Templates apply to the **latest editable version** only.

## 2. Edit adjustments

On the **Scenarios** tab, the adjustment table shows amount, frequency, target, and label. Add, edit, or delete lines on the latest version. Frozen versions (v1/v2 after creating a newer version) are read-only.

Plan recompute runs asynchronously after each save; a **Plan stale** badge appears until the new snapshot is ready or if recompute failed (last successful snapshot is served).

## 3. Savings mode workflow

1. Click **Apply** on the Savings mode card
2. In the modal, check confirmed subscriptions to simulate removing
3. Optionally enable the discretionary cut (−€100/month)
4. Click **Apply savings mode**

Only **confirmed** subscriptions appear in suggestions (from US-0003).

## 4. Create v2 / v3 versions

When viewing a frozen version, click **Create new version**. Adjustments copy forward; the prior version is frozen. Maximum **3 versions per plan**; attempting v4 returns HTTP 409 — create a new named plan instead.

Use the **Compare** tab for side-by-side monthly delta sum and projected month-end balance (v1/v2/v3).

## 5. Plan vs Actual tab

- Dual-line chart: **Planned** vs **Ist (actual)** daily net cashflow
- Daily table: planned, Ist, **deviation** (= actual − planned; positive is better than plan)
- Default view: current calendar month; optional month picker (`YYYY-MM`)
- **Actuals stale** badge when sync is older than ~25 hours
- Planned series extends beyond today; Ist stops at last sync date

## 6. Set active plan

Use the plan selector and **Set active**. Exactly one plan is active globally. Active plan drives Plan vs Actual default, post-forecast refresh, and Grafana Dashboard 3.

## 7. Grafana Dashboard 3

Open Grafana (`http://localhost:3000`):

| Dashboard | UID | Panels |
|-----------|-----|--------|
| Budgets | `budgets` | Active plan stat, Plan time series, Ist time series, Abweichung (deviation), MTD summary table |

Datasource UID: `FlowFinancePostgreSQL`. Requires an **active plan** selected in the Planning UI; otherwise panels show an empty-state annotation.

# Read-only guarantee

All plan data is stored in the Flow Finance PostgreSQL database. **No writes to Firefly III** — Ist (actual) is read from mirrored transactions only.

# Example API

Create a leasing plan:

```bash
curl -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"My Leasing Scenario","template":"leasing"}' \
  http://localhost:8080/api/v1/plans
```

Plan vs actual for active plan:

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "http://localhost:8080/api/v1/plans/active/plan-vs-actual?month=2026-05"
```

# Troubleshooting

| Symptom | Cause | Action |
|---------|-------|--------|
| Plan stale badge | Recompute pending or failed | Wait or trigger sync; check logs |
| No savings suggestions | No confirmed subscriptions | Confirm patterns on `/subscriptions` |
| Compare shows zeros | No successful plan computation | Apply template and wait for recompute |
| Grafana empty | No active plan | Set active plan in Planning UI |
