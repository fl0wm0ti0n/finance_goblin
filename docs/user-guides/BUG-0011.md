# Purpose

BUG-0011 fixes **Planning** mode so you can create empty plans, add adjustment lines, see sensible Compare numbers, and use the Plan vs Actual tab even before you set an active plan.

# Prerequisites

- US-0004 plan engine deployed
- US-0002 forecast with at least one successful computation (for projected balances)
- Synced Firefly transactions for Plan vs Actual Ist series

# What changed

## Empty plan and add lines (defect AD)

**Before:** "Start empty and add lines" did nothing — no form to add adjustments.

**After:**

- When you have no plans, choose **Create empty plan** (or pick a template card)
- An inline form above the adjustments table lets you add and edit lines
- Applying the **Custom** template shows a confirmation instead of a silent no-op

## Compare tab numbers (defect AE)

**Before:** Monthly delta sum included your entire household forecast baseline — empty plans could show large negative numbers like **-127489.44**.

**After:**

- **Monthly delta sum** counts only your scenario adjustments (overlay), not the baseline forecast
- An empty plan with no adjustment lines shows **0.00** monthly delta
- **Projected month-end balance** still includes the baseline forecast — it may be negative if your household forecast is

## Plan vs Actual tab (defect AF)

**Before:** With no active plan selected, the API returned **404** and the tab appeared blank.

**After:**

- The API returns **200** with a clear `no_active_plan` status
- The tab shows guided steps: create a plan and click **Set active**
- This matches how the risk-score endpoint handles missing data

# Usage steps

## 1. Create your first plan

1. Open **Planning** (`/planning`)
2. If the page is empty, enter a plan name and click **Create empty plan**
3. Or use **Create from Leasing template** for a quick start

## 2. Add adjustment lines

1. On the **Scenarios** tab, use the form above the table
2. Set direction, amount, frequency, and label
3. Click **Add** — the line appears in the table
4. Edit or delete lines on the latest version

## 3. Set active (required for Plan vs Actual)

1. Select your plan in the dropdown
2. Click **Set active**
3. Only one plan is active at a time (US-0004)

Without an active plan, Plan vs Actual shows guidance instead of an error.

## 4. Compare versions

Open the **Compare** tab for side-by-side metrics:

- **Monthly delta sum** — net impact of adjustments this month only
- **Projected month-end balance** — full scenario including baseline forecast

## 5. Plan vs Actual

1. Open the **Plan vs Actual** tab
2. With an active plan: chart and daily table as before
3. Without an active plan: follow the on-screen steps to create and activate a plan

# Troubleshooting

| Symptom | Likely cause | Action |
|---------|--------------|--------|
| Compare delta still looks wrong on old deploy | Pre-fix backend | Deploy BUG-0011 fix; numbers may shift for non-empty plans (intentional) |
| Plan vs Actual empty | No active plan | Click **Set active** on your plan |
| Projected balance negative on empty plan | Baseline forecast | Normal — only monthly *delta* should be zero |
| Grafana Dashboard 3 empty | No active plan in UI | Set active in Planning — dashboard unchanged by this fix |
