# Purpose

Flow Finance AI US-0001 gives you a self-hosted platform that mirrors Firefly III data read-only into your own PostgreSQL database, with a web UI, sync scheduler, and Grafana health dashboard.

# Prerequisites

- Docker and Docker Compose v2
- External PostgreSQL 16+ with TimescaleDB extension enabled
- Two databases (or schemas): one for Flow Finance AI (`flow_finance_ai`), one for Firefly III (`firefly`)
- Firefly III Personal Access Token (PAT) with read access
- OIDC identity provider (optional Authentik via Compose `oidc` profile, or external IdP)

# Usage steps

1. Copy `.env.example` to `.env` and set:
   - `DATABASE_*` for Flow Finance AI external PostgreSQL
   - `FIREFLY_DB_*` for Firefly's separate database
   - `FIREFLY_PERSONAL_ACCESS_TOKEN` from Firefly UI (Profile → OAuth / Personal Access Tokens)
   - `FIREFLY_APP_KEY` (32+ char random string, e.g. `base64:...`)
   - `OIDC_ISSUER_URL`, `VITE_OIDC_*` if using authentication (or `AUTH_DEV_BYPASS=true` for local dev only)

2. Ensure TimescaleDB is available on your host PostgreSQL:
   ```sql
   CREATE EXTENSION IF NOT EXISTS timescaledb;
   CREATE DATABASE flow_finance_ai;
   CREATE DATABASE firefly;
   ```

3. Start the minimal stack (with bundled Firefly III):

   ```bash
   docker compose --profile minimal --profile bundled-firefly up --build
   ```

   For app + Grafana only (external DB and external Firefly), omit `bundled-firefly`:

   ```bash
   docker compose --profile minimal up --build
   ```

   For omniflow external deployment on a shared host, see [US-0010 operator guide](US-0010.md).

4. Open Flowfly III at `http://localhost:8081`, complete setup, and create a PAT.

5. Open Flow Finance AI at `http://localhost:8080`. Sign in via OIDC (or dev bypass).

6. Go to **Sync Status** and click **Sync now** to pull accounts, transactions, categories, budgets, tags, and piggy banks.

7. Open Grafana at `http://localhost:3000` (default admin/admin) and view the **Platform Health** dashboard.

# Example

Minimal `.env` excerpt for Linux Docker reaching host PostgreSQL:

```env
DATABASE_HOST=host.docker.internal
DATABASE_PORT=5432
DATABASE_NAME=flow_finance_ai
DATABASE_USER=finance
DATABASE_PASSWORD=your-secret

FIREFLY_DB_HOST=host.docker.internal
FIREFLY_BASE_URL=http://firefly-iii:8080
FIREFLY_PERSONAL_ACCESS_TOKEN=your-firefly-pat
```

Authentik (optional): add `--profile oidc` and set `AUTHENTIK_SECRET_KEY`. Register redirect URI `http://localhost:8080/callback` in your OIDC application.

# Limitations

- Firefly connector is **GET-only**; Flow never writes to Firefly.
- No embedded PostgreSQL in Compose; you must provision external databases.
- Forecasting, subscriptions, and analytics dashboards are not in US-0001.
- Initial transaction sync may take time for large histories; progress appears on Sync Status.

# Troubleshooting

| Issue | Fix |
|-------|-----|
| `DATABASE_PASSWORD required` on Compose up | Set all required vars in `.env` |
| Backend cannot reach PostgreSQL on Linux | Use `host.docker.internal:host-gateway` (already in Compose) |
| `/health/ready` returns 503 | Check external DB credentials and TimescaleDB extension |
| OIDC redirect fails | Match `VITE_OIDC_REDIRECT_URI` to IdP allowed callback URLs |
| Sync returns 401 | Set `AUTH_DEV_BYPASS=true` for dev, or configure valid OIDC/JWT |
| Grafana datasource error | Verify `DATABASE_*` env vars passed to Grafana service |
