# US-0013 — Production ML on omniflow external profile

## Purpose

US-0013 enables the **US-0009 ML-enhanced forecast overlay** on the omniflow **external** Compose profile (`financegnome.omniflow.cc`). Until enabled, deployments are **baseline-only** by design (DEC-0049) — Compare and Grafana ML panels show degraded or empty states with honest messaging (DEC-0066, BUG-0009 banner).

This guide covers **operator enablement** on external profile. For ML feature semantics (Compare modes, bands, risk score), see [US-0009](US-0009.md).

## Prerequisites

- US-0010 external profile operational (Firefly + Postgres on host `traefik` network)
- BUG-0010 baseline fixes deployed — plausible signed balances before ML overlay
- **Full Firefly sync** completed (≥12 monthly net-cashflow points per funded account for ML gate)
- Docker Compose ≥ 2.24 (`!reset` support in external overlay)

## Enabling ML on omniflow

### 1. Compose profile union

Use the two-file merge with **external** profile only (never combine with `minimal`, `standard`, `full`, or `bundled-firefly`):

```bash
export COMPOSE_FILE=docker-compose.yml:docker-compose.external.yml
export COMPOSE_PROFILES=external
docker compose up -d
```

Verify three services run: `flow-finance-ai`, `grafana`, `stats-forecast`.

### 2. Environment variables

Add to operator `.env` (see `.env.example` omniflow block):

| Variable | Omniflow value | Notes |
|----------|----------------|-------|
| `FORECAST_ML_ENABLED` | `true` | Operator opt-in — default-off elsewhere (DEC-0049) |
| `STATS_FORECAST_URL` | `http://stats-forecast:8090` | Internal Docker DNS on traefik network |
| `STATS_FORECAST_PORT` | `8091` | Host port remap (8090 used by other services) |

Restart `flow-finance-ai` after env change.

### 3. Sidecar health smoke

```bash
# From host (optional — uses STATS_FORECAST_PORT)
curl -sf http://localhost:8091/health

# From backend container on traefik network
docker compose exec flow-finance-ai curl -sf http://stats-forecast:8090/health
```

Expected: `{"status":"ok"}`.

**Cold start:** first forecast after sidecar start may take ~5s (numba JIT). First sync immediately after `compose up` may record `sidecar_unavailable` — trigger a **manual sync** once healthcheck is green.

### 4. Full sync + recompute

Run a **Full** Firefly sync from Settings or API. Sync status shows **"ML forecast…"** sub-phase when `FORECAST_ML_ENABLED=true`.

ML failure **does not fail sync** (DEC-0052). Check sync metadata or `/api/v1/forecast/meta` for `ml_status` and `ml_skipped_reason`.

### 5. Verify product surfaces

| Surface | Check |
|---------|-------|
| `/forecast` Long-term **Compare** | Baseline + ML-enhanced lines when ML available |
| `/forecast` degraded | `sidecar_disabled` copy when ML off — not generic skip (DEC-0066) |
| `/wealth` | Portfolio outlook horizons; FX incomplete banner when applicable (R-0034) |
| `/analytics/forecast-horizons` | Set `$forecast_variant=ml_enhanced` — panels populate when computations exist |
| ML off | BUG-0009 banner remains: baseline authoritative |

### API spot-check

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "$API/api/v1/forecast/long-term?account_id=114&variant=ml_enhanced&horizon=12"
```

Expect non-empty `series` with `balance_p10` / `balance_p90` when history sufficient.

## Minimum history

| Gate | Default | Skip reason |
|------|---------|-------------|
| ML cashflow overlay | 12 monthly points | `insufficient_history` |
| MSTL seasonal model | 24 months + strength ≥ 0.35 | SeasonalNaive fallback or low confidence |

Run **Full sync** before expecting ML rows on a fresh omniflow deploy.

## Degraded-mode troubleshooting

| Symptom | Likely cause | Action |
|---------|--------------|--------|
| Compare tabs disabled; `sidecar_disabled` | `FORECAST_ML_ENABLED` false or sidecar not started | Check env + `docker compose ps stats-forecast` |
| Sync OK but no `ml_enhanced` rows | Sidecar unhealthy at sync time | Re-run sync after `/health` OK |
| `insufficient_history` in meta | <12 monthly points | Full sync; wait for mirror history |
| Grafana ML panels empty | No `ml_enhanced` computations | Enable ML + re-sync; check `$forecast_variant` |
| Host curl to 8090 fails | Wrong port on omniflow | Use **8091** (`STATS_FORECAST_PORT`) |

## Disabling ML

Set `FORECAST_ML_ENABLED=false` (or remove) and restart backend. Baseline DEC-0007 forecast remains authoritative for alerts and plan hooks. Existing `ml_enhanced` rows may remain in DB but are not refreshed.
