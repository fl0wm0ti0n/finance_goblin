## Purpose

When Firefly mirror categories are missing or not listed in your `[forecast.category_buckets]` map, Flow Finance AI can propose **income**, **fixed**, or **variable** bucket assignments for **recurring forecast pattern dues** â€” after the DEC-0007 config map is applied.

This improves monthly Income / Fixed / Variable cards when categorized config rules do not cover every recurring merchant.

## Config wins (AC-1)

Categories mapped in `backend/config/default.toml` under `[forecast.category_buckets]` are **never** overridden by AI.

| Config bucket | Behavior |
|---------------|----------|
| `income` | Always Income; AI not consulted |
| `fixed` | Always Fixed; AI not consulted |
| `variable` | Explicit Variable from config; AI not consulted |

Only **ambiguous** rows (missing/unknown category name) enter the AI cascade.

## AI cascade (AC-2)

1. Config map (`resolve_bucket`)
2. In-module sign/pattern rules (high-confidence short-circuit)
3. LLM batch via configured provider (`build_provider()` â€” same as AI Chat)
4. **Variable** fallback when confidence &lt; `ai_bucket_min_confidence` (default **0.75**) or provider unavailable

Operator override: set `ai_bucket_min_confidence` under `[forecast]` in TOML.

## AI-mapped badge (AC-5)

On **Forecast â†’ Monthly**, when any recurring AI-assigned mass exists in a month, the UI shows an **AI-mapped** badge.

- `ai_mapped=true` is authoritative even when dominant `bucket_sources` labels are mixed.
- Config-only months show **no** badge.

## Privacy (AC-3)

Default (`allow_raw_transactions=false`):

| Sent to model | Never sent |
|---------------|------------|
| Lowercase category name | Raw payee/description |
| Hashed merchant token | IBAN |
| Amount sign (âˆ’1/0/+1) | Exact amount |
| Magnitude band (`0-50`, `50-200`, `200+`) | |
| Pattern class / recurring label when available | |

**Opt-in risk:** `allow_raw_transactions=true` permits normalized descriptions for â‰¤50 rows per batch â€” elevated privacy risk; use only when you accept model exposure.

Same allowlist applies to local (Ollama) and cloud providers.

## Rolling residual (MVP limitation)

The **rolling discretionary residual** (`variable_residual` daily rate) stays **Variable** in this release. AI applies to **recurring pattern dues** with ambiguous categories only â€” not aggregate discretionary spend mass.

## Audit trail (AC-6)

Each AI or fallback assignment writes `forecast_bucket_assignment` rows to `ai_tool_audit` with redacted `feature_id`, bucket, confidence, source, and `rationale_code` â€” never raw merchant text.

## Troubleshooting

| Symptom | Likely cause | Action |
|---------|--------------|--------|
| All buckets Variable | Provider absent/misconfigured | Configure AI provider (US-0008); check logs for `provider_unavailable` audit |
| Expected Fixed but Variable | Confidence below threshold | Lower `ai_bucket_min_confidence` or recategorize in Firefly + config map |
| No AI-mapped badge | Config map covers all recurring rows | Expected when categories are mapped |
| Badge but config label on card | Mixed sources in month | Badge reflects any AI mass; dominant label uses config &gt; ai &gt; default precedence |
