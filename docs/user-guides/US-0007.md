# US-0007 — Crypto exchange portfolio

## Purpose

US-0007 integrates **Binance, Bybit, and Bitunix** read-only exchange data into household wealth: crypto balances and positions, hybrid PnL (realized/unrealized/total return), EUR FX conversion, extended net worth, allocation planning scenarios, React `/wealth` Crypto tab, Settings exchange status, Sync Status per-exchange rows, Grafana Dashboard 4 completion, and extended `get_portfolio` AI tool output.

## Prerequisites

- US-0001–US-0006 operational (platform, forecast, subscriptions, planning, wealth/alerts, AI assistant)
- External PostgreSQL with migration 007 applied (automatic on backend startup)
- **Read-only** exchange API keys — never enable trading or withdrawal permissions

## Enabling exchanges

1. Edit `backend/config/default.toml` (or env overlay):

```toml
[exchanges]
enabled = true
interval_seconds = 3600

[exchanges.binance]
enabled = true

[exchanges.bybit]
enabled = false

[exchanges.bitunix]
enabled = false
```

2. Set environment variables (see `.env.example`):

| Exchange | Variables |
|----------|-----------|
| Binance | `BINANCE_API_KEY`, `BINANCE_API_SECRET` |
| Bybit | `BYBIT_API_KEY`, `BYBIT_API_SECRET` |
| Bitunix | `BITUNIX_API_KEY`, `BITUNIX_API_SECRET` |

3. **Restart the backend container** — credentials are env-only; TOML stores env var **names**, never secret values.

### Read-only key setup

| Exchange | Requirement |
|----------|-------------|
| Binance | Enable **Reading** only; disable trading and withdrawal |
| Bybit | Create key with `readOnly=1` |
| Bitunix | No trade or withdraw permissions |

## Sync behavior

| Trigger | Phases |
|---------|--------|
| **Sync now** (Sync Status) | Full pipeline: Firefly → subscriptions → forecast → **exchanges** → alerts |
| **Sync exchanges now** | Partial: exchanges → alerts only |
| Scheduled tick | Full sync when Firefly interval due; exchange-only when exchange interval due |

Recommended: set `[exchanges] interval_seconds = 3600` (or longer) to reduce rate-limit risk on trade history fan-out.

## Wealth UI (`/wealth`)

### Overview tab

- Combined **total EUR** = Firefly subtotal + crypto subtotal (when FX complete)
- Firefly and Crypto subtotal cards
- Account breakdown (Firefly mirror accounts)
- Allocation gap card when active plan has `allocation_target` adjustment

### Crypto tab

- Per-exchange connection cards (state, last sync, subtotal)
- Holdings table (asset, quantity, EUR value, unrealized PnL)
- PnL summary: realized / unrealized / total return %
- **FX incomplete banner** when altcoins lack EUR price — excluded from crypto subtotal; lists unpriced assets

PnL is **wealth analytics**, not tax FIFO reporting.

## Allocation scenarios

1. On `/planning` → Scenarios, choose **Allocation target** template (default 50/50 ETF vs crypto)
2. Activate the plan
3. On `/wealth` Overview or Crypto tab, view current vs target weights and gap
4. Adjust targets on Planning as needed

## Settings — Crypto exchanges

Read-only display:

- Enabled flag from TOML
- **Configured** badge when env vars resolve non-empty
- **Test connection** button (read-only balance read; does not persist secrets)
- Restart-required note for credential changes

## Sync Status — per-exchange rows

Columns: exchange name, status, last sync, entity counts (holdings/trades), error (redacted). Use **Sync exchanges now** for partial re-sync.

Phase indicator shows **Syncing exchanges…** during phase 4.

## Troubleshooting

| Issue | Action |
|-------|--------|
| Connection error | Verify read-only key, env vars, restart backend |
| Rate limits (429) | Increase exchange interval; check Binance symbol fan-out logs |
| First-sync avg-cost inaccuracy | Expected for spot without exchange PnL fields; improves after trade backfill |
| FX incomplete banner | Illiquid alt — add liquid pair or wait for ticker |
| Mutex latency | Check exchanges phase timing in logs; use longer exchange interval |
| PnL reconciliation warning | Exchange cumulative preferred when drift >1% |

## Secret rotation

1. Update env vars in `.env` or orchestrator secrets
2. Restart backend container
3. Use **Test connection** on Settings to verify

## Out of scope

- Additional exchanges (Kraken, Coinbase, etc.)
- On-chain wallets
- Tax reporting / FIFO tax lots
- Browser secret entry or runtime credential edit
