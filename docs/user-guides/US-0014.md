# US-0014 — Planning mode intuitive UX

## Purpose

Planning mode lets you model household budget scenarios, compare versions, and track plan vs actual spending. US-0014 polishes the **first-visit experience** after BUG-0011 functional fixes: clear create confirmations, visible errors when something fails, and guidance to **Set active** so Plan vs Actual and Grafana dashboards use your scenario.

## First visit (no plans yet)

1. Open **Planning** in the sidebar (`/planning`).
2. Enter a **plan name**.
3. Either:
   - Click **Create empty plan** and add adjustments on the Scenarios tab, or
   - Pick a template card (**Current**, **Leasing**, **Savings mode**, **House purchase**, **Custom**) and click **Create from {template}**.
4. A confirmation message appears when the plan is created successfully.

If creation fails (network, validation), a **red error message** appears at the top of the page with a **Dismiss** button — check the message and retry.

## Adding scenario lines

On the **Scenarios** tab, use the inline form to add adjustments (direction, amount, frequency, target). After you submit:

- A **success message** confirms the line was added.
- **Compare** and **Plan vs Actual** refresh when you switch tabs.

Errors (e.g. invalid amount) show a red error banner — they are not silent.

## Set active (required for Plan vs Actual and Grafana)

New plans are **not active** by default. After creating a plan:

1. Read the yellow **Set active** banner (mentions Plan vs Actual and Grafana Dashboard 3 **Budgets**, dashboard `uid=budgets`).
2. Click **Set active** next to the plan selector.

**Why:** Plan vs Actual and **Grafana Dashboard 3 (Budgets)** use the **active** plan only. Until you set active, Plan vs Actual shows a guided card with links back to Scenarios.

## Compare tab

- **Monthly delta** = scenario adjustments only (overlay). An empty plan shows **0.00** delta.
- **Projected month-end balance** includes the baseline household forecast and may be negative even when delta is zero — see the footnote under the compare table.

## Plan vs Actual tab

- With an **active plan**: monthly plan vs actual rows for the selected month.
- **Without an active plan**: guided card explaining how to set active — not a blank tab.

## Templates on existing plans

When you already have plans, templates remain available on the Scenarios tab (**Apply** on template cards). Applying a template shows confirmation; failures show an error banner.

## Troubleshooting

| Symptom | Action |
|---------|--------|
| Red error after create/add | Read message; click **Dismiss** if needed; fix input; retry |
| Plan vs Actual empty | Set active on Scenarios tab |
| Compare shows 0.00 delta but negative projected balance | Expected on empty overlay — baseline forecast still applies |
| Grafana Budgets empty | Ensure a plan is **active** and sync has run |
