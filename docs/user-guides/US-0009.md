# US-0009 — Advanced forecasting with ML & risk assessment

## Purpose

US-0009 adds an optional **ML-enhanced forecast overlay** on the DEC-0007 baseline engine: StatsForecast sidecar, confidence bands, Baseline | ML | Compare UI, plan risk score, crypto portfolio outlook, and Grafana Dashboard 5 extensions. **Baseline remains authoritative** for alerts and plan hooks.

## Prerequisites

- US-0002 baseline forecast operational (migration 002+)
- US-0004 planning for risk score (migration 004+)
- US-0007 exchange data for portfolio outlook (migration 007+)
- Migration 009 applied automatically on backend startup
- For ML features: Docker Compose **`--profile full`** and `[forecast_ml] enabled = true`

## Enabling ML forecasting

1. Start the full profile (includes `stats-forecast` sidecar on port 8090):

```bash
docker compose --profile full up -d
```

2. Edit `backend/config/default.toml`:

```toml
[forecast_ml]
enabled = true
sidecar_url = "http://stats-forecast:8090"
```

Or set environment variables:

| Variable | Effect |
|----------|--------|
| `FORECAST_ML_ENABLED=true` | Enable ML overlay pass after baseline |
| `STATS_FORECAST_URL=http://stats-forecast:8090` | Sidecar URL override |

3. **Restart the backend** after config change.

4. Verify sidecar health: `curl http://localhost:8090/health` → `{"status":"ok"}`

### Minimum history

| Feature | Requirement |
|---------|-------------|
| ML cashflow overlay | ≥12 monthly transaction points per account |
| MSTL seasonal model | ≥24 months + seasonal strength ≥ 0.35 |
| Portfolio outlook | ≥8 weekly crypto EUR snapshots (US-0007 exchanges) |

When history is insufficient, sync still **succeeds**; baseline is served and `ml_skipped_reason` is recorded.

## Forecast UI (`/forecast`)

### Long-term tab — three modes

| Mode | Description |
|------|-------------|
| **Baseline** | DEC-0007 engine (unchanged); default for alerts |
| **ML-enhanced** | Baseline path + ML net-cashflow overlay with p10–p90 bands |
| **Compare** | Dual lines (baseline blue, ML orange) + delta at horizon end |

- Horizon pills: **6 / 12 / 24 months** for ML and Compare; **3 months** baseline-only
- When ML skipped: ML and Compare tabs disabled with tooltip showing skip reason
- **How this forecast works** panel: `model_family`, seasonal periods, backtest WMAPE, skip reason

### Band interpretation

- Bands are **90% prediction intervals** (symmetric) from StatsForecast
- **Low confidence** (WMAPE > 0.35 or sparse data): bands shown at reduced opacity
- ML does not replace baseline — it adjusts monthly net-cashflow deltas onto the baseline daily path

### Monthly tab — seasonal callout

When ML detects seasonality (`seasonal_detected=true`), a badge shows e.g. **"12-month cycle"**. Hidden when ML skipped or no ML metadata.

## Planning risk score (`/planning`)

Deterministic **0–100** index (not an alert type):

| Band | Score | Color |
|------|-------|-------|
| Low | 0–29 | Green |
| Medium | 30–59 | Amber |
| High | 60–100 | Red |

**Components** (hover tooltip on Scenarios badge):

- 45% balance stress — month-ends < €0 in next 6 months on active plan
- 40% plan viability — R-0022 rules without firing alerts
- 15% crypto volatility — CV of last 12 weekly crypto EUR (0 when no exchanges)
- ±5 ML divergence — ML 6mo p10 below scarcity while baseline end above

Compare tab shows risk column on the latest active plan version.

## Wealth Crypto portfolio outlook (`/wealth` → Crypto)

When exchange history and ML portfolio forecast exist:

- Stat row: projected EUR at **3 / 6 / 12 months** with optional bands
- **FX incomplete banner** when >20% crypto value lacks EUR pricing (R-0034)
- Hidden when portfolio forecast skipped or insufficient weekly history

## Grafana Dashboard 5 (`forecast-horizons`)

1. Open Grafana → **Forecast Horizons** dashboard
2. Set **`$forecast_variant`**: Baseline (default) or ML Enhanced
3. New panels (when ML data exists):
   - Confidence band path (balance, p10, p90)
   - Seasonal detected stat
   - Baseline vs ML 12mo end balance
   - Portfolio 3/6/12 mo EUR
   - Active plan risk score

ML panels are empty when no `ml_enhanced` computation exists — expected on minimal/standard profiles.

## AI assistant

`get_forecast` tool accepts optional `variant=baseline|ml_enhanced` (default baseline). ML variant includes band summary at horizon end. Six-tool registry unchanged.

## Troubleshooting

| Symptom | Likely cause | Action |
|---------|--------------|--------|
| ML/Compare tabs disabled | `ml_skipped_reason` set | Check meta API or baseline metadata |
| `sidecar_unavailable` | Sidecar not running or wrong URL | `--profile full`, verify `/health` |
| `insufficient_history` | <12 monthly points | Wait for more Firefly history |
| Unstable bands / low_confidence | Sparse data or high WMAPE | Normal; interpret bands cautiously |
| Portfolio outlook hidden | <8 weekly crypto snapshots | Connect exchanges (US-0007), run sync |
| Config change ignored | TOML loaded at startup | Restart backend container |

## Boundaries

- **Baseline authoritative** for alerts, plan hook (DEC-0023), AI default, Grafana default variant
- No cloud ML APIs; sidecar runs locally in Compose full profile
- ML failure never fails sync (DEC-0052)
- Restart required for `[forecast_ml]` config changes
