# US-0018 — Category filters & expense trend analytics

## Purpose

Category-scoped spending visibility across Forecast, Planning, Wealth, and Grafana dashboards. Mirror transactions drive **actual** monthly outflow series; household forecast and plan compare metrics stay unchanged when you pick a category (DEC-0089).

## Forecast (`/forecast` → Monthly)

- **Category filter** above stat cards scopes the **actual spending trend** chart only.
- **Income / Fixed / Variable / Free cashflow** cards and the household **MonthlyChart** remain DEC-0007 household forecast — not re-projected per category.
- Select a category (or **Uncategorized**) to load `GET /api/v1/categories/expense-series` and view the bar trend with MoM, best, and worst month callouts.
- Tooltip copy on the page explains that the filter does not change forecast buckets.

See also: [US-0002 Forecast overview](US-0002.md).

## Planning (`/planning` → Compare)

- Toolbar **Category filter** + **Actual spending trend** widget beside the version compare table.
- Compare metrics API and version table have **no** `category_id` query param — plan deltas stay household-level.
- Use the widget to preview mirror actuals while reviewing scenario versions.

## Wealth (`/wealth` → Overview)

- **Category spending** subsection: pick a category (no “All” option) to see period trend.
- Net worth, Firefly subtotal, and crypto totals remain household-level.

## Grafana (independent filters)

Cashflow and Budgets dashboards expose a **`$category`** variable (default **All categories** = empty string).

| Dashboard | Behavior when category set |
|-----------|----------------------------|
| **cashflow** | New panel: monthly category outflow from mirror |
| **budgets** | Ist / deviation / MTD **actual** legs filtered; **planned** leg unchanged |

Grafana selection is **not** synced with the React app (DEC-0089).

## Uncategorized (`__uncategorized__`)

- Dropdown value **Uncategorized** maps to `category_id=__uncategorized__`.
- API returns `uncategorized: true` and `category_label: "Uncategorized"`.
- Months with no null-category rows show explicit €0 on the spine — not silent omission.

## Troubleshooting

| Symptom | Likely cause | Action |
|---------|--------------|--------|
| 404 on expense-series | Stale or deleted Firefly category id | Run **Full Firefly sync**; pick another category |
| Empty trend, zero transactions | No mirror rows in window | Widen months param (max 24) or recategorize in Firefly |
| Forecast cards unchanged when filtering | By design (DEC-0089) | Use trend chart for category actuals; forecast stays household |
| Grafana panel empty after category pick | No outflow in time range | Clear `$category` to verify household view; check mirror dates |
