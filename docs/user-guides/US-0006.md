# Purpose

US-0006 adds a **privacy-safe AI financial assistant** to Flow Finance AI: six registered in-process tools, central PrivacyLayer redaction, OpenAI Chat Completions orchestration, SSE chat API, header Sheet drawer + `/chat` page, Settings AI & Privacy display, and operator tool audit log (migration 006).

# Prerequisites

- US-0001‚ÄìUS-0005 operational with synced Firefly mirror data
- External PostgreSQL with migrations through `006_ai_audit.sql`
- `OPENAI_API_KEY` set in the environment (name configurable via `[ai] api_key_env`)
- Optional: `PRIVACY_PEPPER` for stable counterparty hashing (defaults to app-derived pepper)

# Enabling AI

1. Configure `backend/config/default.toml`:

```toml
[ai]
provider = "openai"
model = "gpt-4o-mini"
api_key_env = "OPENAI_API_KEY"
max_tool_rounds = 5
rate_limit_per_min = 20

[privacy]
allow_raw_transactions = false
redact_iban = true
redact_counterparties = true
```

2. Export the API key (never put it in TOML or the browser):

```bash
export OPENAI_API_KEY=sk-...
```

3. Restart the backend. Settings shows **OpenAI: Configured** when the env var is present.

# Privacy defaults

| Setting | Default | Effect |
|---------|---------|--------|
| `allow_raw_transactions` | `false` | `get_transactions` returns category/month aggregates only |
| `redact_iban` | `true` | IBAN patterns ‚Üí `[IBAN_REDACTED]` |
| `redact_counterparties` | `true` | payee/description ‚Üí `Counterparty-{hash8}` |

When raw transactions are disabled, tool output containing row arrays is replaced with `{ "error": "raw_transactions_disabled" }`.

# Using chat

- **Header AI button** ‚Äî opens a ~400px side drawer with the same ChatPanel as the full page
- **`/chat` route** ‚Äî full-page chat; sidebar **AI** nav is enabled (no ‚ÄúComing soon‚Äù badge)
- **Suggested prompts** ‚Äî empty-state chips for common German example questions
- **Privacy badge** ‚Äî ‚ÄúPrivacy: aggregates only‚Äù when raw transactions are off
- **Tools used** ‚Äî collapsible row under assistant messages (name + time; no raw JSON)

Chat history is **ephemeral** in the browser (optional `sessionStorage` not required for MVP). No server-side thread persistence.

# Settings ‚Äî AI & Privacy

Open **Settings** ‚Üí **AI & Privacy**:

- Read-only display of `[ai]` and `[privacy]` TOML values
- OpenAI configured / not configured badge
- **Tool audit log** table from `GET /api/v1/ai/audit` (timestamp, tool, user, duration, status)

To change privacy or model settings: edit TOML, restart backend. Runtime toggles are deferred to a future release.

# Audit log

Each tool invocation records:

- Session id, user subject, tool name, **redacted** `args_summary`, status, duration, model
- **Not stored:** full prompts/responses, API keys, raw transaction rows

**Retention (startup job):** purge rows older than `audit_retention_days` (default 90) and cap at `audit_max_rows` (default 500).

# Troubleshooting

| Symptom | Action |
|---------|--------|
| ‚ÄúOpenAI not configured‚Äù | Set `OPENAI_API_KEY`, restart backend |
| HTTP 429 on chat | Wait ‚Äî per-user rate limit (`rate_limit_per_min`, default 20/min) |
| SSE error / provider_error | Check API key, network egress, OpenAI status |
| Empty aggregates | Run **Sync now** so mirror `transactions` are populated |
| No plan simulation | Create/activate a plan (US-0004) or pass `template` |

# Read-only guarantee

AI tools call **ForecastService, SubscriptionService, PlanService, WealthService, AlertService, TransactionsService** only. No `DbPool` in the tool layer, no Firefly HTTP writes, no direct SQL from `ai/tools/*`.

# US-0008 boundary

This release supports **OpenAI only**. A stub `AiProvider` trait exists for future local/self-hosted providers (Ollama, LM Studio, etc.) in US-0008.
