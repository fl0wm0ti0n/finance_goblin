# Purpose

US-0003 adds **subscription intelligence** to Flow Finance AI: recurring-pattern detection with confidence tiers (95/80/60%), confirm/reject workflow, standing-order separation, price-change alerts, forecast override for confirmed/rejected patterns, a React `/subscriptions` page, and Grafana Dashboard 2.

# Prerequisites

- US-0001 platform running with synced Firefly transactions
- US-0002 forecast engine (shared recurrence core extracted from forecast heuristics)
- At least **3 recurring expense transactions** for the same payee spanning **2+ billing cycles** (12 months of history recommended)
- Successful sync run to trigger inline subscription detection (DEC-0018)

# Usage steps

## 1. Sync to run detection

Subscription detection runs automatically after each successful Firefly sync, before forecast recompute:

1. Open **Sync Status**
2. Click **Sync now**
3. Watch phases: **Syncing Firefly data…** → **Detecting subscriptions…** → **Recomputing forecasts…**
4. When idle, open **Subscriptions** in the sidebar

## 2. Review pending patterns

1. Open `/subscriptions` → **Pending review** tab
2. Each card shows payee, interval, amount, and confidence badge (95 / 80 / 60%)
3. **Confirm** — optional kind override (subscription vs standing order)
4. **Reject** — permanently excludes fingerprint from detection and forecast

Only **confirmed** patterns override forecasts and receive price-change alerts.

## 3. Confirmed list and standing orders

- **All** — every pattern regardless of status
- **Standing orders** — confirmed patterns classified as Dauerauftrag (`kind=standing_order`)
- Click a row to open the detail drawer with linked transaction count and ECharts price history

## 4. Alerts

- Unread alerts appear as a banner on `/subscriptions`
- After sync completes, new detection or price-change alerts may surface via toast
- Mark alerts read individually from the banner

## 5. Grafana Dashboard 2

Open Grafana (`http://localhost:3000`):

| Dashboard | UID | Panels |
|-----------|-----|--------|
| Subscriptions | `subscriptions` | Confirmed count, normalized monthly spend, pending count, price-change table (90d), new detections time series |

Datasource UID: `FlowFinancePostgreSQL`. MVP scope is global (no per-account filter).

# Confidence tiers

| Tier | Meaning |
|------|---------|
| **95%** | ≥4 occurrences, stable cadence, amounts within ±5% |
| **80%** | ≥3 occurrences, stable cadence, amounts within ±10% |
| **60%** | ≥3 occurrences, stable cadence, minor outlier or ±15% amounts |

Patterns below 60% are not emitted to the pending queue.

# Forecast integration

After you confirm or reject patterns:

- **Confirmed** — validated amount and interval **replace** heuristic recurring projection for the same payee
- **Rejected** — fingerprint excluded from forecast recurring detection entirely
- **Pending / no decision** — US-0002 heuristic baseline unchanged (DEC-0007)

Recompute happens in the same sync cycle as detection (no extra trigger needed).

# Example API

List pending patterns:

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "http://localhost:8080/api/v1/subscriptions?status=pending"
```

Confirm with standing-order kind:

```bash
curl -X POST -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"kind":"standing_order"}' \
  "http://localhost:8080/api/v1/subscriptions/{id}/confirm"
```

# Limitations

- Detection uses expense transactions only (`amount < 0`); internal transfers excluded
- Price-change alerts on **confirmed** patterns only; dual threshold (≥€1.00 AND ≥5%)
- In-app alerts scoped to `/subscriptions` page (global inbox deferred to US-0005)
- Promotional pricing may produce false price-change alerts — confirm carefully
- Concurrent sync triggers return **409** during sync, subscriptions, or forecast phases

# Troubleshooting

| Issue | Fix |
|-------|-----|
| No pending patterns | Ensure ≥3 recurring expense txs; run sync; check backend logs for detection phase |
| Detection phase fails | Sync still succeeds; forecast uses prior confirmed state — check `DATABASE_URL` and migration 003 |
| Price history empty | Confirm pattern first; billing events append on each detection pass for confirmed patterns |
| Grafana panels empty | Run at least one detection pass; panels handle zero rows gracefully |
