# US-0020 — Subscription discover, majority category & operator tags

## Purpose

Search for recurring expense **candidates** manually, confirm them without the pending-only auto-detection path, see the **majority display category** from linked transactions, and organize confirmed subscriptions with **operator tags** — all stored in the product database with **no Firefly write-back**.

## Discover tab (`/subscriptions` → Discover)

1. Select an **account** (required). The first asset account is pre-selected when available.
2. Optionally enter a **payee** substring (matches normalized payee key or display name).
3. Pick an **interval bucket** (Weekly 7d, Biweekly 14d, Monthly 30d, Quarterly 90d) or **Custom** days, or leave interval open.
4. Click **Search**. Results are capped at **50** candidates sorted by confidence and transaction count.
5. If truncated, refine payee or interval filters.

## Confirm from Discover (AC-2)

1. Click **Confirm** on a candidate row.
2. Choose **Subscription** vs **Standing order** if needed.
3. On success the pattern is stored as **confirmed** immediately (no pending intermediate).
4. If a confirmed subscription already exists for the same payee + interval (±3 days), the row is **merged** (DEC-0085) instead of duplicated.
5. Manual confirm does **not** emit a `new_detection` alert.

Rejected payee+interval combinations return **409** until the operator clears the rejection on the Pending tab.

## Majority display category (AC-3)

Confirmed rows show a **category badge** resolved via the category catalog (`GET /api/v1/categories`).

- Computed as the **most common** `category_id` among linked mirror transactions.
- Ties break by **most recent** transaction date.
- When all linked transactions are uncategorized, the badge reads **Uncategorized**.
- Tooltip: *Majority category from linked transactions; ties broken by most recent charge.*

Category is recomputed on confirm and when merge links new transactions — not on every sync tick.

## Operator tags (AC-4)

1. Open **Manage tags** on the Subscriptions page.
2. **Create**, **rename**, or **delete** tags (delete requires confirmation; assignments are removed via CASCADE).
3. Open a confirmed subscription **detail drawer** and checkbox-select tags, then **Save tags**.
4. On **All** or **Standing orders** tabs, use **filter chips** to narrow the list (`?tag=` slug filter).

Tags live in `operator_tags` / `subscription_pattern_tags` — never written back to Firefly.

## Auto-detection unchanged (AC-6)

- **Pending review** tab confirm/reject flows are unchanged.
- Sync-triggered `DetectionPipeline` thresholds and skip order are unchanged.
- Alert dedup (US-0008) remains on auto-detected pending patterns only.

## Troubleshooting

| Symptom | Likely cause | Action |
|---------|--------------|--------|
| No discover results | &lt;3 txs or below confidence | Widen window via full sync; relax filters |
| 409 on confirm | Prior rejection for payee+interval | Clear rejection or pick another candidate |
| Uncategorized badge | Linked txs lack category_id | Categorize in Firefly; re-confirm or merge |
| Tag filter empty | No patterns tagged | Assign tags in detail drawer |
