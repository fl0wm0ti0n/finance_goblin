# US-0021 — Subscription transaction explorer with rich filters

## Purpose

Browse **individual expense transactions** on the Discover tab with rich filters (account, payee, category, Geldbereich, date), see **recurring hint badges** on filtered subsets, and **manually activate** multi-selected transactions — while the **Suggested patterns** sub-tab preserves the US-0020 discover candidate workflow unchanged.

## Discover dual mode (`/subscriptions` → Discover)

The Discover tab uses a segmented control:

| Mode | Default | What you see |
|------|---------|--------------|
| **Transactions** | **Yes** | Paginated individual expense rows |
| **Suggested patterns** | No | US-0020 `DiscoverCandidate` groups |

Account selection is shared when switching modes. Payee filter applies to both modes. Category, Geldbereich, and date filters are **Transactions-only**.

## Transactions mode (AC-1, AC-2)

1. Select an **account** (required). The first asset account is pre-selected when available.
2. Optionally filter by **payee** substring, **category** (`CategoryFilter`), **Geldbereich** (account role), and **date range**.
3. Toggle **Show recurring hints** (default on) to attach hint badges without auto-detection side effects.
4. Click **Search**. Results are paginated at **100 transactions per page**.
5. If truncated or hint scan capped at 500 txs, refine filters using the banner message.

### Hint badges (AC-3)

- Hints run a **separate** recurrence pass on the filtered subset (`min_emit_confidence: 60`).
- Badges show interval label and confidence % — they do **not** create pending patterns or `new_detection` alerts.
- Operator repro: account **114**, payee **SEPA-Lastschrift** — expect individual rows with hint badges in Transactions mode after deploy.

## Multi-select activate (AC-4)

1. Select **≥2** transaction checkboxes in the results table.
2. Click **Activate selected**.
3. Review the confirm modal (payee, interval, median amount from server preview-group).
4. Choose **Subscription** vs **Standing order** if needed.
5. Confirm creates or **merges** (DEC-0085) a confirmed pattern via existing `POST /discover/confirm` — no pending-only path.

Rejected payee+interval combinations return **409** until cleared on Pending.

## Suggested patterns mode (AC-5)

Unchanged from US-0020 / DEC-0098:

1. Filters: account, payee, interval bucket only.
2. Candidate table with per-row **Confirm**.
3. Auto-detection thresholds and sync pipeline are **not** modified by US-0021.

## Auto-detection unchanged

- **Pending review** tab flows unchanged.
- `DetectionPipeline::run_candidates` and global `min_emit_confidence: 60` unchanged.
- Alert dedup (US-0008) remains on auto-detected pending patterns only.

## Troubleshooting

| Symptom | Likely cause | Action |
|---------|--------------|--------|
| No transaction rows | Missing account or over-narrow filters | Select account; widen date/payee |
| No hint badges | Hints off or txs below confidence | Enable hints; need ≥3 matching txs @ ≥60% |
| Activate disabled | &lt;2 rows selected | Select more transactions |
| 409 on confirm | Prior rejection for payee+interval | Clear rejection or adjust selection |
| Patterns tab empty | Below candidate threshold | Use Transactions mode or widen interval |
