# Purpose

US-0005 adds **wealth analysis** and a unified **Alert Engine** to Flow Finance AI: net worth aggregation from Firefly mirror accounts, scarcity/budget-drift/plan-viability alerts, React `/wealth` and `/alerts` pages, header bell inbox, daily wealth snapshots, and Grafana Dashboard 4 (`portfolio`). Dashboard 1 scarcity threshold now reads from centralized `alert_config`.

# Prerequisites

- US-0001 platform with synced asset accounts (`include_net_worth=true` in Firefly)
- US-0002 forecast with at least one successful `forecast_computation` (scarcity alerts)
- US-0004 active plan with **category-targeted** adjustments (budget drift alerts)
- Run **Sync now** on Sync Status — alerts evaluate in the post-forecast `"alerts"` phase

# Usage steps

## 1. Net worth view (`/wealth`)

1. Open **Wealth** in the sidebar
2. Review the **Net worth** stat card (EUR headline by default)
3. If accounts use multiple currencies, a **Mixed currency warning** banner appears — totals sum native balances without FX conversion
4. **Account breakdown** table lists name, role, currency, balance, and % of total (same-currency only)
5. **Crypto placeholder** row shows "Connect exchanges — US-0007" — excluded from total
6. **Wealth over time** chart uses daily snapshots from post-sync upserts
7. **Stale sync** badge when last successful sync is older than ~25 hours

## 2. Alert types

| Type | Trigger |
|------|---------|
| **Scarcity** | Household aggregate forecast balance falls below threshold in next 45 days or current-month month-end |
| **Budget drift** | MTD category spend exceeds prorated active-plan target by more than drift % |
| **Plan viability** | Active plan projected month-end balance &lt; 0 (or two consecutive negative month-ends) |

Alerts deduplicate by fingerprint. When a condition clears, the alert resolves; a re-breach creates a new row.

## 3. Threshold configuration

Edit `backend/config/default.toml` (or env overlay):

```toml
[alerts]
scarcity_threshold_eur = 200.0
budget_drift_pct = 20.0
reporting_currency = "EUR"
```

On backend startup, thresholds mirror into PostgreSQL `alert_config` singleton. **Grafana Dashboard 1** (`cashflow`) uses template variable `$scarcity_threshold` from this table — changing TOML and restarting updates the scarcity reference line.

## 4. Header bell and `/alerts` inbox

- **Bell badge** shows count of `active` alerts where `acknowledged_at IS NULL`
- **Popover** previews latest 5 active alerts; link to full `/alerts` page
- **Acknowledge** — marks read; alert stays visible with muted styling while condition persists ("Acknowledged — condition still active")
- **Dismiss** — hides from bell and default list **until condition clears or re-triggers** (not permanent suppress)
- **Empty state** — "No active alerts"

Polling refreshes unread count every 30 seconds; sync completion invalidates alert queries.

## 5. US-0003 subscription alert boundary

Subscription detection alerts remain on **`/subscriptions`** (`subscription_alerts` table). The header bell popover optionally links "View subscription alerts (N)" when unread subscription alerts exist. They are **not** migrated into the unified inbox.

## 6. Grafana Dashboard 4

Open Grafana (`http://localhost:3000`):

| Dashboard | UID | Panels |
|-----------|-----|--------|
| Portfolio | `portfolio` | Total wealth stat, account count, mixed-currency warning, account breakdown table, wealth-over-time, crypto placeholder |

Datasource UID: `FlowFinancePostgreSQL`. Snapshots upsert once per calendar day during sync alerts phase.

Dashboard 1 (`cashflow`) scarcity line uses `$scarcity_threshold` from `alert_config`.

# Read-only guarantee

Wealth reads from mirrored Firefly **accounts** only. Alerts and snapshots are stored in the Flow Finance PostgreSQL database. **No writes to Firefly III.**

# Example API

Net worth breakdown:

```bash
curl -H "Authorization: Bearer $TOKEN" \
  http://localhost:8080/api/v1/wealth
```

Acknowledge alert:

```bash
curl -X PATCH -H "Authorization: Bearer $TOKEN" \
  http://localhost:8080/api/v1/alerts/{id}/acknowledge
```

Unread count (header bell):

```bash
curl -H "Authorization: Bearer $TOKEN" \
  http://localhost:8080/api/v1/alerts/unread-count
```

# Troubleshooting

| Symptom | Check |
|---------|-------|
| No scarcity alert | Forecast recompute succeeded; balance projection breaches threshold |
| No budget drift | Active plan with category-targeted adjustments; transactions synced MTD |
| Bell count stuck | Acknowledged alerts excluded; dismissed hidden |
| Dashboard 4 empty | At least one successful sync after US-0005 deploy |
| Mixed-currency banner | Expected when accounts use different currencies — no FX in MVP |
