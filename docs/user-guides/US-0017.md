# US-0017 — Omniflow smoke and troubleshooting quick reference

## Purpose

Operator-facing quick reference for validating the omniflow **external** profile
(`financegnome.omniflow.cc`) after deploy or regression fixes. Distills the root
[README Examples H3](../../README.md#omniflow-smoke-external-profile) and
[Troubleshooting H3](../../README.md#troubleshooting) without duplicating full runbook depth.

For initial external deploy, see [US-0010](US-0010.md). For ML overlay enablement, see
[US-0013](US-0013.md).

## Prerequisites

- US-0010 external profile operational on host `traefik` network
- Firefly PAT configured in operator `.env`
- Traefik `auth` basic-auth credentials (host-managed — not in this repo)
- OIDC session in SPA or `AUTH_DEV_BYPASS=true` for API-only curls

## Operator gate sequence

Complete **in order** before attributing flat analytics panels to code defects:

1. **BACKEND_FRONTEND_DEPLOY** — deploy current backend + frontend release on omniflow
2. **GRAFANA_PROVISIONING_RELOAD** — reload Grafana provisioning after dashboard JSON changes
3. **FULL_FIREFLY_SYNC** — Full sync (not exchanges-only) + forecast recompute

Detail and row-level probes:
[runbook §23 BUG-0013](../engineering/runbook.md#23-bug-0013-hotfix--omniflow-analytics-regression-q0020--released-2026-06-09).

## Omniflow smoke commands

Host: `https://financegnome.omniflow.cc` (override via `TRAEFIK_HOST`). Use placeholder
`-u '<basic-auth-user>:<pass>'` for Traefik edge routes only — never commit credentials.

```bash
OMNI=https://financegnome.omniflow.cc
AUTH='-u <basic-auth-user>:<pass>'

curl -sf "$OMNI/health" $AUTH
curl -s "$OMNI/api/v1/sync/status" | jq .
curl -s "$OMNI/api/v1/sync/entities" | jq .
curl -s -X POST "$OMNI/api/v1/sync/trigger" \
  -H 'Content-Type: application/json' \
  -d '{"mode":"full"}' | jq .
curl -s "$OMNI/api/v1/forecast/meta" | jq '.last_computed_at, .computation_id'
curl -s "$OMNI/api/v1/wealth" | jq '.crypto.subtotal_eur, .total_eur'
curl -s -o /dev/null -w '%{http_code}\n' "$OMNI/analytics/grafana/api/health" $AUTH
```

### Six analytics routes

| Route | Smoke focus |
|-------|-------------|
| `$OMNI/analytics/cashflow` | Baseline balances acct **114** (not **116**) |
| `$OMNI/analytics/subscriptions` | Price-changes panel or documented empty-state |
| `$OMNI/analytics/budgets` | MTD planned/actual plausible post-DEC-0079 |
| `$OMNI/analytics/portfolio` | Crypto stat non-zero post-DEC-0080 |
| `$OMNI/analytics/forecast-horizons` | Baseline + optional ML banner |
| `$OMNI/analytics/platform-health` | Stack health |

## Troubleshooting symptom table

| Symptom | Likely cause | Operator action |
|---------|--------------|-----------------|
| All analytics panels flat **0 €** after deploy | Stale image / gates skipped | Run full gate sequence above |
| Budgets MTD **−€150K** planned, **€0** actual | Pre-DEC-0079 MTD SQL artifact | Deploy DEC-0079 build + Grafana reload |
| Crypto **€0** in wealth/portfolio | Pre-DEC-0080 pricing or exchanges-only sync | Deploy DEC-0080 build + Full sync; probe `crypto.subtotal_eur` |
| Forecast **0 €** on default panels | Wrong `$account_id` or no recompute | Full sync; verify acct **114** (not **116**) |
| **ML unavailable** banner on forecast-horizons | ML overlay off (US-0013 / DEC-0049) | **Expected** — baseline statistical forecast still applies |
| Grafana **Failed to fetch** (browser console) | Embed annotation cancel / WS edge | curl ds/query **200**; Traefik session; do not Save dashboard overrides |

**Distinction:** empty Grafana SQL panels **after** the gate sequence indicate a data or deploy
defect; an **ML unavailable** banner is honest degraded mode when the stats-forecast sidecar is
disabled — not missing ledger data.
