# Purpose

US-0002 adds **cashflow forecasting** to Flow Finance AI: precomputed balance projections stored in TimescaleDB hypertables, a React `/forecast` page with ECharts charts, and Grafana analytics dashboards (Cashflow + Forecast Horizons). Forecasts recompute automatically after each successful Firefly sync.

# Prerequisites

- US-0001 platform running (Compose minimal profile, external PostgreSQL, Firefly sync)
- **TimescaleDB extension enabled** on the Flow Finance AI database (required for migration 002):
  ```sql
  CREATE EXTENSION IF NOT EXISTS timescaledb;
  ```
- At least one **asset account** with synced transactions (90+ days recommended for high-confidence forecasts)

# Usage steps

## 1. Trigger sync to populate forecasts

Forecasts are **not** computed on demand. After migration 002 applies at backend startup, run a sync:

1. Open Flow Finance AI → **Sync Status**
2. Click **Sync now**
3. Wait for sync to finish — the status shows **Syncing Firefly data…** then **Recomputing forecasts…** while the mutex is held
4. When idle, forecasts are available via API and UI

## 2. View forecasts in the React UI

1. Open **Forecast** in the sidebar (`/forecast`)
2. Select an asset account from the dropdown
3. Switch tabs:
   - **Daily** — milestone cards (tomorrow, next week, month-end) + current-month balance line chart
   - **Monthly** — grouped bar chart (income, fixed, variable, free cashflow) + summary stats
   - **Long-term** — 3 / 6 / 12 / 24 month horizon pills with balance path chart
4. Check **Last computed** timestamp; link to Sync Status if data is missing

## 3. Grafana dashboards

Open Grafana (`http://localhost:3000`, default admin/admin):

| Dashboard | UID | Folder | Purpose |
|-----------|-----|--------|---------|
| Cashflow | `cashflow` | Analytics | Balance time series + forecast overlay; €200 scarcity threshold (visual only) |
| Forecast Horizons | `forecast-horizons` | Analytics | End balances at 1 / 3 / 6 / 12 / 24 month horizons |

Use the **account_id** template variable to filter panels. All queries use the latest successful `computation_id`.

Datasource UID: `FlowFinancePostgreSQL` (must match provisioning YAML).

# Interpreting flags

| Flag | Meaning |
|------|---------|
| **stale** | Latest recompute failed; API serves the prior successful snapshot. Run sync again or check backend logs. |
| **low_confidence** | Account has fewer than 90 days of transaction history. Projection uses all available data with wider uncertainty — not an error. |

Both flags appear on `/forecast` meta and in `GET /api/v1/forecast/meta`.

# Example

After a successful sync, fetch daily milestones:

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "http://localhost:8080/api/v1/forecast/daily?account_id=1"
```

Response includes `milestones.tomorrow`, `milestones.next_week`, `milestones.month_end`, and a `series` array for the current month.

# Limitations

- Rule-based hybrid model only (recurring heuristics + 3-month rolling average) — no ML (US-0009)
- No subscription confirm/reject adjustments (US-0003)
- Aggregate endpoint sums same-currency asset accounts only
- Scarcity threshold on Dashboard 1 is visual only until US-0005 Alert Engine
- Concurrent sync triggers return **409** during both sync and forecast recompute

# Troubleshooting

| Issue | Fix |
|-------|-----|
| Migration 002 fails | Ensure `CREATE EXTENSION timescaledb` on external PostgreSQL |
| Empty `/forecast` page | Run Sync Status → Sync now; verify transactions exist for asset accounts |
| `stale: true` in meta | Check backend logs for forecast recompute errors; prior snapshot still served |
| `low_confidence: true` | Normal for new/sparse accounts; accuracy improves with more history |
| Grafana "datasource not found" | Verify datasource uid `FlowFinancePostgreSQL` in provisioning |
| 409 on sync trigger | Wait for running sync + forecast recompute to finish |
