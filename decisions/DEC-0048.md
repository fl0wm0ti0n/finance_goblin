# Decision DEC-0048

**Status:** Accepted  
**Date:** 2026-06-02  
**Story:** US-0008  
**Research:** R-0042, R-0030  
**Extends:** DEC-0034 audit schema, US-0008 AC5

## Context

Operators need audit traceability by provider. AC5 requires automated proof that local configuration produces **zero outbound requests to `api.openai.com`**. Discovery asked whether to encode provider in model string vs dedicated column (R-0042).

## Options

1. **Migration 008 `provider` column + wiremock CI guard** — recommended
2. **Encode provider in `model` field prefix** — rejected (breaks operator queries)
3. **DNS-level outbound block in Rust** — rejected (OS concern)
4. **Manual tcpdump only for AC5** — rejected (not CI-automatable)

## Recommendation

**Migration `008_ai_audit_provider.sql`:**

```sql
ALTER TABLE ai_tool_audit ADD COLUMN IF NOT EXISTS provider TEXT;
CREATE INDEX IF NOT EXISTS ai_tool_audit_provider ON ai_tool_audit (provider, created_at DESC);
```

Populate `provider` from `AiProvider::name()` on each tool invocation audit insert. Existing rows: `NULL` or backfill `'openai'` optional at migration time.

**AC5 verification — two layers:**

1. **CI (required):** wiremock registers `https://api.openai.com` → test **panics/fails if matched**. Configure `AiConfig { provider: "ollama", base_url: "http://127.0.0.1:<wiremock_port>/v1" }`. Run orchestrator `complete()` with mocked Ollama-compat response including `tool_calls`. Assert OpenAI mock **zero requests**, local mock **≥1**.

2. **Operator UAT (documented):** `docker compose --profile full up` with `[ai] provider = "ollama"`; pull model; run US-0006 example query. Optional manual tcpdump — not CI gate.

Automated network namespace blocking deferred (fragile in CI).

## Impact

- New migration 008
- `backend/src/ai/orchestrator.rs` — pass provider name to audit insert
- `backend/tests/` — new local provider isolation test with wiremock
- Operator user guide US-0008 — Compose full profile steps

## Risks

- Wiremock test doesn't catch misconfigured `base_url` pointing at OpenAI proxy — factory presets must never default local to `api.openai.com` (DEC-0043)
- Migration additive only — low rollback risk
