# Decision DEC-0012

**Status:** Accepted  
**Date:** 2026-05-31  
**Story:** US-0002, US-0005  
**Research:** R-0008

## Context

Grafana Dashboard 1 needs scarcity threshold reference lines. US-0005 Alert Engine will centralize threshold config. R-0008 notes Grafana dashboard JSON cannot use env substitution for panel thresholds.

## Options

1. **Static default €200 in panel threshold** — visual only until US-0005
2. **Grafana dashboard variable from DB config table** — extra schema for MVP
3. **No scarcity markers in US-0002** — rejected (acceptance + discovery scope)
4. **Prometheus metrics duplicate** — rejected (data already in TimescaleDB)

## Recommendation

**Static default scarcity threshold €200** on Dashboard 1 panels as Grafana threshold/reference line. Document in panel description that US-0005 Alert Engine will wire dynamic config.

Also standardize provisioning identifiers (R-0008):

- Datasource explicit `uid: FlowFinancePostgreSQL` in `postgres.yaml`
- Dashboard 1: `uid: cashflow`, folder `Analytics`
- Dashboard 5: `uid: forecast-horizons`, folder `Analytics`
- Second dashboard provider block or `folder` field in JSON for Analytics vs Platform

## Impact

- Update existing datasource YAML during US-0002 execute
- Threshold duplication until US-0005 — acceptable visual-only scope

## Risks

- uid mismatch breaks panels if datasource YAML not updated atomically with dashboard JSON
- Operator currency not EUR — threshold label may mislead; use generic "Scarcity threshold" title
