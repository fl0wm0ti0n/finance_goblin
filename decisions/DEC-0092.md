# DEC-0092 — Per-plan goal-stats API, yearly rollup, and feasibility copy (US-0019)

**Status:** Accepted  
**Date:** 2026-06-09  
**Story:** US-0019  
**Research:** [R-0084 §2, §6](research.md#r-0084--us-0019-goal-plans-per-plan-stats-category-overlay--ai-savings)  
**Extends:** DEC-0073 (overlay monthly delta), DEC-0019 (plan_daily_cashflow), **DEC-0096** (PVA scope)

## Context

US-0019 AC-2 requires statistics **scoped to the selected plan only** — monthly delta vs baseline, yearly rollup, projected balance at target date — not household-wide aggregates on plan detail.

## Decision

### 1. New endpoint (frozen)

`GET /api/v1/plans/{plan_id}/goal-stats?version_id={optional}`

- Defaults `version_id` to latest successful computation version when omitted
- Returns **404** when plan `template != 'goal_balance'`
- Keyed by `plan_id` + `version_id` — never aggregates across plans (AC-2)

### 2. Response fields

| Field | Source |
|-------|--------|
| `target_balance_eur`, `target_date`, `goal_account_id` | `plans` row |
| `monthly_delta_vs_baseline` | Existing `build_compare_metrics` overlay delta (DEC-0073) for this version |
| `yearly_rollup[]` | Calendar year `SUM(planned_net)` from `plan_daily_cashflow` for successful `computation_id` |
| `projected_balance_at_target` | Last `planned_balance` where `ts::date <= target_date` (no interpolation) |
| `gap_eur` | `target_balance_eur - projected_balance_at_target` when projection present |
| `on_track` | `projected_balance_at_target >= target_balance_eur` |
| `beyond_horizon` | `true` when `target_date > today + 730d` (`overlay_horizon_end`) |
| `required_monthly_savings_eur` | `ceil(gap_eur / months_remaining)` when `gap_eur > 0` and months > 0; **0% interest** (no PMT) |

When `beyond_horizon`: `projected_balance_at_target: null`; UI shows horizon copy — no silent extrapolation.

### 3. Yearly rollup grain

**Calendar year** `SUM(planned_net)` — aligns with US-0018 month labels. Rolling 12m rejected.

Full scenario yearly sums only (not overlay-only synthetic series) for MVP; overlay monthly delta remains separate field.

### 4. Feasibility copy (no auto-lines)

UI displays `gap_eur` and `required_monthly_savings_eur` as **informational copy only** — does **not** auto-insert savings adjustment lines (PO recommendation).

Annuity PMT with savings rate **deferred** — no product savings-rate config today.

### 5. UI placement

- Scenarios summary: goal stats strip when `template=goal_balance`
- Compare tab: per-**selected** plan stats above version table
- PVA tab: **unchanged** household active plan per **DEC-0096**

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| Extend `/compare` only | Overloads compare payload; goal plans need dedicated contract |
| Rolling 12m yearly rollup | Mismatches calendar month UX |
| PMT + auto-insert adjustments | Scope creep; operator applies cuts explicitly (AC-4) |
| Interpolate balance at target_date | Misleading vs daily projection points |

## Risks

- Stale `plan_computations` if recompute pending — surface `computed_at` in response meta
- Month-end vs arbitrary date — last daily point ≤ target_date only
- Goal account vs household compare mismatch — document in UI; goal-stats uses DEC-0095 account scope
