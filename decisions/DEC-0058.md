# Decision DEC-0058

**Status:** Accepted  
**Date:** 2026-06-03  
**Story:** US-0012  
**Research:** R-0055, R-0053 ¬ß1  
**Extends:** DEC-0003 (external DB startup retry), DEC-0056 (omniflow external Postgres preflight ‚Äî DB create step automated; TimescaleDB host install unchanged)

## Context

US-0010 external deploy requires database `flow_finance_ai` on shared host `postgres` before `docker compose up`. Operators today run manual `CREATE DATABASE` + grants + extension SQL (runbook ¬ß Omniflow, `.env.example` comments). Backend startup (`backend/src/lib.rs`) calls `DbPool::connect_with_retry` then `run_migrations` ‚Äî missing database (`SQLSTATE 3D000`) exhausts retry budget with no bootstrap hook. Intake and R-0055 recommend in-app pre-migration bootstrap with optional admin maintenance URL when app role lacks `CREATEDB`.

## Options

### 1. In-app pre-migration bootstrap (recommended)

New `DbPool::ensure_database(&AppConfig)` in `backend/src/db/bootstrap.rs`, invoked from `run()` **before** `connect_with_retry`:

```
maintenance connect ‚Üí existence check ‚Üí [CREATE DATABASE OWNER app_user] ‚Üí extension on app DB ‚Üí app connect_with_retry ‚Üí run_migrations
```

**Alternative:** Compose one-shot init container ‚Äî rejected (extra service; duplicates env on external profile per R-0055).

**Alternative:** script-only bootstrap ‚Äî rejected (not automatic on first start per AC-1).

### 2. Maintenance URL resolution (recommended)

| Priority | Source | Notes |
|----------|--------|-------|
| 1 | `DATABASE_BOOTSTRAP_URL` (optional env) | Full URL to maintenance DB (typically `‚Ä¶/postgres`); admin/superuser; **never committed** |
| 2 | Derived from `DATABASE_*` | `postgres://{USER}:{PASSWORD}@{HOST}:{PORT}/postgres` ‚Äî same creds as app |

Runtime `DATABASE_*` remain app connection after bootstrap. No `DATABASE_URL` override for bootstrap (use `DATABASE_BOOTSTRAP_URL` only).

**Alternative:** require bootstrap URL always ‚Äî rejected (greenfield dev with `CREATEDB` app role needs zero extra env).

### 3. Create semantics ‚Äî `OWNER` not separate `GRANT` (recommended)

When database absent:

```sql
CREATE DATABASE "{validated_name}" OWNER "{app_user}";
```

- Idempotent: skip when `pg_database.datname` already exists ‚Äî **never** `DROP` / recreate.
- `OWNER` assigns database ownership in one step (preferred over superuser create + `GRANT ALL` on shared homelab `postgres`).
- When bootstrap user equals app user with `CREATEDB`, same statement applies.
- When DB already exists with different owner, skip create; extension step still runs if extension missing.

**Alternative:** superuser create + `GRANT ALL PRIVILEGES ON DATABASE` ‚Äî rejected (extra privilege step; OWNER is simpler and sufficient for app migrations).

### 4. TimescaleDB extension step (recommended)

After DB exists (created or pre-existing):

1. Connect to **app database** using **maintenance credentials** (bootstrap URL user or derived ‚Äî not app pool).
2. `SELECT extversion FROM pg_extension WHERE extname = 'timescaledb'`.
3. If absent: `CREATE EXTENSION IF NOT EXISTS timescaledb` (no `CASCADE` required for MVP).
4. Fail closed with `database_bootstrap_failed_timescaledb` when extension files unavailable ‚Äî pointer runbook US-0010 ¬ß1 / R-0053 ¬ß1.

Migration `001_initial.sql` keeps `CREATE EXTENSION IF NOT EXISTS timescaledb` ‚Äî harmless duplicate when bootstrap succeeded.

**Alternative:** remove extension from migration 001 for external mode ‚Äî rejected (breaks US-0002‚ÄìUS-0009).

**Alternative:** extension via app role ‚Äî rejected (app role often lacks `CREATE EXTENSION` even when it owns DB on shared hosts).

### 5. `DATABASE_NAME` validation (recommended)

Validate at `AppConfig::load()` before any dynamic SQL:

- Allowlist: `^[a-zA-Z_][a-zA-Z0-9_]*$` (default `flow_finance_ai` passes).
- Reject with `ConfigError::MissingEnv` / dedicated invalid-name error ‚Äî fail before network.

Use PostgreSQL quoted identifiers from validated name only ‚Äî no string concatenation from untrusted input.

### 6. Retry budget (recommended)

Maintenance connection reuses existing `database.startup_retry_*` fields from DEC-0003 (same initial/max/total ms). Bootstrap is part of startup; one operator knob.

**Alternative:** separate shorter maintenance budget ‚Äî deferred unless execute shows maintenance connect dominates startup time.

### 7. Structured log reason codes (recommended)

Stable tracing field **`bootstrap_reason`** (operator/CI contract):

| Code | Level | When |
|------|-------|------|
| `database_bootstrap_started` | info | Maintenance pool connected |
| `database_bootstrap_created` | info | `CREATE DATABASE` executed |
| `database_bootstrap_skipped_exists` | info | DB already present |
| `database_bootstrap_grants_applied` | info | Log when `OWNER` set on create (bootstrap user ‚âÝ app user) |
| `database_bootstrap_extension_ok` | info | Extension present or created |
| `database_bootstrap_failed_privilege` | error | `42501` / insufficient privilege on create ‚Äî hint `DATABASE_BOOTSTRAP_URL` |
| `database_bootstrap_failed_timescaledb` | error | Extension unavailable ‚Äî runbook TimescaleDB host install |
| `database_bootstrap_failed_connect` | error | Maintenance DB unreachable |

Log host, port, maintenance db name, app db name, app user ‚Äî **never** passwords or full URLs with credentials.

## Recommendation

Accept DEC-0058 as the US-0012 bootstrap contract:

| Sub-decision | Execute requirement |
|--------------|---------------------|
| Hook placement | `ensure_database` ‚Üí `connect_with_retry` ‚Üí `run_migrations` in `backend/src/lib.rs` |
| Module | `backend/src/db/bootstrap.rs` + config helpers in `backend/src/config/mod.rs` |
| Env | Optional `DATABASE_BOOTSTRAP_URL`; derived maintenance URL fallback |
| Create | `CREATE DATABASE ‚Ä¶ OWNER TO app_user`; existence check via parameterized `pg_database` query |
| Extension | Maintenance creds on app DB before migrations; fail-closed reason code |
| Validation | `DATABASE_NAME` allowlist at config load |
| Retry | Share DEC-0003 `startup_retry_*` for maintenance connect |
| Logs | `bootstrap_reason` codes; redact secrets |
| Docs | `.env.example` + runbook ¬ß Omniflow ‚Äî replace manual `CREATE DATABASE` with bootstrap env; TimescaleDB **host** install remains operator step |

**Amends DEC-0056 operator preflight:** step ‚Äúcreate database `flow_finance_ai`‚Äù ‚Üí automatic on first backend start; steps for TimescaleDB **server packages** + `shared_preload_libraries` unchanged.

## Risks

| Risk | Mitigation |
|------|------------|
| Bootstrap URL logged with password | Redact in all log paths; log host/db/user only |
| Shared host grant/owner semantics | `OWNER` on create; skip recreate when DB exists |
| TimescaleDB absent after DB create | Fail before migrations with `database_bootstrap_failed_timescaledb` + runbook pointer |
| Wrong `DATABASE_PASSWORD` after bootstrap | Document unchanged ‚Äî app connect still fails (intake constraint) |
| Dynamic SQL injection via `DATABASE_NAME` | Allowlist validation at config load |
| Maintenance retry consumes full startup budget | Shared DEC-0003 config; bootstrap runs once per process |
