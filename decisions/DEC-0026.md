# Decision DEC-0026

**Status:** Accepted  
**Date:** 2026-05-31  
**Story:** US-0005  
**Research:** R-0022  
**Extends:** DEC-0007, DEC-0019, DEC-0021

## Context

Alert Engine must evaluate scarcity, budget drift, and plan viability after sync. Discovery open questions: per-account vs household scarcity scope; budget drift grain (Firefly category vs plan-targeted only); plan viability rule definition.

## Options

1. **Household scarcity + category-targeted budget drift + month-end plan viability** — per R-0022
2. **Per-account scarcity alerts** — rejected (noisy for multi-account households; Dashboard 1 retains per-account visual)
3. **All Firefly categories for budget drift** — rejected (no plan target = false positives per R-0017)
4. **N consecutive daily below-zero for plan viability** — rejected (too sensitive to mid-month timing)

## Recommendation

**Scarcity (household aggregate):**
- Sum `forecast_balance_daily.balance` across all asset accounts per projected day
- Fire when any day in `[today, today+45d]` OR current-month month-end falls below `scarcity_threshold_eur`
- Severity: `critical` if breach tomorrow or already below; else `warning`
- Entity ref: `household`; fingerprint: `scarcity:household:{YYYY-MM}`

**Budget drift (active-plan category targets only):**
- For each active plan adjustment with `target_type = category`, compare MTD actual expense vs prorated plan target
- Fire when `actual > target × (1 + budget_drift_pct/100)` (default +20%)
- Entity ref: `category:{firefly_id}`; fingerprint: `budget_drift:category:{id}:{YYYY-MM}`

**Plan viability (active plan overlay):**
- Primary: projected `planned_balance` at current month-end < 0
- Secondary: 2 consecutive month-end balances (current + next) both < 0
- Requires active plan; uses plan overlay not raw Ist forecast
- Entity ref: `plan:{plan_id}:version:{version_id}`

## Impact

- `AlertService::evaluate` reads forecast, plan, mirror transactions, and `AlertsConfig`
- Dashboard 1 per-account threshold line remains visual-only per account selector

## Risks

- Category MTD proration skewed by one-time mid-month plan deltas
- Household scarcity masks single-account overdraft risk
- Plan viability on stale baseline if forecast/plan recompute failed
