# Decision DEC-0057

**Status:** Accepted  
**Date:** 2026-06-02  
**Story:** US-0011  
**Research:** R-0054 (extends DEC-0012, DEC-0056, R-0052, R-0053, R-0056)  
**Supersedes:** ad-hoc `VITE_GRAFANA_URL` external-tab pattern on WealthPage

## Context

US-0011 embeds six provisioned Grafana dashboards inside the financegnome SPA at `https://financegnome.omniflow.cc`. US-0010 / DEC-0056 keeps Grafana **internal-only** on the Docker `traefik` network (no default `GRAFANA_TRAEFIK_HOST`). Browser iframes cannot reach `http://grafana:3000` directly; a **same-origin reverse proxy** on `flow-finance-ai` is required (R-0054).

Discovery defined a canonical dashboard → React route map and proxied kiosk URLs. Open questions (Grafana auth, CSP, subpath, WebSocket) are resolved here for execute.

## Options

### 1. Public Grafana host + iframe (`GRAFANA_TRAEFIK_HOST`)

Expose Grafana on a second Traefik host; iframe from SPA.

- **Rejected as default:** double basic-auth friction, extra attack surface, violates AC-7 (single-URL UX). Remains optional operator escape hatch via existing DEC-0056 gate only — not used by US-0011 embed paths.

### 2. Backend same-origin reverse proxy (recommended)

Mount proxy on financegnome origin at **`/analytics/grafana/`** → upstream **`http://grafana:3000/`** with prefix strip, WebSocket upgrade, and framing-header rewrite.

- **Accepted:** one Traefik `auth` context; OIDC session on SPA unchanged; iframe `src` same-origin.

**Alternative:** `/api/v1/analytics/grafana/*` — rejected (mixes with JWT API namespace; complicates `require_auth` layering and static asset paths).

### 3. Grafana subpath serve (`GF_SERVER_SERVE_FROM_SUB_PATH`)

Configure Grafana with `root_url` under `/analytics/grafana`.

- **Rejected for MVP:** path rewriting complexity for provisioned links, live WS, and static assets (R-0054; discovery open question). Use **root upstream proxy** with path-prefix strip instead.

### 4. Grafana auth modes

| Mode | Fit |
|------|-----|
| Anonymous viewer + org role Viewer | **Accepted** — trust boundary = Traefik `auth` + financegnome OIDC; Grafana not internet-routable |
| Auth-proxy headers from OIDC | Deferred — needs header contract with IdP; out of US-0011 MVP |
| Service account token in proxy | Rejected — token leakage risk in browser-visible iframe traffic |

### 5. Full React port of dashboards

- **Rejected** (US-0011 scope) — phased follow-up per dashboard (R-0054).

## Recommendation

Accept **DEC-0057** as the unified analytics proxy and embed contract:

| Contract element | Value |
|------------------|-------|
| Public proxy prefix | `/analytics/grafana/` (trailing slash on upstream join) |
| Upstream env | `GRAFANA_UPSTREAM` default `http://grafana:3000` |
| SPA embed base | `VITE_GRAFANA_EMBED_BASE` default `/analytics/grafana` (build-time) |
| Iframe path pattern | `{embed_base}/d/{uid}/{slug}?kiosk=tv` |
| React shell routes | `/analytics/{slug}` — six slugs per discovery map |
| Deprecate | `VITE_GRAFANA_URL` — remove Wealth external-tab default |
| Grafana env (compose) | `GF_AUTH_ANONYMOUS_ENABLED=true`, `GF_AUTH_ANONYMOUS_ORG_ROLE=Viewer`, `GF_SECURITY_ALLOW_EMBEDDING=true`, `GF_USERS_ALLOW_SIGN_UP=false` (existing) |
| Proxy behavior | Strip request prefix; forward method/body/query; upgrade WebSocket; remove or replace `X-Frame-Options: deny` on responses; do not forward `Set-Cookie` from Grafana to browser |
| SPA CSP | `frame-src 'self'` sufficient when embed base is same-origin; no third-party Grafana host in default build |
| Auth middleware | Proxy route **outside** `/api/v1` JWT stack — edge Traefik basic-auth (+ optional OIDC on SPA) is the gate; anonymous Grafana behind proxy only |
| Future charts | React + API default; Grafana embed exception list in user guide (AC-6) |

## Impact

- `backend` — new `analytics` proxy router merged in `build_router` before SPA fallback
- `frontend` — `AnalyticsEmbedPage`, routes, sidebar group, Wealth link
- `docker-compose.yml` / `.env.example` — `GRAFANA_UPSTREAM`, Grafana anonymous env block, `VITE_GRAFANA_EMBED_BASE`; document deprecated `VITE_GRAFANA_URL`
- `docs/user-guides/US-0011.md` — single-URL UX + future-chart guideline (execute)
- No change to Grafana JSON/SQL definitions (DEC-0012 uids)

## Risks

| Risk | Mitigation |
|------|------------|
| Anonymous Grafana behind weak edge auth | Keep DEC-0056 internal-only; require Traefik `auth` on production; document dev bypass path |
| WebSocket/live refresh fails through proxy | Explicit upgrade forwarding in proxy; smoke iframe load in QA |
| Panel links break (absolute Grafana URLs) | Root proxy + `GF_SERVER_ROOT_URL` default; fix in execute if smoke finds broken assets |
| Duplicate metrics confusion | Document canonical UX table above in user guide |
| Proxy SSRF if upstream misconfigured | `GRAFANA_UPSTREAM` server-side only; validate URL host allowlist (`grafana`, localhost dev) |

## Decision gate

If omniflow smoke shows anonymous Grafana insufficient (org switching, edit affordances), escalate **auth-proxy** sub-decision — do not enable public `GRAFANA_TRAEFIK_HOST` without new DEC.
