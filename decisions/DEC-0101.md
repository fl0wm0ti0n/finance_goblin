# DEC-0101 — Operator tag schema (US-0020)

**Status:** Accepted  
**Date:** 2026-06-10  
**Story:** US-0020  
**Research:** [R-0085 §4](research.md#r-0085--us-0020-subscription-discover-majority-category--operator-tags)  
**Extends:** product DB overlay posture (DEC-0004 read-only Firefly)

## Context

US-0020 AC-4/AC-5 require operator-defined tags (e.g. luxus, important) stored in product DB with CRUD, attachable to confirmed subscriptions, filterable on list — no Firefly tag write-back.

## Decision

### 1. Tables (frozen)

```sql
CREATE TABLE operator_tags (
    id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name       TEXT NOT NULL,
    slug       TEXT NOT NULL UNIQUE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE subscription_pattern_tags (
    pattern_id UUID NOT NULL REFERENCES subscription_patterns(id) ON DELETE CASCADE,
    tag_id     UUID NOT NULL REFERENCES operator_tags(id) ON DELETE CASCADE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    PRIMARY KEY (pattern_id, tag_id)
);

CREATE INDEX idx_subscription_pattern_tags_tag ON subscription_pattern_tags(tag_id);
```

- `slug` = lowercase trimmed name; spaces → hyphen; UNIQUE dedupe
- **Global** operator namespace (not per-account)

### 2. CRUD API (frozen)

| Method | Path | Contract |
|--------|------|----------|
| `POST` | `/api/v1/subscription-tags` | `{ "name": "luxus" }` → create |
| `GET` | `/api/v1/subscription-tags` | list all tags |
| `PATCH` | `/api/v1/subscription-tags/:id` | rename; slug regenerated |
| `DELETE` | `/api/v1/subscription-tags/:id` | **hard delete**; CASCADE removes junction rows |

Tag delete: confirm dialog in UI — assignments removed; patterns unaffected.

### 3. Delete semantics

**Hard delete** + `ON DELETE CASCADE` on junction — no `deleted_at` soft-delete for MVP.

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| JSON array on `subscription_patterns` | No normalized filter/index; poor Grafana join |
| Soft-delete tags | Complicates unique name constraint; low volume |
| Per-account tag namespace | PO global operator MVP |
| Firefly tag sync | Violates AC-5 |

## Risks

- Tag delete surprise — UI confirm dialog
- Slug collision on rename — enforce UNIQUE; return 409

## Implementation anchor

- `backend/migrations/`
- `backend/src/subscriptions/tags.rs` (new module) or extend `repository.rs`
- `backend/src/api/subscription_tags.rs` (new router)
