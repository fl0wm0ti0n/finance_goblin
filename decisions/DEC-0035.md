# Decision DEC-0035

**Status:** Accepted  
**Date:** 2026-05-31  
**Story:** US-0006  
**Research:** R-0031  
**Extends:** DEC-0019 (plan overlay), DEC-0026 (budget drift rules), DEC-0004

## Context

Six AI tools must wrap existing services without SQL or Firefly access. Discovery asked `simulate_plan` mapping, missing transactions API, and payload size limits.

## Options

1. **In-process service calls + new TransactionsService + ephemeral plan projection** — per R-0031
2. **HTTP self-calls to REST handlers** — rejected (latency, auth recursion)
3. **`simulate_plan` active plan only** — rejected (template what-ifs need ephemeral path)
4. **Persist ephemeral simulations** — rejected (pollutes plan list)
5. **Unbounded tool JSON to model** — rejected (context window)

## Recommendation

**Tool → service mapping:**

| Tool | Service |
|------|---------|
| `get_transactions` | NEW `TransactionsService::aggregates` |
| `get_subscriptions` | `SubscriptionService::list_patterns` |
| `get_forecast` | `ForecastService` latest computation |
| `get_budget_status` | Compose from active plan + MTD actuals + `AlertService` rules (R-0022) |
| `get_portfolio` | `WealthService::compute_breakdown` |
| `simulate_plan` | `PlanService::project_ephemeral` / `project_readonly` |

**`simulate_plan` resolution:** plan_id → template → active plan → error.

**Payload cap:** max **8192 bytes** per tool result; downsample series to 30 points; `{ truncated, summary }` on overflow.

## Impact

- New `transactions/` module in backend
- `PlanService` gains read-only projection methods sharing `overlay` math

## Risks

- `get_budget_status` must stay aligned with Alert Engine — share evaluation inputs
- Ephemeral projection drift if overlay diverges — single module source of truth
