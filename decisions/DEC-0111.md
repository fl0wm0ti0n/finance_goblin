# DEC-0111 — Wealth account_role attributes path + display labels (BUG-0021 EB)

**Status:** Accepted  
**Date:** 2026-06-11  
**Work item:** BUG-0021 (sub-defect **EB**, acceptance **BL**)  
**Research:** [R-0091](../docs/engineering/research.md#r-0091--bug-0021-categoryfilter-lazy-load-delay--wealth-role-column-empty), [R-0001](../docs/engineering/research.md#r-0001--wealth-net-worth-mirror-contract)  
**Extends:** **DEC-0021** (wealth mirror read path); **DEC-0065** (wealth API assembly — field passthrough unchanged)  
**Supersedes:** none

## Context

R-0091 confirmed Firefly sync stores the full API **item** in `accounts.payload` with `account_role` under `attributes`, but `WealthRepository::load_asset_accounts` reads `payload->>'account_role'` at JSON root — always null for 3/3 probe asset accounts. API, daily snapshot `payload.accounts[]`, and Grafana `portfolio.json` (`elem->>'account_role'`) inherit null. Frontend correctly renders em dash; bug is extraction path + opaque enum display.

## Options

| Option | Verdict | Rationale |
|--------|---------|-----------|
| **EB-A — SQL attributes path + COALESCE root fallback** | **Accepted** | Matches sync storage and `plan/repository.rs` nested pattern; no migration; snapshot/Grafana heal on next upsert |
| **EB-B — frontend human label map on raw enum** | **Accepted** (with A) | BL requires useful column — map Firefly roles to operator labels; unknown enum → show raw value |
| EB-C — denormalize at sync (`upsert_account`) | Rejected | Duplicates data; diverges from Firefly item shape |
| EB-D — Rust service-layer parse only | Rejected | Duplicates path logic; Grafana SQL still broken |
| EB-E — hide Role column when null | Rejected | Fails BL when metadata exists post-fix; workaround not fix |
| EB-F — API DTO label field only (no frontend map) | Rejected | Unnecessary API churn; frontend map sufficient for single table column |

## Decision

Accept **DEC-0111** as the wealth `account_role` contract for BUG-0021:

| Contract element | Value |
|------------------|-------|
| SQL extract | `COALESCE(payload->'attributes'->>'account_role', payload->>'account_role') AS account_role` in `load_asset_accounts` and matching test SQL constant (`repository.rs` L133) |
| API shape | `GET /api/v1/wealth` → `firefly.accounts[*].account_role` carries **raw Firefly enum** string when mirror provides it (`defaultAsset`, `cashWalletAsset`, `savingAsset`, etc.) |
| UI display | `WealthPage` Account breakdown **Role** column uses frontend helper `formatAccountRole(raw)` — human label when mapped, raw enum when unknown, em dash only when API value is null/absent |
| Label map (canonical) | `defaultAsset` → "Checking"; `cashWalletAsset` → "Cash wallet"; `savingAsset` → "Savings"; `sharedAsset` → "Shared"; `ccAsset` → "Credit card" (per R-0001 / Firefly account types) |
| Snapshots | `upsert_daily_snapshot` inherits fixed API row — **no separate snapshot SQL change**; roles populate on next daily upsert after deploy |
| Grafana | `portfolio.json` panel SQL unchanged (`elem->>'account_role'` from snapshot payload) — auto-heals after snapshot upsert; optional operator manual sync for immediate proof |
| Forbidden | Sync-time payload rewrite; hiding Role column; hardcoded account ids; Firefly write-back |

### SQL sketch

```sql
SELECT firefly_id, name, currency, balance::float8 AS balance,
       COALESCE(
         payload->'attributes'->>'account_role',
         payload->>'account_role'
       ) AS account_role
FROM accounts
WHERE type = 'asset'
  AND COALESCE((payload->>'active')::boolean, true) = true
  AND COALESCE((payload->>'include_net_worth')::boolean, true) = true
ORDER BY name;
```

### Frontend label helper sketch

```typescript
const ACCOUNT_ROLE_LABELS: Record<string, string> = {
  defaultAsset: "Checking",
  cashWalletAsset: "Cash wallet",
  savingAsset: "Savings",
  sharedAsset: "Shared",
  ccAsset: "Credit card",
};

export function formatAccountRole(role: string | null | undefined): string {
  if (!role) return "—";
  return ACCOUNT_ROLE_LABELS[role] ?? role;
}
```

| OIDC | Standard omniflow smoke on `/api/v1/wealth` and `/wealth` |

**Post-fix probe:**

```sql
SELECT name,
       payload->>'account_role' AS root_role,
       payload->'attributes'->>'account_role' AS attr_role,
       COALESCE(payload->'attributes'->>'account_role', payload->>'account_role') AS effective_role
FROM accounts WHERE type = 'asset' ORDER BY name;
```

## Risks

| Risk | Mitigation |
|------|------------|
| Legacy rows with only root-level role | COALESCE root fallback |
| Snapshot/Grafana lag until next daily upsert | Document in UAT; operator manual sync optional |
| Label map incomplete for rare roles | Fallback to raw enum (not em dash) |
| Root-level `active`/`include_net_worth` same bug class | Out of scope — filters masked by COALESCE true; note for future hygiene |

## Rollback

`git revert` repository + WealthPage label helper. No migration. Snapshots retain last-written role values until next upsert.
