# Decision DEC-0063

**Status:** Accepted  
**Date:** 2026-06-05  
**Bug:** BUG-0005  
**Research:** R-0059  
**Extends:** DEC-0040 (exchange secrets/config pattern); parity with `BitunixConfig::effective_enabled()` (BUG-0003 G1)

## Context

`enabled_futures = false` in `default.toml` gates all futures ingestion even when operator has Bitunix API credentials configured. Live omniflow settings show `enabled_futures: false` while operator expects multi-product sync. Acceptance **N** requires futures endpoints populate data — not remain empty stubs behind default alone.

## Options

### 1. `effective_enabled_futures()` runtime override (recommended)

| Priority | Rule |
|----------|------|
| 1 | Env `BITUNIX_ENABLED_FUTURES=false` → futures **off** (explicit spot-only opt-out) |
| 2 | Env `BITUNIX_ENABLED_FUTURES=true` → futures **on** |
| 3 | TOML `enabled_futures = true` → futures **on** |
| 4 | `effective_enabled() && credentials()` → futures **on** (auto-enable) |
| else | futures **off** |

Settings API exposes **effective** value, not raw TOML default.

**Alternative:** Flip TOML default to `true` — rejected (silent behavior change for deployments without credentials).

**Alternative:** Manual operator flag only — rejected (root cause is unknown flag blocking ingestion).

### 2. Gate all futures paths

`sync_balances` futures branch, `sync_positions`, `sync_funding`, and `test_connection` futures probe use `effective_enabled_futures()`.

## Recommendation

Accept options **1** and **2**. Execute in Q0012 task **N3** before M1/N2/N4.

## Impact

- `BitunixConfig::effective_enabled_futures()` in `backend/src/config/mod.rs`
- Settings view + `.env.example` document `BITUNIX_ENABLED_FUTURES` opt-out

## Risks

- Spot-only operators must set env opt-out — document in runbook
- Auto-enable only when credentials present — limits unintended fapi traffic
