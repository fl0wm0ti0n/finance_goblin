# DEC-0113 — Discover dual-mode UX (US-0021)

**Status:** Accepted  
**Date:** 2026-06-13  
**Story:** US-0021  
**Research:** [R-0092 §6.1](docs/engineering/research.md#r-0092--us-0021-subscription-transaction-explorer-vs-recurrence-only-discover)  
**Extends:** **DEC-0098** (Suggested patterns tab = existing discover contract, unchanged)

## Context

Operator intake expects **transaction-first** subscription search (browse individual expenses, rich filters, manual tx-group activate). US-0020 shipped recurrence-candidate Discover (`DiscoverCandidate` rows) — valuable but insufficient for AC-1..AC-4. Replacing the patterns table entirely would regress US-0020 AC-1 narrow contract.

## Decision

### 1. Layout (frozen)

Discover tab on `/subscriptions` uses **dual sub-mode** segmented control:

| Mode | Default | API | Result unit |
|------|---------|-----|-------------|
| **Transactions** | **Yes** | `GET /transactions/search` (DEC-0112) | Individual expense tx rows |
| **Suggested patterns** | No | `GET /discover` (DEC-0098 unchanged) | `DiscoverCandidate` groups |

**Default on tab visit:** Transactions mode.

### 2. Transactions mode UX (frozen)

- Shared filter bar: **account** (required), **payee**, **category** (`CategoryFilter`), **Geldbereich** (`formatAccountRole` labels per DEC-0111), **date range**
- Optional P2: amount band inputs
- Paginated table: date, description, amount, category, Geldbereich, optional **hint badge** (DEC-0114)
- Row **multi-select** checkboxes → confirm modal (AC-4)
- Pagination controls + truncated banner per DEC-0112

### 3. Suggested patterns mode UX (frozen)

- **No contract change** from US-0020 / DEC-0098
- Filters: account, payee, interval bucket only
- Candidate-row confirm modal unchanged
- Heading may clarify "Suggested recurring patterns" vs Transactions "Search transactions"

### 4. Shared elements

- Account selection persists when switching sub-modes (same account context)
- Payee filter may persist where applicable (Transactions + Patterns both support payee)
- Category / Geldbereich / date filters **Transactions-only** — hidden in Patterns mode

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| Replace recurrence table entirely | Regresses US-0020 trained workflow + AC-1 narrow contract |
| Single merged table with hint badges | Conflates row-level browse with group-level confirm; breaks AC-4 multi-select arbitrary txs |
| Separate top-level nav tab | Discover is one subscription-ops surface; sub-modes sufficient |

## Risks

| Risk | Mitigation |
|------|------------|
| UI surface complexity | Transactions default; Patterns one click away |
| Filter state confusion across modes | Hide incompatible filters in Patterns mode |
| Duplicate API calls on tab switch | React-query cache keyed by mode + params |

## Implementation anchor

- `frontend/src/pages/SubscriptionsPage.tsx` — sub-mode tabs, filter bar, tables
- `frontend/src/lib/api.ts` — dual API clients
