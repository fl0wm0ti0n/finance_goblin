# DEC-0112 — Subscription transaction search API (US-0021)

**Status:** Accepted  
**Date:** 2026-06-13  
**Story:** US-0021  
**Research:** [R-0092 §6.2–6.4](docs/engineering/research.md#r-0092--us-0021-subscription-transaction-explorer-vs-recurrence-only-discover)  
**Extends:** **DEC-0098** (discover patterns tab unchanged); **DEC-0111** (Geldbereich `account_role` COALESCE path); US-0018 category catalog

## Context

US-0021 AC-1/AC-2 require paginated **individual expense transactions** with rich SQL filters — not pre-grouped `DiscoverCandidate` rows. Extending `GET /discover?mode=transactions` would break the frozen DEC-0098 response shape.

## Decision

### 1. Route (frozen)

**New** dedicated route — do **not** extend `/discover`:

```
GET /api/v1/subscriptions/transactions/search
```

### 2. Query parameters (frozen)

| Param | Required | Contract |
|-------|----------|----------|
| `account_id` | **Yes** | Same guard as DEC-0098 — 400 when missing |
| `payee` | No | `description ILIKE '%' \|\| $n \|\| '%'` |
| `category_id` | No | Mirror category id (US-0018 catalog) |
| `account_role` | No | Geldbereich enum; JOIN `accounts` + DEC-0111 COALESCE path |
| `date_from` / `date_to` | No | ISO date; clamped to `detection_window_days` (730 default) |
| `amount_min` / `amount_max` | No | **P2 stretch** — negative expense amounts |
| `recurring_hint` | No | When `true`, run hint pass (DEC-0114); default **true** in UI |
| `page` | No | Default 1; offset pagination |
| `limit` | No | Default **100**, max **100** hard cap |

### 3. Response (frozen)

```json
{
  "transactions": [{
    "firefly_id": "...",
    "account_id": "...",
    "account_role": "defaultAsset",
    "date": "2026-01-15",
    "amount": -9.99,
    "description": "...",
    "category_id": "...",
    "category_name": "...",
    "recurring_hint": null
  }],
  "meta": {
    "page": 1,
    "limit": 100,
    "total_count": 247,
    "has_more": true,
    "truncated": false,
    "window_days": 730
  }
}
```

`recurring_hint` populated per **DEC-0114** when hint pass runs; otherwise `null`.

### 4. SQL push-down (frozen)

All filters execute in SQL — not in-memory post-load:

| Filter | SQL |
|--------|-----|
| account_id | `WHERE t.account_id = $n` |
| date range | `t.date >= $from AND t.date <= $to` within window |
| category_id | `t.category_id = $n` |
| payee | `t.description ILIKE ...` |
| amount band | `t.amount BETWEEN $min AND $max` (P2) |
| Geldbereich | `JOIN accounts a ON t.account_id = a.firefly_id` + DEC-0111 COALESCE |

Reuse `load_expense_transactions` patterns; extend repository with parameterized search query + `COUNT(*)` for `total_count`.

### 5. Pagination (frozen)

- Offset pagination MVP; keyset cursor **deferred** (R-0092 §6.4)
- `has_more = page * limit < total_count`
- UI documents cap: "Showing up to 100 transactions per page — refine filters or narrow account."

### 6. Preview-group helper (frozen)

`POST /api/v1/subscriptions/transactions/preview-group`

**Body:** `{ "transaction_ids": ["…"] }` (≥2 ids)

**Response:** `{ "payee_key", "interval_days", "median_amount", "transaction_ids" }` — computed server-side via existing Rust helpers; feeds existing `POST /discover/confirm` body (**DEC-0099** unchanged).

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| `GET /discover?mode=transactions` | Breaks DEC-0098 `candidates[]` contract; couples unrelated response shapes |
| In-memory filter after full load | Latency/memory on 730d windows; category/date/role need SQL |
| Paginate beyond 100/page | Operator browse not export; 100 sufficient per R-0092 household probe |
| Client-side preview-group math | Diverges from `median_amount` / interval Rust helpers; DEC-0084/0086 drift |

## Risks

| Risk | Mitigation |
|------|------------|
| JSON `accounts.payload` JOIN perf | Household account count low; account_id required |
| Deep offset cost | Account-scoped windows typically &lt;500 txs; keyset P2 |
| `COUNT(*)` on wide filters | Same indexed predicates as search; acceptable at household scale |
| Amount band scope creep | P2 stretch; not blocking AC-2 MVP |

## Implementation anchor

- `backend/src/subscriptions/repository.rs` — search SQL + count
- `backend/src/subscriptions/discovery.rs` or new `transaction_search.rs` — orchestration
- `backend/src/api/subscriptions.rs` — routes
- `frontend/src/lib/api.ts` — types
