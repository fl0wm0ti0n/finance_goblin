# DEC-0080 — Bitunix futures wallet parse and linear unrealized EUR (BUG-0013 AN/AK)

**Status:** Accepted  
**Date:** 2026-06-08  
**Work item:** BUG-0013 (sub-defects **AN**, **AK**)  
**Research:** [R-0076 §6](../docs/engineering/research.md#r-0076--omniflow-analytics-regression-hypotheses-post-us-0015), [R-0034](../docs/engineering/research.md#r-0034--fx-conversion-for-crypto--eur-reporting-currency)  
**Extends:** **DEC-0064** (wallet in subtotal; linear notional excluded), **DEC-0038** (PnL boundary), **DEC-0039** (FX)

## Context

Omniflow Bitunix sync succeeds (**7** `product_type=linear` rows) but **`market_value_eur` NULL** on all holdings → portfolio **Crypto value €0**, `holdings_top` empty, FX incomplete warning. Root causes (code audit):

1. **`parse_futures_wallet`** expects `data.account` object; Bitunix API returns `data: [{ marginCoin, available, margin, … }]` array — **no futures wallet row** created.
2. **`recompute_pnl`** calls `fx.to_eur(qty, asset)` with full symbol (`INJUSDT`) for linear rows → `FxError::Unpriced`; loop `continue`s before unrealized PnL is converted.
3. Position field **`unrealizedPNL`** (Bitunix casing) missing from `parse_f64_field` key list (`unrealizedPnl` only).
4. **`ExchangePriceBook::default()`** empty during recompute — spot tickers deferred (tier 2); not blocking AN1 MVP.

## Decision

**AN1 MVP = Option A + B** from R-0076 §6. **No DEC-0064 amendment** for this sprint.

### 1. Wallet ingest (frozen)

| Step | Contract |
|------|----------|
| Array shape | When `data` is JSON array, select first element with `marginCoin` / `available` (or sole element) as account object |
| Object shape | Preserve existing `data.account` path for test mocks |
| Equity | `accountEquity` → `available + margin + frozen` fallback (existing) |
| Row | `product_type: "futures"`, `asset: marginCoin` (default USDT), `market_value_usd: Some(qty)` for USDT/USDC only |
| Field keys | Add `unrealizedPNL` to wallet and position unrealized key lists |

### 2. Valuation in `recompute_pnl` / `compute_hybrid_pnl` (frozen)

| `product_type` | `market_value_eur` (wealth subtotal) | `unrealized_pnl_eur` |
|----------------|--------------------------------------|----------------------|
| `futures` | `fx.to_eur(quantity, asset)` — USDT/USDC via fiat stable path | Optional; wallet equity already embeds cross-margin unrealized per exchange |
| `linear` | **Excluded** — `holding_value_eur` returns skip; **do not** add symbol to `unpriced_assets` or set `fx_incomplete` when exchange unrealized present (DEC-0064) | Parse from `payload` (`unrealizedPNL` / `unrealizedPnl` / …); treat as **USDT**; `fx.to_eur(pnl, "USDT")`; persist via `update_holding_eur` |
| `spot` | Existing `to_eur(qty, asset, price_book)` — tier 2 may populate price book later | Existing avg-cost fallback |

**`crypto_value_eur`:** Sum of priced `market_value_eur` only (futures wallet rows after recompute). Linear rows do not inflate crypto stat.

**`holdings_top`:** Requires non-null `market_value_eur` — wallet row unlocks top holdings when equity &gt; 0.

### 3. Explicitly deferred (tier 2)

- `ExchangePriceBook` population from exchange tickers during recompute (spot + display)
- Symbol→base strip + mark price for linear notional (**option C** — wrong units without contract size)
- `entryValue` exposure display column (**option D** — needs DEC-0064 amend if merged into subtotal)
- Multi-coin margin wallets beyond USDT/USDC

### 4. AM / performance %

- **AM:** Waived per [R-0077](../docs/engineering/research.md#r-0077--bug-0013-grafana-embed-failed-to-fetch-annotation-runner) — no execute unless browser HAR shows non-200
- **AK performance %:** NULL with &lt;2 priced snapshots — **AK2** documents empty-state; not AN1 blocker

## Alternatives rejected

| Alternative | Why |
|-------------|-----|
| **E — Full notional in `market_value_eur`** | Violates DEC-0064 double-count with wallet |
| **C — INJUSDT→INJ ticker price** | Linear qty is contracts; wrong economics without contract size |
| **D — exposure_eur in subtotal** | Requires DEC-0064 amend; wallet+unrealized sufficient for acceptance AK/AN |

## Impact

- **AN1** primary files: `backend/src/exchanges/bitunix.rs`, `backend/src/portfolio/pnl.rs`; tests in `bitunix.rs`
- **`upsert_holdings`** unchanged — valuation phase only (`recompute_pnl` after sync)
- Operator: **Full sync** or exchanges sync + wealth recompute path after deploy

## Risks

- Wallet-only equity **€0** with open positions but no margin in account API — rare; unrealized PnL still surfaces in portfolio unrealized stat
- USDT≈USD FX drift — acceptable per DEC-0039 stablecoin path
- `fx_incomplete` false when only linear positions exist and wallet missing — mitigated by wallet parse fix
