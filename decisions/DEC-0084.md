# DEC-0084 — Card billing descriptor normalization (BUG-0015 Layer 1)

**Status:** Accepted  
**Date:** 2026-06-07  
**Work item:** BUG-0015 (sub-defect **H1** / acceptance **AU**, **AV**)  
**Research:** [R-0082](../docs/engineering/research.md#r-0082--card-billing-descriptor-normalization-for-subscription-identity)  
**Extends:** **DEC-0072** (SEPA / legal suffix normalization), **DEC-0013** (shared `recurrence` module)

## Context

Card and PSP billing descriptors vary month-to-month: comma-separated memos (`CURSOR, AI POWERED IDE, CURSOR.COM`), asterisk suffixes (`DBA*Plan`), and aggregator roots (`APPLE.COM/BILL` vs `ITUNES.COM`). DEC-0072 reduced SEPA drift but did not collapse these patterns. Fingerprint drift (H1) creates new `pending` rows while prior `confirmed` rows remain on old fingerprints.

## Decision

Extend `payee_key()` in `backend/src/recurrence/normalize.rs` with **card-billing rules** applied after existing DEC-0072 passes:

| Rule | Algorithm |
|------|-----------|
| **Asterisk** | If `*` present, take substring before first `*` |
| **Comma memo** | If `,` present, take leftmost segment (trim, lowercase) |
| **Billing root alias** | Map known roots to canonical token: `apple.com/bill`, `itunes.com`, `apple.com/bill itunes` → `apple` |
| **Domain tail** | After above, strip trailing `.com`, `/bill` on segment when remainder is alphanumeric merchant token |

Rules run **in order**; output feeds both recurrence grouping and subscription detection.

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| Normalization-only (no Layer 2) | Median amount / interval drift still breaks fingerprint — insufficient for AU |
| Global fingerprint without amount (option B) | Tier-change collisions; out of scope |
| External merchant enrichment API | Over-engineered for MVP (option F) |
| Substring-only without alias list | Insufficient for Apple aggregator class per R-0082 |

## Risks

- **Over-merge** distinct products under same biller — mitigated by DEC-0085 payee+**interval** composite (not payee-only skip).
- **Forecast side effect** — shared `recurrence` module may regroup forecast recurring lines; conservative alias list + unit tests required.
- **EU descriptor charset** — rules are ASCII-first; Layer 2 inheritance catches residual variance.

## Implementation anchor

- `backend/src/recurrence/normalize.rs` — `payee_key()` + unit tests (AU1)
- No migration — pure function change
