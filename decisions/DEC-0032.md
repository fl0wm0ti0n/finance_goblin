# Decision DEC-0032

**Status:** Accepted  
**Date:** 2026-05-31  
**Story:** US-0006  
**Research:** R-0028  
**Extends:** DEC-0004 (read-only data boundary)

## Context

US-0006 Privacy Layer must enforce Projectplan TOML defaults before data reaches OpenAI or audit storage. Discovery asked central middleware vs per-tool redaction and semantics when `allow_raw_transactions=false`.

## Options

1. **Central `PrivacyLayer` on all tool output** — per R-0028
2. **Per-tool redaction helpers only** — rejected (bypass risk)
3. **Block all output when raw disabled** — rejected (model needs aggregates)
4. **Token vault with rehydration** — deferred to US-0008
5. **Runtime UI toggles** — deferred (read-only Settings display for MVP)

## Recommendation

**Defaults (Projectplan):**

```toml
[privacy]
allow_raw_transactions = false
redact_iban = true
redact_counterparties = true
```

**Placement:** orchestrator calls `PrivacyLayer::redact_tool_result` after every `AiTool::execute`, before OpenAI tool message and audit `args_summary`.

**`allow_raw_transactions=false`:** `get_transactions` returns category/month aggregates with `raw_transactions_included: false`. Structured error when raw rows requested — not empty payload.

**`redact_iban`:** ISO IBAN regex + MOD-97 → `[IBAN_REDACTED]`.

**`redact_counterparties`:** deterministic `Counterparty-{hash8}` pseudonym (SHA-256 + app pepper).

**Settings:** read-only display of TOML values in AI & Privacy section.

## Impact

- Single enforcement point for all current and future tools
- Audit rows never contain unredacted sensitive fields

## Risks

- Regex false positives in descriptions — document in user guide
- User-typed sensitive data in chat not pre-redacted in MVP — system prompt warns model
