# DEC-0069 — AI merchant/category discovery tool contracts (BUG-0007)

**Status:** Accepted  
**Date:** 2026-06-07  
**Bug:** BUG-0007  
**Research:** [R-0065](../docs/engineering/research.md#r-0065--bug-0007-ai-merchant-category-discovery-tool-contracts-vs-rag)  
**Extends:** DEC-0032 (aggregate-only privacy), DEC-0035 (six-tool registry), DEC-0034 (audit `result_rows` column), BUG-0006 aggregate contract (R1 / `period_status`)

## Context

Post-BUG-0006 mirror ingest fixes, AI Chat still fails merchant/category discovery (sub-defects **S/T/U**). Root cause is **tool-contract + orchestration gaps**, not missing sync data:

- LLM passes keywords (`"Strom"`, `"amazon"`) as `category_id` instead of Firefly IDs (146, 47).
- `get_subscriptions` returns `display_name` values but LLM does not enumerate them; follow-up calls misuse privacy hashes (`Counterparty-*`) as enum filters.
- Orchestrator system prompt biases aggregates without category-resolution or subscription-enumeration guidance; audit `result_rows` is always `NULL`.

Acceptance footer requires **six-tool registry preserved** and **`allow_raw_transactions=false` default unchanged**. A seventh `get_categories` tool is **rejected** ([R-0065](research.md#r-0065--bug-0007-ai-merchant-category-discovery-tool-contracts-vs-rag) §4).

## Decision

Freeze **A′ + E + F** fix bundle — extend existing tools and orchestrator within the six-tool registry. **Defer:** payee aggregates (B), RAG (D/V), raw opt-in (C except documentation).

### A′ — `get_transactions.category_search` (frozen)

#### Parameter contract

| Field | Type | Required | Semantics |
|-------|------|----------|-----------|
| `category_search` | `string` | optional | Case-insensitive keyword; server resolves against synced `categories.name` via `ILIKE '%{keyword}%'` after trim. Min length **2** after trim; shorter → `InvalidArgs`. |
| `category_id` | `string` | optional | Firefly category ID (exact match) — unchanged from BUG-0006. |
| `period_start` / `period_end` | `YYYY-MM-DD` | required | unchanged |
| `group_by` | `category` \| `month` | optional | unchanged; default `category` |

**Precedence:** When both `category_search` and `category_id` are non-empty, **`category_search` wins**; `category_id` is ignored. Response includes `category_id_ignored: true` when both were supplied.

**Rejected alternatives:**

| Alternative | Why rejected |
|-------------|--------------|
| Seventh `get_categories` tool | Violates six-tool acceptance footer |
| Description/payee text search | Blocked under DEC-0032 defaults; Amazon signal is category name on omniflow |
| Fuzzy/synonym expansion (Strom→Stromkosten) | ILIKE substring sufficient for acceptance probes; US-0015 owns AI mapping |

#### Server resolution

1. Query `categories` table: `WHERE name ILIKE '%' || $keyword || '%' ORDER BY name ASC LIMIT 10`.
2. Collect matching `firefly_id` values (string).
3. Aggregate mirror `transactions` in period where `category_id IN (...)` — **union** across all matches.
4. If zero category matches: return aggregates with `category_matches: []`, `search_attempted: true`, existing `period_status` (typically `no_rows_in_period` or zero outflow in period).

**Match cap:** **10** categories maximum. When more exist, truncate by name ASC; set `category_matches_truncated: true`.

#### Response extensions (always on `get_transactions` JSON)

| Field | Type | Semantics |
|-------|------|-----------|
| `mirror_date_bounds` | `{ min: string, max: string }` | ISO dates from `MIN(date)`, `MAX(date)` on mirror `transactions` (global, not period-filtered). Always present. |
| `category_search` | `string` \| null | Echo of keyword when used |
| `category_matches` | `[{ category_id, category_name }]` | Resolved matches (≤10); empty when no keyword or no hits |
| `category_matches_truncated` | `boolean` | true when >10 name matches |
| `search_attempted` | `boolean` | true when `category_search` was provided |
| `category_id_ignored` | `boolean` | true when both id + search supplied |

Existing BUG-0006 fields (`total_transaction_count`, `period_status`, `by_category`, etc.) **unchanged**. `raw_rows` still gated by DEC-0032.

**T-a empty period:** When requested period is outside `mirror_date_bounds`, LLM must cite bounds + `period_status: no_rows_in_period` — not blanket "no expenses".

### E — Orchestrator prompt + audit (frozen)

#### System prompt additions (append to existing `SYSTEM_PROMPT`)

1. **Subscription enumeration:** When listing subscriptions or cancelable services, enumerate **every** `display_name` from `get_subscriptions` results with amounts — never generic industry-only lists when tool data exists.
2. **Category keywords:** For utility/merchant/category keywords (e.g. Strom, Amazon, streaming), use `get_transactions` with **`category_search`** — **never** pass keywords as `category_id`.
3. **Empty period:** When `period_status` is `no_rows_in_period`, state **`mirror_date_bounds`** from tool response so operator knows whether the period is outside synced data.
4. **Enum filters:** `get_subscriptions` `status`/`kind` accept only documented enum values — never privacy hashes (`Counterparty-*`).

**Rejected:** `tool_choice: required` for subscription prompts — breaks local providers (DEC-0045 / R-0041).

#### Audit `result_rows` population

After successful tool execution, persist row/bucket count in `ai_tool_audit.result_rows`:

| Tool | `result_rows` value |
|------|---------------------|
| `get_transactions` | `by_category.len()` if present, else `total_transaction_count` as `i32` |
| `get_subscriptions` | `patterns_count` from response (see F) |
| Other tools | unchanged (`NULL`) |

On tool error: `result_rows` remains `NULL`.

#### Parameter schema descriptions (OpenAI tool JSON)

- `category_id`: "Firefly category ID (numeric string). For keyword search use **category_search** instead."
- `category_search`: "Keyword to match category **names** (case-insensitive). Example: 'amazon', 'strom'."

### F — `get_subscriptions` schema enrichment (frozen)

#### Parameter schema

| Field | Change |
|-------|--------|
| `status` | unchanged enum: `pending`, `confirmed`, `rejected` |
| `kind` | **Add enum:** `subscription`, `standing_order` |
| `include_price_events` | unchanged |

**Server-side enum guard:** Before calling `SubscriptionService::list_patterns`, reject args where `status` or `kind` starts with `Counterparty-` → `ToolError::InvalidArgs("privacy hash is not a valid filter")`.

#### Tool description

Replace with: "List detected subscription and standing-order patterns. When the user asks to list services, enumerate **all** `display_name` values from results with amounts. Use `kind: subscription` for discretionary subs; omit kind for all patterns."

#### Response extensions (additive JSON only)

| Field | Type | Semantics |
|-------|------|-----------|
| `patterns_count` | `i64` | `patterns.len()` |
| `merchant_names` | `string[]` | Deduped `display_name` values in stable order (patterns array order, first occurrence wins) |

Existing `patterns[]` shape **unchanged** (`display_name`, `status`, `kind`, `current_amount`, etc.). **Do not** expose `payee_key` to AI tool (privacy-adjacent; not required for S).

## Risks

| Risk | Mitigation |
|------|------------|
| Broad `category_search` matches many categories | Cap 10; union aggregate; truncated flag |
| German synonym gaps (Strom vs Stromkosten) | ILIKE substring; omniflow category names include both tokens |
| Local model ignores prompt rules | Schema descriptions + existing LOCAL_TOOL_NUDGE (R-0041) |
| BUG-0008 shared service regression | Additive JSON in AI tool layer only; no REST/filter changes |
| `mirror_date_bounds` query on every call | Single `MIN/MAX` index scan — acceptable for chat latency |
| LLM still passes Counterparty-* as filters | Server-side enum guard returns clear InvalidArgs |
