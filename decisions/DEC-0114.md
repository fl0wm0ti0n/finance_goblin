# DEC-0114 — Transaction-search hint pass boundary (US-0021)

**Status:** Accepted  
**Date:** 2026-06-13  
**Story:** US-0021  
**Research:** [R-0092 §6.3](docs/engineering/research.md#r-0092--us-0021-subscription-transaction-explorer-vs-recurrence-only-discover)  
**Extends:** **DEC-0014** (confidence tiers unchanged globally); **DEC-0098** (discover candidate pass unchanged); US-0003 auto-detection unchanged

## Context

US-0021 AC-3 requires recurring **hints** on filtered transaction rows — including patterns operators could not find in candidate-only discover. Lowering global `min_emit_confidence` in `detection.rs` would violate AC-5 regression guard. Code audit (`recurrence/detect.rs`): scores below qualifying groups return **0**, not 30–59 — so merely lowering threshold does not surface new groups without scorer changes.

## Decision

### 1. Three-pass boundary (frozen)

| Pass | Location | `min_emit_confidence` | Side effects |
|------|----------|----------------------|--------------|
| Auto-detection | `detection.rs` / sync pipeline | **60** (unchanged) | Pending patterns, alerts — AC-5 guard |
| Discover candidates | `discovery.rs` / Patterns tab | **60** (unchanged) | DEC-0098 `candidates[]` only |
| **Tx-search hints** | search service on SQL-filtered subset | **60** | Row `recurring_hint` metadata only |

### 2. Hint pass behavior (frozen)

1. Run **after** SQL-filtered tx load for current search (same filter set as DEC-0112)
2. Invoke `detect_recurrence_groups` on loaded txs with `min_emit_confidence: 60`
3. For each qualifying group, attach `recurring_hint` object to **matching transaction rows** in the response:

```json
"recurring_hint": {
  "interval_days": 30,
  "confidence_pct": 95,
  "payee_key": "...",
  "group_transaction_ids": ["..."]
}
```

4. **No** auto-emit alerts, **no** pending pattern creation, **no** writes to `subscription_patterns`
5. Triggered when query `recurring_hint=true` (UI default on) or architecture-default always-on in search service

### 3. Hint scan budget (frozen)

- Hint input capped at **500 transactions** max per search (post-SQL filter, pre-pagination hint map)
- Pagination slice applied after hint attachment for current page rows
- If filter result exceeds 500 txs, hint pass runs on first 500 by date DESC; `meta.truncated_hint_scan: true` optional flag

### 4. Sub-threshold weak hints (P2 defer — GATE-HINT-2)

Relaxing `txs.len() < 3` to `>= 2` for hint-only display with synthetic low confidence is **out of MVP scope**. Current scorer returns 0 for non-groups — AC-3 primary path is surfacing txs that **do** score ≥60 on filtered subset but were invisible in candidate-only UX.

Revisit only if qa strict AC-3 reading fails on operator repro (account 114 SEPA-Lastschrift).

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| Lower global `min_emit_confidence` | AC-5 regression; changes auto-detection + alert volume |
| Client-side interval calc on selection | Diverges from R-0009 cadence; DEC-0084/0086 drift |
| Hint pass on full mirror without account filter | O(all txs); account required in DEC-0112 |
| Merge hints into `/discover` response | Violates DEC-0098 shape; DEC-0112 separation |

## Risks

| Risk | Mitigation |
|------|------------|
| Hint scan on 500 txs O(payees) | Acceptable post SQL cap; monitor if all-accounts opened later |
| Operator expects sub-60 hints | P2 GATE-HINT-2; document MVP boundary in user guide |
| Duplicate hint vs confirmed pattern | Reuse DEC-0098 exclude maps (confirmed/rejected fingerprint skip) |

## Implementation anchor

- `backend/src/subscriptions/discovery.rs` or `transaction_search.rs` — hint attachment
- `backend/src/recurrence/detect.rs` — **read-only reuse**, no threshold edits
- `frontend/src/pages/SubscriptionsPage.tsx` — hint badge rendering
