# DEC-0089 — Cross-surface category filter semantics and Grafana independence (US-0018)

**Status:** Accepted  
**Date:** 2026-06-08  
**Story:** US-0018  
**Research:** [R-0083 §3, §5–6](research.md#r-0083--us-0018-category-filters-expense-series-api--trend-analytics)  
**Extends:** DEC-0007 (forecast engine), DEC-0019 (plan overlay), US-0011 (Grafana embed)

## Context

AC-1 spans forecast, planning, wealth, and two Grafana dashboards. Research gates asked whether category filter should re-project forecasts, recompute plan compare, or sync Grafana with SPA state.

## Decision

### 1. Forecast monthly (frozen)

| Element | With category selected |
|---------|------------------------|
| Stat cards (Income/Fixed/Variable/Free) | **Unchanged** — household DEC-0007 forecast |
| `MonthlyChart` (bucket bars) | **Unchanged** — household |
| `CategoryTrendChart` | **Filtered actuals** from expense-series only |

Full category-scoped forecast re-projection → **US-0019 / follow-on**.

### 2. Planning compare (frozen)

- `CategoryFilter` in compare toolbar (page state)
- Adjacent `CategoryTrendChart` shows **actuals trend** for selected category
- Compare API and `build_compare_metrics` **unchanged** — no `category_id` query param
- Plan lines with `target_type=category` remain visible in compare table

### 3. Wealth overview (frozen)

- New "Category spending" subsection with `CategoryFilter` + period summary from expense-series
- Optional "View trend" deep link to `/forecast?category_id=` (stretch; not AC blocker)
- Net worth / crypto totals remain household-level

### 4. Grafana independence (frozen)

- Add `$category` template variable to **cashflow** and **budgets** dashboards
- Panel SQL: `AND ('${category}' = '' OR t.category_id = '${category}')`
- Default empty = All categories (pre-US-0018 behavior)
- **No** bidirectional sync with SPA iframe — independent operator filters per US-0011 embed

| Dashboard | Change |
|-----------|--------|
| cashflow | New monthly category outflow panel + `$category` variable |
| budgets | Category filter on Ist/deviation **actual** leg only; planned leg household |

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| Category-scoped forecast recompute | Unnecessary for AC-1..AC-5; DEC-0007 fork risk |
| Server-side compare recompute per category | Plan engine scope creep |
| iframe `category_id` URL sync | Auth/query coupling on US-0011 embed |
| Grafana-only (no React filter) | Fails AC-1 SPA surfaces |

## Risks

- Operator expects forecast buckets to filter → UI copy/tooltip required
- Grafana vs SPA category mismatch → document in user guide
- Budgets planned vs actual semantic when category set → planned leg stays household; document in panel subtitle
