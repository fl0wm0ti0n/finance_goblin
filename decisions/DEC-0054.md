# Decision DEC-0054

**Status:** Accepted  
**Date:** 2026-06-01  
**Story:** US-0009  
**Research:** R-0048, R-0022  
**Extends:** US-0004 Plan Engine, US-0005 plan_viability alerts

## Context

AC4 requires a risk assessment score (0–100) for active plan scenarios. Operators need explainability in self-hosted context. R-0022 already defines plan_viability alert rules — score must complement, not duplicate, inbox alerts.

## Options

1. **Deterministic weighted index with components JSON** — recommended (R-0048)
2. **ML classifier on scenario features** — rejected (black-box; no training data)
3. **Reuse alert severity only (binary)** — rejected (AC wants 0–100 score)
4. **ML forecast bands as primary risk input** — rejected (plan overlay answers scenario viability framing)

## Recommendation

**Formula:**

```
raw = 0.45*balance_stress + 0.40*plan_viability + 0.15*crypto_volatility + ml_divergence_modifier(±5)
risk_score = clamp(round(raw), 0, 100)
```

**Components:**

| Component | Weight | Mapping |
|-------------|--------|---------|
| Balance stress | 45% | Month-ends < €0 in next 6mo on plan overlay: 0→0, 1→50, ≥2→85; +15 if any < −€500 |
| Plan viability | 40% | R-0022 mirror: current month-end < 0 → 80; consecutive → 100; else 0 |
| Crypto volatility | 15% | CV of last 12 weekly crypto EUR changes; 0 when no exchanges |
| ML divergence | ±5 | ML 6mo p10 below scarcity threshold while baseline 6mo end above |

**UI bands:** 0–29 Low, 30–59 Medium, 60–100 High.

**Persistence:** `plan_risk_scores` table keyed by `plan_computation_id`.

**Compute trigger:** after plan refresh (DEC-0023) and after ML pass (divergence modifier).

**Not** a new Alert type — badge on Planning Scenarios + Compare column only.

## Impact

- `backend/src/plan/risk.rs` — `PlanRiskService`
- Migration 009 `plan_risk_scores`
- React Planning badges + tooltip breakdown
- Grafana Dashboard 5 optional risk stat panel

## Risks

- Mid-month plan edits skew viability — same limitation as R-0022 alerts
- Crypto CV noisy on small portfolios — 0 weight when exchanges disabled
- Operators over-trust precision — mandatory component breakdown tooltip
