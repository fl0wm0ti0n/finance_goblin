# Decision DEC-0009

**Status:** Accepted  
**Date:** 2026-05-31  
**Story:** US-0002  
**Research:** R-0006, R-0008

## Context

Discovery asked whether the account selector defaults to a single asset account or household aggregate. React UI and Grafana both need an account variable.

## Options

1. **Per-account primary** — selector defaults to first asset account by name
2. **Household aggregate default** — sum all asset accounts; harder to interpret for multi-account users
3. **Per-account only, no aggregate** — simpler but limits Grafana overview

## Recommendation

**Per-account primary; optional aggregate secondary.**

- React `/forecast` account selector lists synced Firefly `asset` accounts; default first by name
- API accepts `account_id` query param (required for per-account endpoints)
- Optional `GET /api/v1/forecast/aggregate` sums daily/monthly series across asset accounts for power users
- Grafana `$account_id` template variable from `accounts WHERE type = 'asset'`; add "All accounts" custom option only if aggregate query shipped in execute

## Impact

- Forecast Engine runs per asset account then optionally merges for aggregate endpoint
- Dashboard 1 and 5 use same account variable pattern as React

## Risks

- Internal transfers between own accounts must be excluded before aggregation (R-0006)
- Multi-currency households need currency filter — MVP: single-currency per account; aggregate same-currency only or defer cross-currency sum to US-0005
