# DEC-0093 ‚Äî Category-scoped plan overlay semantics (US-0019)

**Status:** Accepted  
**Date:** 2026-06-09  
**Story:** US-0019  
**Research:** [R-0084 ¬ß3](research.md#r-0084--us-0019-goal-plans-per-plan-stats-category-overlay--ai-savings)  
**Extends:** DEC-0019 (overlay deltas), DEC-0087 (`expense_series_by_month`), DEC-0007 (forecast baseline unchanged)

## Context

US-0019 AC-3 requires category-scoped spend changes to affect compare/PVA after recompute. `AdjustmentTarget::Category` exists in DB and form (`CategoryFilter` on PlanningPage) but `build_overlay_deltas` applies category adjustments identically to household ‚Äî no `category_id` join or cap (`overlay.rs`).

## Decision

### 1. `remove_outflow` + `category` (frozen)

Each overlay application capped at **trailing 3-month average monthly outflow** for `target_key` (category Firefly id):

```
effective_amount = min(adj.amount, avg_monthly_outflow_last_3mo)
```

- Data: `expense_series_by_month` (DEC-0087) for last 3 calendar months ‚Üí `avg(outflow_eur)`
- Empty mirror for category ‚Üí overlay contributes **0**; UI warning on plan line
- Spread on existing monthly/weekly frequency schedule (unchanged timing model)

### 2. `add_outflow` + `category`

Treat as **household-labeled outflow** ‚Äî category is metadata for compare/PVA display only; **no cap**. API accepts; does not fork forecast buckets.

Alternative **API reject** rejected ‚Äî blocks legitimate "increase category spend" what-ifs.

### 3. Other target types

| Target | Behavior |
|--------|----------|
| `subscription` | Unchanged ‚Äî payee match |
| `household` | Unchanged ‚Äî full amount |
| `category` + `remove_outflow` | Capped per ¬ß1 |

### 4. Forecast interaction

Overlay modifies **plan delta layer only** (R-0015). Category removal does **not** rewrite Income/Fixed/Variable DEC-0007 projection. US-0015 `bucket_source` unchanged.

### 5. Compare tab regression

DEC-0089 `CategoryTrendChart` on Compare remains **actuals only** ‚Äî no `category_id` on compare API. Category overlay affects recompute only.

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| Display-only category lines | Fails AC-3 |
| Daily mirror-weighted removal | Higher SQL cost; monthly cap sufficient MVP |
| Fork forecast per category | Scope explosion |
| Reject `add_outflow` + category | Blocks valid what-ifs |

## Risks

- Fixed-cost categories (rent) reducible in overlay ‚Äî savings suggestions filter separately (DEC-0094)
- Uncategorized spend invisible to category overlay ‚Äî document; use household adjustment instead
- Cap exceeds future actual spend ‚Äî bounded by historical avg; operator can lower amount manually
- Double-count if multiple category lines same id ‚Äî sum caps per application event; audit in tests
