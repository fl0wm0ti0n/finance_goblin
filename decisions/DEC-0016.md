# Decision DEC-0016

**Status:** Accepted  
**Date:** 2026-05-31  
**Story:** US-0003  
**Research:** R-0010

## Context

German household finance distinguishes Dauerauftrag (standing orders: rent, insurance) from discretionary subscriptions. Discovery asked: whitelist vs rule-based classification?

## Options

1. **Rule-based primary + optional config payee patterns** — per R-0010
2. **Payee whitelist only** — operator burden; misses unlisted payees
3. **Separate detection engine for standing orders** — rejected (same recurrence core)
4. **Firefly tags as sole signal** — rejected (inconsistent operator tagging)

## Recommendation

Post-detection classification on `subscription_patterns.kind`:

| Rule | Classify `standing_order` when |
|------|-------------------------------|
| Exact-amount | Amount CV < 0.02 AND all amounts identical to €0.01 |
| Near-exact | All amounts within ±1% AND interval monthly/quarterly |
| Category boost | ≥50% txs match config category name list |
| Large fixed | Median ≤ −€200 AND amount CV < 0.01 |
| Default | `subscription` |

Optional TOML `standing_order_payee_patterns` merges with rules. User may override `kind` on confirm via API body.

UI: Standing orders tab filters `kind = standing_order`; excluded from "subscription spend" stat but included in fixed outflows.

## Impact

- `classify` submodule in subscriptions engine
- Confirm API accepts optional `{ "kind" }` override
- Grafana monthly spend stat filters `kind = 'subscription'`

## Risks

- Insurance premium adjustments may misclassify
- Rent with Nebenkosten variance may flip between kinds — user override required
