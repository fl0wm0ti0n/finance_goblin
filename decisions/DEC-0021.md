# Decision DEC-0021

**Status:** Accepted  
**Date:** 2026-05-31  
**Story:** US-0004  
**Research:** R-0017, R-0006

## Context

US-0004 acceptance requires daily plan-vs-Ist with planned, actual, and deviation. Discovery asked which primary number: free cashflow, category spend, or account balance path.

## Options

1. **Household daily net cashflow** — sum non-transfer asset-account transaction amounts per day (R-0017)
2. **Account balance path** — rejected (balance mixes history; deltas are cashflow-oriented)
3. **Firefly budget limits as plan** — rejected (Flow plans are explicit user deltas; no Firefly mutation)
4. **Monthly-only comparison** — rejected (acceptance requires daily grain)

## Recommendation

- **Planned:** `plan_daily_cashflow.planned_net` (signed; positive = net inflow day)
- **Actual (Ist):** mirror `transactions` aggregated by calendar day across asset accounts; exclude internal transfers
- **Deviation:** `actual_net - planned_net` (positive = better than plan)
- Default API month = current calendar month; include zero-transaction days (actual = 0)
- API metadata: `actuals_stale`, `plan_stale`, `reporting_currency`
- Per-category drill-down in React only (not Grafana MVP)

MVP assumes **single reporting currency** for household aggregate (TOML `plans.reporting_currency`).

## Impact

- `GET /api/v1/plans/active/plan-vs-actual` implements aggregation
- Grafana Dashboard 3 uses same Ist SQL and planned series join (DEC-0024)

## Risks

- Multi-currency households need future conversion (US-0005+)
- Savings mode: plan assumes cancelled sub but Ist still shows charges until real cancellation
- Sparse sync days show flat zero actual
