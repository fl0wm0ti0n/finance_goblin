# Decision DEC-0046

**Status:** Accepted  
**Date:** 2026-06-02  
**Story:** US-0008  
**Research:** R-0041  
**Extends:** DEC-0031 orchestration loop, AC5 privacy boundary

## Context

Local models may return assistant text without `tool_calls`, or emit malformed JSON arguments. Operators expect graceful chat rather than hard failure; AC5 forbids silent fallback to OpenAI when local is configured.

## Options

1. **Graceful text + SSE warning + optional single nudge retry (local only)** — recommended
2. **Hard-fail chat when no tool_calls** — rejected (blocks legitimate conversational replies)
3. **Auto-fallback to OpenAI when local fails** — rejected (violates AC5)
4. **Reduce tool set for local providers** — rejected (AC4 requires same six tools)

## Recommendation

**When local model omits `tool_calls` and returns text (`finish_reason=stop`):**
1. Return assistant text to user (graceful degradation)
2. Emit SSE `event: warning` with `{ "code": "no_tool_calls", "hint": "model answered without tools" }`
3. UI shows subtle warning when tools row empty on finance factual queries

**Optional single soft retry (local only):**
- Gate: `[ai] local_tool_nudge_retry = true` (default **true** when `is_local`)
- Append system nudge *"Use a tool to fetch data before answering."* and re-prompt **once**
- Cap prevents infinite loops

**Malformed tool arguments:** unchanged US-0006 path — `serde_json::from_str` failure → `role: tool` error message → continue loop. Unknown tool name → registry allowlist rejects before execute.

**System prompt addition for local providers:**
*"You have access to finance tools. When the user asks about balances, spending, subscriptions, or forecasts, you MUST call the appropriate tool before answering."*

**No OpenAI fallback** when `is_local=true` — ever.

## Impact

- `backend/src/ai/orchestrator.rs` — local branch after tool round parse
- SSE event type `warning` added to chat contract (frontend handles optionally)
- ChatPanel may show operator hint badge alongside provider badge

## Risks

- Local models may hallucinate numbers without tools — mitigated by system prompt + tool transparency row + provider badge
- Soft retry doubles latency on weak models — acceptable; disable via TOML
