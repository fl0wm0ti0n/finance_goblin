# DEC-0097 — AI category savings tool optional; REST primary (US-0019)

**Status:** Accepted  
**Date:** 2026-06-09  
**Story:** US-0019  
**Research:** [R-0084 §4 AI path](research.md#r-0084--us-0019-goal-plans-per-plan-stats-category-overlay--ai-savings)  
**Extends:** DEC-0035 (six-tool registry), DEC-0032 (privacy), **DEC-0094** (deterministic ranking)

## Context

US-0019 AC-4/AC-5 cover savings suggestions and privacy-safe AI path. Research recommends deterministic REST ranking as primary; optional chat wrapper for same aggregates.

## Decision

### 1. REST is canonical (AC-4)

`GET /api/v1/plans/{plan_id}/category-savings-suggestions` (**DEC-0094**) powers Scenarios savings modal. Execute **must** ship REST before or with modal.

### 2. Optional seventh tool (stretch / P2)

If capacity in sprint, register `get_category_savings`:

| Property | Value |
|----------|-------|
| Input | `months` (default 6), `limit` (default 10) |
| Output | Same aggregate shape as REST — no raw rows |
| Privacy | `allow_raw_transactions=false` safe — category totals only |
| Audit | US-0006 / DEC-0034 patterns |

Tool wraps same service function as REST — **no LLM ranking**.

`get_transactions` with `group_by: category` remains available for chat but is **not** the savings modal data source.

### 3. Registry constraint

If tool ships: extends six-tool registry to seven — update DEC-0035 documentation and AI settings copy. If over sprint threshold, **defer tool** — REST + modal satisfy AC-4/AC-5 without chat path.

### 4. No auto-apply from chat

Chat may **suggest** categories; applying adjustments always requires explicit plan mutation (same as REST modal).

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| Chat-only savings | Scenarios modal needs REST |
| LLM re-ranking | Non-deterministic; privacy review |
| Silent tool-side adjustment create | Violates AC-4 |

## Risks

- Seven-tool registry drift vs docs — update user guide + audit table if shipped
- Chat duplicates REST lists — acceptable; same deterministic backend
- Sprint scope creep — gate tool as P2 task in sprint-plan

## Implementation anchor

- `backend/src/ai/tools/` (optional `category_savings.rs`)
- `backend/src/plan/savings_service.rs` (shared with REST)
- `docs/user-guides/US-0019.md` (execute)
