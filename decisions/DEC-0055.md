# Decision DEC-0055

**Status:** Accepted  
**Date:** 2026-06-01  
**Story:** US-0009  
**Research:** R-0051, R-0008  
**Extends:** DEC-0012 (dashboard uid stability), US-0002 Dashboard 5

## Context

AC5 requires Grafana Dashboard 5 extended with ML forecast and risk panels. Existing `forecast-horizons.json` serves baseline-only queries. Operators bookmark stable dashboard uids per DEC-0012.

## Options

1. **Extend existing JSON with `$forecast_variant` + new panel row** — recommended (R-0051)
2. **New Dashboard 6 for ML** — rejected (AC5 specifies Dashboard 5 extensions only)
3. **Prometheus export of ML series** — rejected (duplicate storage per R-0008)
4. **Grafana ML plugin** — rejected (PostgreSQL datasource sufficient)

## Recommendation

- Keep **`uid: forecast-horizons`** unchanged; bump dashboard `version`
- Add template variable **`$forecast_variant`**: `baseline` | `ml_enhanced` (default baseline)
- Replace computation_id subqueries with `model_kind='$forecast_variant'` filter
- **New panels:** confidence band timeseries, seasonal detected stat, baseline vs ML end balance stat row, portfolio 3/6/12 mo stat row, active plan risk score stat
- Compare overlay panel: dual targets (baseline blue + ML orange); band fill on ML target only
- When ML skipped and variant=ml_enhanced: panels show "No data" — dashboard description note

## Impact

- `grafana/provisioning/dashboards/analytics/forecast-horizons.json`
- No new datasource or uid changes

## Risks

- Empty ML panels confuse operators — description text + default variant baseline
- Metadata JSON path typos break seasonal stat — validate in provisioning test
- Panel count growth — new row only; existing baseline panels unchanged
