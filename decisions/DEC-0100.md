# DEC-0100 — Subscription display majority category (US-0020)

**Status:** Accepted  
**Date:** 2026-06-10  
**Story:** US-0020  
**Research:** [R-0085 §3](research.md#r-0085--us-0020-subscription-discover-majority-category--operator-tags)  
**Extends:** **DEC-0087** (category catalog display names), US-0018 `__uncategorized__` labeling

## Context

US-0020 AC-3/AC-5 require confirmed subscription **display category** = mode of linked transaction categories, stored in product DB with no Firefly write-back. `subscription_patterns` has no `display_category_id`; `RecurrenceGroup.category_ids` collected in detection but unused for display.

## Decision

### 1. Schema (frozen)

Migration adds to `subscription_patterns`:

| Column | Type | Notes |
|--------|------|-------|
| `display_category_id` | `TEXT NULL` | Firefly category id; NULL when all linked txs uncategorized |

No join table; single computed overlay field per pattern.

### 2. Algorithm (frozen — deterministic)

Do **not** use bare PostgreSQL `mode()` for tie-break.

```sql
WITH linked AS (
  SELECT t.category_id, t.date
  FROM subscription_pattern_transactions spt
  JOIN transactions t ON t.firefly_id = spt.transaction_firefly_id
  WHERE spt.pattern_id = $1 AND t.category_id IS NOT NULL
),
ranked AS (
  SELECT category_id,
         COUNT(*) AS cnt,
         MAX(date) AS last_date,
         RANK() OVER (ORDER BY COUNT(*) DESC, MAX(date) DESC) AS rnk
  FROM linked
  GROUP BY category_id
)
SELECT category_id FROM ranked WHERE rnk = 1 LIMIT 1;
```

Rust equivalent on confirm payload acceptable — same policy.

**Miscategorization (1-of-12):** pure mode sufficient for MVP; no singleton exclusion rule.

### 3. Refresh policy (frozen)

| Event | Recompute `display_category_id`? |
|-------|----------------------------------|
| Initial confirm (discover or pending→confirmed) | **Yes** |
| `merge_confirmed_pattern` links new txs | **Yes** |
| Routine sync tick without new links | **No** |

### 4. Display

- Resolve name via `GET /api/v1/categories` (DEC-0087)
- NULL → "Uncategorized" badge (`__uncategorized__` labeling)
- Tooltip: *"Display category is the most common category among linked transactions; ties broken by most recent charge."*

Operator override column — **defer** (stretch).

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| `mode() WITHIN GROUP` | Arbitrary tie-break in PostgreSQL |
| Recompute every sync | Unstable display; unnecessary churn |
| Write category to Firefly txs | Violates read-only contract (AC-5) |

## Risks

- All txs uncategorized — NULL badge; expected
- Category deleted in Firefly — catalog JOIN fallback to raw id
