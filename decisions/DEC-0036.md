# Decision DEC-0036

**Status:** Accepted  
**Date:** 2026-05-31  
**Story:** US-0006  
**Research:** R-0029, discovery UX refs  
**Extends:** US-0001 UI shell, Settings page

## Context

US-0006 delivers conversational Q&A via header drawer and full-page chat. Discovery captured Finanzguru-style suggested prompts, tool transparency, and Settings AI & Privacy section.

## Options

1. **Shared `ChatPanel` in header Sheet drawer + `/chat` route + Settings audit** — recommended
2. **`/chat` only without header drawer** — rejected (discovery UX)
3. **Separate drawer vs page implementations** — rejected (duplicate logic)
4. **Inline ECharts in chat** — deferred if sprint tight
5. **Runtime privacy toggles in Settings** — deferred (read-only TOML MVP)

## Recommendation

**Navigation:** enable sidebar AI nav at `/chat` (replace disabled placeholder).

**Header:** AI button opens shadcn `Sheet` drawer (~400px) with shared `ChatPanel`.

**ChatPanel features:**
- Streaming message render via SSE hook
- Empty-state suggested prompt chips (5 Projectplan examples)
- Collapsible "Tools used" row under assistant messages (name + timestamp; no raw JSON)
- Privacy badge when aggregates-only mode active
- Abort stream on unmount/navigation

**Settings:** new **AI & Privacy** section — read-only `[ai]` + `[privacy]` TOML display; audit log table from `GET /api/v1/ai/audit`.

**Styling:** extend existing shadcn/Tailwind patterns; add `@radix-ui/react-dialog` Sheet primitive if not present.

## Impact

- `AppLayout` header gains AI button alongside `AlertBell`
- New route in React router

## Risks

- Sheet + full page state not shared across navigation — acceptable (ephemeral threads)
- Mobile drawer width — use responsive Sheet side
