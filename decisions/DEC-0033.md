# Decision DEC-0033

**Status:** Accepted  
**Date:** 2026-05-31  
**Story:** US-0006  
**Research:** R-0029  
**Extends:** DEC-0006 (SPA JWT auth — BFF gate closed for SSE)

## Context

US-0006 chat UI requires responsive streaming. Discovery asked SSE vs buffered responses, chat history persistence, rate limiting, and whether DEC-0006 requires cookie-based auth for SSE.

## Options

1. **SSE via POST `/api/v1/chat/stream` + Bearer JWT + fetch ReadableStream** — per R-0029
2. **Buffered JSON only** — rejected (UX latency)
3. **WebSockets** — rejected (SSE sufficient; simpler proxies)
4. **BFF cookie session for SSE** — rejected (DEC-0006 gate closed)
5. **DB-persisted chat threads** — deferred (ephemeral client MVP)

## Recommendation

**Primary endpoint:** `POST /api/v1/chat/stream` with same `require_auth` JWT middleware as other `/api/v1/*` routes.

**SSE events:** `token`, `tool_start`, `tool_end`, `done`, `error`.

**Client:** React `fetch()` + `ReadableStream` (not `EventSource` — no POST/Authorization support).

**History:** ephemeral — React state + optional `sessionStorage`; no migration for threads.

**Rate limit:** in-memory token bucket per JWT `sub` — default 20 requests / 10 min; configurable `[ai] rate_limit_per_min`.

**Disconnect:** `CancellationToken` aborts upstream OpenAI stream.

**Headers:** `X-Accel-Buffering: no`, 15s SSE keep-alive.

**Fallback:** optional non-streaming `POST /api/v1/chat/completions` for tests.

## Impact

- No BFF architecture change
- ChatPanel works in header drawer and full `/chat` page with same hook

## Risks

- Corporate proxy buffering — keep-alive + no-buffer header required
- Multi-tab shares rate limit bucket per user
