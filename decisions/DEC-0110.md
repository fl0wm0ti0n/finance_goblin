# DEC-0110 — CategoryFilter static import on BK surfaces (BUG-0021 EA)

**Status:** Accepted  
**Date:** 2026-06-11  
**Work item:** BUG-0021 (sub-defect **EA**, acceptance **BK**)  
**Research:** [R-0091](../docs/engineering/research.md#r-0091--bug-0021-categoryfilter-lazy-load-delay--wealth-role-column-empty)  
**Extends:** **DEC-0087** (category filter contract — component unchanged); **DEC-0088** (filter UX — interaction only)  
**Supersedes:** none

## Context

R-0091 confirmed the **Loading category filter…** Suspense fallback on Forecast **Monthly** and Wealth **Overview** tracks `React.lazy` dynamic import resolution (3–5 s perceived on tab switch), not `GET /api/v1/categories` (2–5 ms). `CategoryFilter` chunk is ~1.5 KB / ~96 LOC with no ECharts dependency. `CategoryTrendChart` remains a separate lazy boundary and is out of scope for BK.

## Options

| Option | Verdict | Rationale |
|--------|---------|-----------|
| **EA-A — static import on ForecastPage + WealthPage** | **Accepted** | Eliminates Suspense fallback on BK surfaces; satisfies BK ≤1 s interactive; bundle impact negligible vs main vendor splits |
| EA-B — `import()` prefetch on page mount + keep lazy | Rejected | Race still shows fallback; more complex than A for tiny component |
| EA-C — skeleton-only Suspense UX | Rejected | Cosmetic; fails BK if import remains slow |
| EA-D — Vite magic-comment prefetch | Rejected | Unreliable vs A; unnecessary at 1.5 KB |

## Decision

Accept **DEC-0110** as the CategoryFilter load contract for BUG-0021:

| Contract element | Value |
|------------------|-------|
| ForecastPage | Replace `React.lazy` CategoryFilter with top-level `import { CategoryFilter } from "../components/category/CategoryFilter"`; remove `<Suspense>` wrapper around CategoryFilter on **Monthly** tab only |
| WealthPage | Same static import; remove `<Suspense>` around CategoryFilter in **Overview** → Category spending card |
| CategoryTrendChart | **Unchanged** — remain `React.lazy` + Suspense (mounts after `categoryId` set) |
| Other lazy charts | **Unchanged** — `MonthlyChart`, `WealthChart`, etc. |
| PlanningPage parity | **P2 optional** — same static import policy recommended for consistency (out of BK acceptance); sprint may include as EA3 |
| Forbidden | Route-level prefetch-only without static import on BK pages; hiding filter behind longer skeleton; moving CategoryFilter into main index chunk via barrel re-export hacks |

## Verification gates (BK)

| Gate | Proof |
|------|-------|
| **BK** | Forecast → Monthly and Wealth → Overview: no multi-second **Loading category filter…** text; combobox interactive ≤1 s of tab visit (browser smoke or Performance trace) |
| Bundle | `npm run build` — CategoryFilter not emitted as separate lazy chunk for Forecast/Wealth entry (or chunk unused on BK path) |
| Regression | Category selection still drives `CategoryTrendChart`; categories API unchanged |
| OIDC | Standard omniflow frontend smoke on `/forecast` and `/wealth` |

## Risks

| Risk | Mitigation |
|------|------------|
| Slight Forecast/Wealth chunk growth | ~1.5 KB — monitor build report; acceptable per intake bundle constraint |
| Removing Suspense breaks if CategoryFilter later grows heavy | Re-evaluate only if component gains ECharts or large deps; document in DEC-0110 |
| PlanningPage inconsistency if EA3 deferred | Document in UAT; BK unaffected |

## Rollback

`git revert` ForecastPage/WealthPage (and PlanningPage if touched) import changes. No schema or API state.
