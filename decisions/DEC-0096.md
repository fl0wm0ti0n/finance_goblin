# DEC-0096 — Plan vs Actual remains household active-plan scope (US-0019)

**Status:** Accepted  
**Date:** 2026-06-09  
**Story:** US-0019  
**Research:** [R-0084 §2 scope guard](research.md#r-0084--us-0019-goal-plans-per-plan-stats-category-overlay--ai-savings)  
**Extends:** DEC-0074 (PVA 200 empty-state), DEC-0021 (household daily net), DEC-0024 (single active plan)

## Context

US-0019 adds per-plan goal stats on Scenarios and Compare. Discovery asked whether PVA should also scope to selected plan vs active plan.

## Decision

**No change** to Plan vs Actual tab contract:

- `GET /api/v1/plans/active/plan-vs-actual` remains **household** metrics for **globally active plan** only (DEC-0024)
- PVA guided UX (DEC-0074) unchanged when no active plan
- Goal-stats strip (`DEC-0092`) is the per-plan statistics surface for `goal_balance` template
- Category overlay adjustments on any plan version still flow into that plan's compare/PVA **when that plan is active** — existing DEC-0021 deviation math

Per-plan PVA endpoint **deferred** — would duplicate Grafana Dashboard 3 household contract.

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| Per-plan PVA for selected scenario | Breaks active-plan Grafana + alert semantics |
| PVA switches to goal plan automatically | Confusing vs explicit Set active (US-0014 AC-6) |

## Risks

- Operators expect PVA to follow Compare selection — UI copy: PVA = active plan; Compare stats = selected plan
- Multiple goal plans — only active plan drives PVA/alerts

## Implementation anchor

- No backend PVA changes required for MVP
- `frontend/src/pages/PlanningPage.tsx` — contextual help on Compare vs PVA tabs
