# Decision DEC-0042

**Status:** Accepted  
**Date:** 2026-06-01  
**Story:** US-0007  
**Research:** R-0037, R-0018, R-0026, R-0031  
**Extends:** DEC-0025 (`net_worth_snapshots`), US-0004 plan adjustments

## Context

Portfolio data needs normalized persistence. Discovery asked separate hypertable vs extend existing snapshots; allocation template storage; `get_portfolio` payload extension.

## Options

1. **Migration 007 normalized tables; extend `net_worth_snapshots`; `allocation_target` plan adjustment kind** — per R-0037
2. **Separate `portfolio_snapshots` hypertable** — rejected (duplicate daily grain; YAGNI)
3. **Dedicated `allocation_targets` table** — rejected (plan adjustment kind sufficient)
4. **Raw exchange JSON blobs only** — rejected (need normalized holdings for UI + Grafana SQL)

## Recommendation

**Migration 007 tables:**
- `exchange_connections`, `exchange_sync_state`, `exchange_holdings`, `exchange_trades`
- `exchange_transfers`, `exchange_funding_events` (MVP import surfaces)
- `portfolio_pnl_snapshots`, `portfolio_baselines`, `fx_rates`

**Extend `net_worth_snapshots`:**
- Columns: `crypto_value_eur`, `firefly_value_eur`, `total_return_pct`
- Payload: `{ crypto_holdings[], exchanges[], allocation, fx_incomplete }`

**Allocation template:** new `plan_adjustments.kind = allocation_target` with weights `{ etf_traditional_pct, crypto_pct, cash_pct }` summing 100.

**Grafana Dashboard 4:** SQL on extended columns/payload for crypto stat, allocation pie, performance series.

**`get_portfolio`:** crypto totals + top 5 holdings within 8 KB cap (extends DEC-0035).

Trade retention: 2 years default prune job.

## Impact

- Migration `007_exchanges_portfolio.sql`
- Plan templates module adds allocation preset
- Grafana `portfolio.json` panel updates replace US-0005 placeholder

## Risks

- `exchange_trades` volume unbounded — retention job required
- Same asset spot+perp needs distinct `(exchange_id, asset, product_type)` rows
