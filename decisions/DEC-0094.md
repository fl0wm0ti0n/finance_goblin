# DEC-0094 — Deterministic category savings suggestions (US-0019)

**Status:** Accepted  
**Date:** 2026-06-09  
**Story:** US-0019  
**Research:** [R-0084 §4](research.md#r-0084--us-0019-goal-plans-per-plan-stats-category-overlay--ai-savings)  
**Extends:** DEC-0032 (privacy aggregates), DEC-0007 (`resolve_bucket` fixed categories), DEC-0087 (category catalog)

## Context

US-0019 AC-4 requires AI-ranked reducible expense categories with evidence; operator **selects** suggestions to add as plan adjustments — no silent auto-apply. AC-5 requires aggregate/category signals only under `allow_raw_transactions=false`.

Current `GET /api/v1/plans/templates/savings-mode/suggestions` returns confirmed subscription payees only.

## Decision

### 1. New REST endpoint (primary path)

`GET /api/v1/plans/{plan_id}/category-savings-suggestions?months=6&limit=10`

Algorithm (deterministic):

1. `aggregates_by_category(period_start, period_end, None)` — sort `total_outflow` DESC
2. Filter: `total_outflow / months >= 20` (min €20/mo average)
3. Exclude categories already in plan adjustments (`target_type=category`, `direction=remove_outflow`)
4. Exclude DEC-0007 **fixed** bucket categories via `resolve_bucket` map (rent, utilities, etc.)
5. Return top-N with evidence fields

### 2. Response shape

```json
{
  "suggestions": [
    {
      "category_id": "42",
      "category_name": "Entertainment",
      "avg_monthly_outflow_eur": "85.50",
      "transaction_count": 24,
      "suggested_reduction_eur": "42.75",
      "evidence_summary": "6-month avg €85.50/mo; 24 transactions"
    }
  ],
  "meta": { "months": 6, "limit": 10, "ranking": "deterministic_aggregate" }
}
```

`suggested_reduction_eur` = **50%** of `avg_monthly_outflow_eur` (default reduction hint; operator edits before apply).

### 3. Operator apply flow (AC-4)

Modal parallel to savings-mode checkboxes:

1. Operator selects one or more suggestions
2. `POST` batch `remove_outflow` adjustments with `target_type=category`
3. Trigger plan recompute
4. **No silent auto-apply**

### 4. Privacy (AC-5)

- REST returns aggregates only — no raw transaction rows
- Audit log per adjustment create (US-0006 / US-0015 patterns)
- LLM-only ranking **rejected** for MVP — non-deterministic; privacy review burden

### 5. AI chat path

Optional second surface per **DEC-0097** — REST remains canonical for Scenarios modal.

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| LLM-only ranking | Non-deterministic; audit burden |
| Chat-only (no REST) | Scenarios modal needs REST |
| Show all categories incl. fixed | Suggests cutting rent/utilities |
| Auto-apply top suggestion | Violates AC-4 |

## Risks

- Multi-account mirror includes all asset txs — consistent with household actuals
- 50% default reduction arbitrary — operator adjusts; document in modal
- Category with sparse history below €20/mo threshold — excluded by design
- Fixed bucket map stale vs Firefly — use same DEC-0007 config as forecast
