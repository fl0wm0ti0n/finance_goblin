## Context

Planning adjustment form offers `household | subscription | account` in React. Database enum is `household | subscription | category | custom_label | allocation_target`. **`account` is invalid** ‚Äî insert would fail at API. Templates internally use `category` and `allocation_target`. Operator reports *"why only 3 categories?"* ‚Äî UX/doc gap, not missing enum values.

## Decision

### 1. UI options (frozen)

Replace invalid `account` with valid enum values exposed to operators:

| UI label | `target_type` value | Meaning (help copy) |
|----------|---------------------|---------------------|
| Household | `household` | All accounts ‚Äî whole-household adjustment |
| Subscription | `subscription` | Payee / recurring subscription pattern |
| Category | `category` | Firefly category scope |
| Custom label | `custom_label` | Free-text label target |
| Allocation target | `allocation_target` | Internal template allocation row |

**Remove `account`** from select options ‚Äî never send to API.

### 2. Help copy (frozen)

Inline help or tooltip below target-type select (one paragraph max):

- Household = applies across all linked accounts
- Subscription = matches detected subscription payees
- Category = scopes to a budget category
- Custom label / allocation target = advanced; templates may pre-fill

**No enum expansion** this sprint ‚Äî `custom_label` and `allocation_target` exposed but not required for omniflow smoke.

### 3. Out of scope

- New `target_type` enum values (e.g. `account`, `asset`)
- Wizard for category picker ‚Äî existing free-text or template flow unchanged
- Grafana Dashboard 3 changes

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| Add `account` to DB enum | Scope creep; Firefly account binding not modeled in adjustments MVP |
| Keep 3 options + doc-only | `account` remains broken; operator confusion persists |
| Hide advanced types entirely | Templates use `category` / `allocation_target` ‚Äî must remain valid |

## Risks

- More select options may overwhelm ‚Äî group advanced types or use optgroup if component supports
- Existing saved adjustments unaffected ‚Äî UI change forward-only

## Implementation anchor

- `frontend/src/pages/PlanningPage.tsx` ‚Äî select options + help copy
- `frontend/src/lib/api.ts` ‚Äî types align with backend enum (no `account`)
