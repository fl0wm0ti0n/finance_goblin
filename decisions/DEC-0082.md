# DEC-0082 — Block delete of active plan (BUG-0014 AS)

**Status:** Accepted  
**Date:** 2026-06-09  
**Work item:** BUG-0014 (sub-defect **AS**, task **AS1**)  
**Research:** [R-0079 §6](../docs/engineering/research.md#r-0079--bug-0014-post-rebuild-omniflow-ml-sidecar-crypto-display-grafana-planning)  
**Extends:** **DEC-0024** (single global active plan), **DEC-0074** (`no_active_plan` guided UX), **US-0004** Set active semantics

## Context

`DELETE /api/v1/plans/:id` exists in backend with CASCADE to versions/adjustments but **no active-plan guard**. React `PlanningPage.tsx` has adjustment delete only — no plan delete control. Deleting the active plan would leave `no_active_plan` (DEC-0074 handles PVA tab) but breaks Grafana Dashboard 3 budgets overlay until operator activates another plan.

Research recommended blocking active-plan delete (option **a**) over silent cascade (option **b**).

## Decision

### 1. Backend guard (frozen)

When `DELETE /api/v1/plans/:id` targets a plan with `is_active = true`:

- HTTP **409 Conflict**
- Body: `{ "error": "active_plan_delete_forbidden", "message": "…" }` (human-readable copy in implementation)

Non-active plans: delete proceeds with existing CASCADE (`plans` → `plan_versions` → adjustments).

### 2. Frontend contract (frozen)

| Surface | Behavior |
|---------|----------|
| Delete control | Per-plan row or detail action with confirmation modal (plan name) |
| Active plan | Delete button **disabled** with tooltip, or modal explains must deactivate first |
| Success | Invalidate `plans` query; clear `selectedPlanId` if deleted plan was selected |
| 409 response | Show error card per DEC-0077 mutation feedback pattern |

**Deactivate-first flow:** Operator sets another plan active, then deletes — no auto-activate on delete.

### 3. Out of scope

- Soft-delete / archive plans
- Bulk delete
- Cascade delete of active plan with auto-pick successor

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| Allow delete leaving `no_active_plan` | Breaks Grafana plan overlay silently; operator confusion |
| Auto-activate oldest remaining plan | Hidden state change; violates explicit Set active (US-0004) |
| 404 on active delete | Misleading — plan exists but operation forbidden |

## Risks

- Operator with single plan cannot delete until creating second plan — acceptable; copy must explain
- Stale `selectedPlanId` after delete — invalidate + redirect to plan list

## Implementation anchor

- `backend/src/plan/service.rs` or `backend/src/api/plans.rs` — active guard before delete
- `frontend/src/pages/PlanningPage.tsx` — `deletePlanMutation`, confirm modal, disabled state
