## Context

Two confirmed root causes (R-0090, live DB probes + code audit):

- **DA (BI):** `list_patterns` returns one row per `subscription_patterns` row with no dedup. **Strom** triplicate = 1 `rejected` + 2 `pending` with three `payee_key` variants (SEPA normalization drift; no confirmed row). **YouTube** duplicate = 2 `confirmed` rows with divergent `payee_key` strings (pre-DEC-0084 card-rule debt; DEC-0085 merge key mismatch on 2026-06-07). Frontend **All** tab omits `status` filter — rejected/pending noise surfaces as visually identical merchant rows. BUG-0015 forward merge prevents **new** pending for matching payee-interval but does not heal historical confirmed rows or collapse pending siblings.
- **DB (BJ):** Migration 014 added `display_category_id` column only — no retroactive backfill. All 7 confirmed patterns confirmed before 014; `refresh_display_category_id` runs only on confirm/merge paths (DEC-0100 §3). Live probe: 0/24 non-null despite 24/24 patterns having categorized linked txs.

## Decision

Accept **DEC-0109** as the BUG-0020 data-quality contract:

| Contract element | Value |
|------------------|-------|
| Migration | `backend/migrations/016_bug0020_subscription_list_quality.sql` — idempotent one-time reconcile + confirmed `display_category_id` backfill |
| Reconcile — YouTube | Merge 2 confirmed rows sharing `display_name` + `interval_matches` (both 30d): **survivor** = earliest `confirmed_at` (alert history); relink `subscription_pattern_transactions`; reassign `subscription_alerts.pattern_id`; mark loser `inactive`; update survivor `payee_key`/`fingerprint` to current normalized form |
| Reconcile — Strom pending | Collapse 2 pending rows sharing `display_name` + `interval_matches`: **survivor** = highest `last_seen_at`; relink txs; mark loser `rejected`; leave existing `rejected` row unchanged (hidden by All-tab filter) |
| Reconcile — policy | **Only** merge when `interval_matches` (DEC-0086 ±3d); **never** collapse distinct cadences; exact `payee_key` dup groups that are all `rejected` — no action |
| Backfill | `UPDATE subscription_patterns SET display_category_id = …` for `status = 'confirmed'` using DEC-0100 RANK SQL (same as `compute_display_category_id`); NULL when no categorized links |
| Forward guard | `detection.rs` / `upsert_pending_pattern` path: skip pending INSERT when `(payee_key, interval_days)` matches existing **confirmed** or **rejected** via `interval_matches` (extends DEC-0085 rejection load) |
| All-tab contract | **All** = `pending` + `confirmed` only — exclude `rejected` and `inactive` from default list query (frontend filter on API response; Pending tab unchanged) |
| API list | `GET /api/v1/subscriptions` without `status` **unchanged** (full fidelity for ops); BI primary gate on `?status=confirmed` post-reconcile + All-tab visible set |
| Forbidden | API-only dedup collapse; recompute `display_category_id` on list/sync; hardcoded merchant ids; Firefly write-back |

### Backfill SQL sketch (DEC-0100 RANK — per confirmed row)

```sql
UPDATE subscription_patterns p
SET display_category_id = sub.category_id,
    updated_at = NOW()
FROM (
  SELECT p2.id AS pattern_id,
         (
           WITH linked AS (
             SELECT t.category_id, t.date
             FROM subscription_pattern_transactions spt
             JOIN transactions t ON t.firefly_id = spt.transaction_firefly_id
             WHERE spt.pattern_id = p2.id AND t.category_id IS NOT NULL
           ),
           ranked AS (
             SELECT category_id,
                    RANK() OVER (ORDER BY COUNT(*) DESC, MAX(date) DESC) AS rnk
             FROM linked
             GROUP BY category_id
           )
           SELECT category_id FROM ranked WHERE rnk = 1 LIMIT 1
         ) AS category_id
  FROM subscription_patterns p2
  WHERE p2.status = 'confirmed'
) sub
WHERE p.id = sub.pattern_id;
```

## Risks

| Risk | Mitigation |
|------|------------|
| Merge deletes wrong YouTube row | Survivor = earliest `confirmed_at`; transaction boundary; relink before inactive |
| DEC-0086 multi-interval policy violated | `interval_matches` gate on every merge cluster |
| Backfill picks unexpected category (youtube→66) | DEC-0100 contract — document in UAT |
| All-tab scope change surprises operator | Release notes: All excludes rejected/inactive |
| `fingerprint` UNIQUE violation on survivor update | Update survivor in-place by `id`; run merge before fingerprint rotation |
| spt relink PK conflict | `ON CONFLICT DO NOTHING` then delete loser links |
