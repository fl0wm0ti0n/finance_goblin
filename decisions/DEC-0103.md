# DEC-0103 — Grafana subscriptions `$tag` variable (US-0020 P2)

**Status:** Accepted (P2 stretch)  
**Date:** 2026-06-10  
**Story:** US-0020  
**Research:** [R-0085 §6](research.md#r-0085--us-0020-subscription-discover-majority-category--operator-tags)  
**Extends:** **DEC-0089** (independent Grafana variables), **DEC-0068** (provisioning contract)

## Context

Discovery listed optional Grafana `subscriptions` dashboard **`$tag`** variable (stretch). AC-1..AC-6 do **not** require Grafana; SPA tag filter is sufficient for MVP per DEC-0089 precedent.

## Decision

### 1. Deferral gate (frozen)

Implement **only if** sprint task count ≤ `SPRINT_MAX_TASKS` (12) after mandatory US-0020 tasks. Otherwise defer post-MVP — **does not block release**.

### 2. Variable SQL (when implemented)

Mirror cashflow `$category` pattern:

```sql
SELECT '' AS __value, 'All tags' AS __text
UNION ALL
SELECT t.slug AS __value, t.name AS __text
FROM operator_tags t
ORDER BY 2
```

Panel filter:

```sql
… FROM subscription_patterns p
WHERE p.status = 'confirmed'
  AND ('${tag}' = '' OR EXISTS (
    SELECT 1 FROM subscription_pattern_tags spt
    JOIN operator_tags ot ON ot.id = spt.tag_id
    WHERE spt.pattern_id = p.id AND ot.slug = '${tag}'
  ))
```

### 3. Provisioning

Update `grafana/provisioning/dashboards/analytics/subscriptions.json` only — operator reload gate (**GRAFANA_PROVISIONING_RELOAD**) applies.

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| Required in MVP sprint | AC silent; risks sprint overflow |
| iframe SPA sync | DEC-0089 independent filters |

## Risks

- Empty tag table — variable still works ("All tags" only)
- Stretch slips sprint — acceptable; document in user guide as optional
