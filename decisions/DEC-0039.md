# Decision DEC-0039

**Status:** Accepted  
**Date:** 2026-06-01  
**Story:** US-0007  
**Research:** R-0034, R-0021  
**Extends:** DEC-0021 (EUR reporting currency)

## Context

Crypto holdings must convert to EUR for combined net worth headline. Discovery asked exchange ticker vs external FX API vs manual map.

## Options

1. **Two-layer: Frankfurter for fiat/stablecoin; exchange tickers for alts; `fx_incomplete` banner** — per R-0034
2. **Manual TOML price map** — rejected (operator burden; stale prices)
3. **CoinGecko primary for all crypto** — rejected (external dependency; exchange mismatch)
4. **Block headline until all assets priced** — rejected (partial data better than none with banner)

## Recommendation

**FxService** with daily Frankfurter cache in `fx_rates` table:
- Stablecoins/fiat → USD/EUR via Frankfurter ECB daily rate
- Crypto alts → `{ASSET}USDT` exchange ticker × USDT/EUR
- Bybit unified coins → response `usdValue` × USD/EUR when present

**Incomplete FX:** exclude unpriced alts from crypto subtotal; set `fx_incomplete=true` + UI banner listing assets.

**Self-hosted:** document optional Frankfurter sidecar; default public API with 24h cache + last-good offline rate.

CoinGecko fallback deferred post-MVP.

## Impact

- New `backend/src/fx/` module
- Wealth API returns `{ firefly, crypto, total_eur, fx_incomplete }` structure
- Extends US-0005 mixed-currency banner pattern for crypto

## Risks

- USDT≠USD peg drift acceptable for wealth view (document ±0.5% caveat)
- Illiquid alts without USDT pair fail conversion — banner required
