# Decision DEC-0025

**Status:** Accepted  
**Date:** 2026-05-31  
**Story:** US-0005  
**Research:** R-0021  
**Extends:** DEC-0021 (reporting currency EUR default from plan config pattern)

## Context

US-0005 acceptance requires net worth aggregation from Firefly-linked asset accounts and Grafana Dashboard 4 wealth-over-time. Discovery asked whether to store daily snapshots or compute on-demand, and how to handle multi-currency households without FX conversion (US-0007 deferred).

## Options

1. **Sum asset accounts with daily post-sync snapshots** — per R-0021
2. **On-demand compute from mirror only** — rejected (no trend history for Dashboard 4 / ECharts)
3. **Live FX conversion to reporting currency** — rejected (needs exchange-rate source; US-0007)
4. **Include liabilities in net worth** — rejected (Projectplan wealth phase focuses on assets)

## Recommendation

- Filter mirror `accounts` where `type = 'asset'`, `include_net_worth = true`, `active = true`
- Household total: `SUM(balance)` with **EUR reporting default**; if `COUNT(DISTINCT currency) > 1`, set `mixed_currency=true` and show UI disclaimer (no FX in MVP)
- **Crypto excluded** from total; React placeholder row "Connect exchanges — US-0007"
- **Daily upsert** to `net_worth_snapshots` (one row per `snapshot_date`) after successful sync in alerts phase; `payload` JSONB holds per-account breakdown
- Relational table sufficient (≈365 rows/year); no hypertable required for MVP

## Impact

- Migration 005 includes `net_worth_snapshots`
- `WealthService::snapshot_and_breakdown` called from `AlertService::run_post_sync`
- React `/wealth` and Grafana Dashboard 4 consume snapshots + live breakdown

## Risks

- Mixed-currency sum misleading without banner — mandatory `mixed_currency` flag in API
- `ccAsset` negative balances — MVP include only `balance >= 0` or document debt exclusion
