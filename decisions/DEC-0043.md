# Decision DEC-0043

**Status:** Accepted  
**Date:** 2026-06-02  
**Story:** US-0008  
**Research:** R-0040, R-0038, R-0039  
**Extends:** US-0006 `AiProvider` stub, `OpenAiProvider` HTTP client

## Context

US-0006 shipped a hardcoded `OpenAiProvider` posting to `https://api.openai.com/v1/chat/completions`. US-0008 must support Ollama and arbitrary OpenAI-compatible endpoints without duplicating HTTP logic or changing the six-tool orchestration loop (AC4).

## Options

1. **Unified `OpenAiCompatibleProvider` + factory presets** — recommended (R-0040)
2. **Separate `OllamaProvider` struct** — rejected (100% duplicate reqwest code)
3. **`async-openai` crate with `with_base_url`** — rejected (US-0006 uses reqwest directly; minimal diff)
4. **Native Ollama `/api/chat` client** — rejected (second request schema; forks orchestrator types)

## Recommendation

Replace `OpenAiProvider` with **`OpenAiCompatibleProvider`** implementing extended **`AiProvider` trait**:

```rust
pub trait AiProvider: Send + Sync {
    fn name(&self) -> &str;
    fn is_configured(&self) -> bool;
    fn is_local(&self) -> bool;
    fn display_label(&self) -> &str;
    fn omit_tool_choice(&self) -> bool;
    async fn chat_completion(&self, req: ChatCompletionRequest) -> Result<ChatCompletionResponse, ProviderError>;
    async fn chat_completion_stream(&self, req: ChatCompletionRequest) -> Result<reqwest::Response, ProviderError>;
}
```

**Factory** `build_provider(config: &AiConfig) -> Result<Arc<dyn AiProvider>, ProviderError>` resolves preset URLs and auth; wire once at `AiService` startup. Orchestrator signatures change from `&OpenAiProvider` to `&dyn AiProvider`.

HTTP posts to `{base_url}/chat/completions` (base URL includes trailing `/v1`).

## Impact

- `backend/src/ai/provider.rs` — refactor + factory module
- `backend/src/ai/orchestrator.rs` — trait-object injection
- `backend/src/api/chat.rs` — use `state.ai.provider()` not per-request `OpenAiProvider::from_config`
- Tool registry, PrivacyLayer, audit insert path **unchanged**

## Risks

- Trait-object refactor touches chat handlers — keep diff scoped to provider/orchestrator wiring (R-0040)
- Misconfigured `base_url` could point at OpenAI proxy while labeled local — validate presets never default local modes to `api.openai.com`
