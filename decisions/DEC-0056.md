# Decision DEC-0056

**Status:** Accepted  
**Date:** 2026-06-02  
**Story:** US-0010  
**Research:** R-0052, R-0053  
**Extends:** US-0001 Compose profiles (DEC-0003 external DB wiring)

## Context

US-0010 attaches Flow Finance AI to an existing omniflow Debian host where Firefly III (`firefly`), PostgreSQL (`postgres`), and Traefik already run on the external Docker network `traefik`. Partial implementation exists (`docker-compose.external.yml`) but profile union can still start bundled `firefly-iii`, Traefik labels are hardcoded, and Grafana still publishes host port `:3000` on the external merge.

## Options

### 1. `bundled-firefly` profile split (recommended)

Move `firefly-iii` from `[minimal, standard, full]` to `[bundled-firefly, standard, full]` only.

- Greenfield dev: `docker compose --profile minimal --profile bundled-firefly up`
- Omniflow: `COMPOSE_FILE=docker-compose.yml:docker-compose.external.yml` + `COMPOSE_PROFILES=external` only

**Alternative:** doc-only warning forbidding profile combination — rejected (R-0053 §2; operators will combine profiles).

### 2. Two-file external overlay contract (recommended)

- **Base:** `docker-compose.yml` — service definitions, healthchecks, profile membership, dev defaults (`host.docker.internal`).
- **Overlay:** `docker-compose.external.yml` — merge-only; joins `networks.traefik.external: true`; overrides `DATABASE_HOST=postgres`, `FIREFLY_BASE_URL=http://firefly:8080`; `flow-finance-ai.ports: !reset []`; Traefik router labels; **never** defines `postgres` or `firefly` services.

**Alternative:** single monolithic compose with env-conditional networks/labels — rejected at discovery (Traefik label leakage into local runs).

### 3. Env-parameterized Traefik labels (recommended)

Overlay labels use Compose interpolation with stable defaults:

```yaml
- traefik.http.routers.financegnome.rule=Host(`${TRAEFIK_HOST:-financegnome.omniflow.cc}`)
- traefik.http.routers.financegnome.middlewares=${TRAEFIK_MIDDLEWARE:-auth}
```

Router/service name `financegnome` stays fixed (collision guard vs host `firefly` router per R-0052). Defaults, not `${VAR:?}` — omniflow production host is stable; overrides support staging.

**Alternative:** hardcoded host in YAML — rejected (requires YAML edits for non-production hosts).

### 4. Grafana internal-only default on external profile (recommended)

- External overlay: `grafana.ports: !reset []`; join `traefik` network only; **no** Traefik Host router labels by default.
- Optional public host gated by `${GRAFANA_TRAEFIK_HOST:-}` — empty = no labels; when set, second router with same `${TRAEFIK_MIDDLEWARE:-auth}`.
- Reach Grafana via SSH tunnel, `docker exec`, or Traefik-network curl — not public internet by default.

**Alternative:** always publish Grafana on Traefik subdomain — rejected (admin surface + weak base defaults increase attack surface).

## Recommendation

Accept all four sub-decisions as one integrated external-deploy contract (**DEC-0056**):

| Sub-decision | Execute requirement |
|--------------|---------------------|
| Profile split | Move `firefly-iii` to `bundled-firefly`; update US-0001 start commands in runbook and `.env.example` |
| Overlay contract | Keep two-file merge; canonical invocation documented; CI asserts external-only services = `flow-finance-ai`, `grafana` |
| Traefik env | Parameterize `TRAEFIK_HOST`, `TRAEFIK_MIDDLEWARE` in overlay; document in `.env.example` |
| Grafana default | `grafana.ports: !reset []` in overlay; optional `GRAFANA_TRAEFIK_HOST` gate for public router |

**Operator preflight (documented, not automated):** TimescaleDB on shared `postgres` for database `flow_finance_ai` before `compose up` (R-0053 §1). **CI guard:** `docker compose config --services` assert; fail if `minimal+external` includes `firefly-iii` after split.

## Impact

- `docker-compose.yml` — profile membership for `firefly-iii`
- `docker-compose.external.yml` — Traefik env labels, Grafana `!reset`, optional Grafana router gate
- `.env.example` — omniflow block: `COMPOSE_FILE`, `COMPOSE_PROFILES`, `TRAEFIK_*`, `GRAFANA_TRAEFIK_HOST`, OIDC redirect notes
- `docs/engineering/runbook.md` — omniflow external deploy section (no host secrets)
- `tests/run-tests.sh` or `scripts/compose-config-check.sh` — external merge service-list regression
- US-0001 user guide start commands — `bundled-firefly` profile

## Risks

| Risk | Mitigation |
|------|------------|
| TimescaleDB absent on host `postgres:latest` | Operator preflight SQL; migration 001 fail-fast; runbook block before `up` |
| Profile union starts duplicate Firefly | `bundled-firefly` split + CI guard + runbook "never combine external with minimal/standard/full/bundled-firefly" |
| Compose `!reset` unsupported | Document Compose ≥2.24 minimum |
| Grafana unreachable without tunnel | Runbook documents SSH tunnel / exec access |
| OIDC redirect mismatch on public URL | Document IdP URIs; smoke path distinguishes `AUTH_DEV_BYPASS` vs auth-on |
| Breaking greenfield dev start command | Runbook + `.env.example` show `minimal --profile bundled-firefly` |

## Decision gate

None — all options converge on research-backed recommendation (R-0053). Execute may proceed after `/sprint-plan`.
