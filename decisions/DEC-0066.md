# DEC-0066 — ML posture metadata when forecast_ml disabled

**Status:** Accepted  
**Date:** 2026-06-05  
**Context:** BUG-0010 AC — external profile has ML off (DEC-0049); UI shows "ML skipped" when ML was never configured  
**Research:** [R-0062](../docs/engineering/research.md#r-0062--firefly-account-balance-mirror-vs-forecastwealth-inputs)

## Decision

When `[forecast_ml] enabled = false`:

1. **Sync path:** After successful baseline recompute, call `record_skip_on_baseline(baseline_id, ForecastMlError::Disabled)` → metadata `ml_status: "skipped"`, `ml_skipped_reason: "sidecar_disabled"`.
2. **Meta API fallback:** `GET /api/v1/forecast/meta` derives `ml_skipped_reason: "sidecar_disabled"` when config disabled **and** baseline metadata lacks `ml_skipped_reason` (stale rows pre-fix).

UI must distinguish three states (AC2): **ML available**, **ML not enabled** (`sidecar_disabled`), **ML skipped** (other reasons).

## Rationale

- DEC-0052 only records skip reasons when ML phase runs and fails — disabled config skips the phase entirely, leaving null metadata.
- Null metadata + default UI copy misleads operators into thinking ML infrastructure failed rather than profile intentionally baseline-only.
- Full ML on omniflow remains **US-0013** (AC3); this decision covers honest degraded messaging only.

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| Meta-only derive without sync persist | Stale baselines until next recompute; inconsistent with DEC-0052 metadata pattern |
| Treat disabled as `ml_status: "unconfigured"` new enum | Breaks existing consumers; `sidecar_disabled` already mapped in `ForecastMlService::record_skip_on_baseline` |
| Enable ML by default on external profile | Violates DEC-0049; requires sidecar ops epic |

## Risks

- Existing baselines show derived reason only after meta fallback until next Full sync recompute — acceptable transitional state.
- Operators may still want ML — pointer to US-0013 in UI copy, not in this bug scope.

## Implementation anchor

- `backend/src/sync/mod.rs` — post-baseline branch when `!forecast_ml.enabled`
- `backend/src/api/forecast.rs` — meta derive fallback
- `frontend/src/pages/ForecastPage.tsx` — three-state copy
