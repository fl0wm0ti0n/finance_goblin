# Decision DEC-0053

**Status:** Accepted  
**Date:** 2026-06-01  
**Story:** US-0009  
**Research:** R-0046, R-0049  
**Extends:** US-0002 forecast REST API, DEC-0009 (per-account scope)

## Context

AC2 requires 6–24 month projections with p10–p90 confidence bands. AC6 requires baseline vs ML compare in UI. Existing `/forecast/long-term` must remain backward compatible for Grafana and clients defaulting to baseline.

## Options

1. **`variant` query param + nullable band columns + `/compare` endpoint** — recommended (R-0046)
2. **Separate `/forecast/ml/*` route tree** — rejected (duplicate handlers; Grafana duplication)
3. **Bands only in JSONB metadata** — rejected (Grafana SQL panels need columns)
4. **Single payload always returning both variants** — rejected (payload bloat; breaks existing clients)

## Recommendation

**Storage:** nullable `balance_p10`, `balance_p90` on `forecast_balance_daily` (NULL for baseline rows).

**API:**

| Endpoint | Change |
|----------|--------|
| `GET /forecast/long-term?variant=baseline\|ml_enhanced` | Default `baseline`; ML adds band fields + explainability |
| `GET /forecast/compare?account_id=&horizon=` | Both series + delta at horizon end |
| `GET /forecast/meta` | `baseline_computation_id`, `ml_computation_id`, `ml_status`, `ml_skipped_reason` |

**Interval semantics:** StatsForecast `level=90` symmetric prediction intervals labeled p10/p90 in API — document in explainability panel.

**AI tool:** `get_forecast` optional `variant` param (default `baseline`) — no new tool.

**Horizons:** ML bands on 6, 12, 24 months; 3-month remains baseline-only fallback.

## Impact

- `backend/src/api/forecast.rs` — query parsing + response types
- `forecast::repository` — band column read/write
- React Long-term tab variant state
- Grafana panels use `$forecast_variant` (DEC-0055)

## Risks

- Symmetric intervals understate skewed spend tails — document + dim when low_confidence
- Compare endpoint stale when ML skipped — return 404 or partial with skip reason
- Existing integration tests insert without bands — NULL default preserves compatibility
