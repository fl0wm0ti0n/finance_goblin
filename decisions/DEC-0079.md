# DEC-0079 — Budgets MTD summary upper date bound (BUG-0013 AL)

**Status:** Accepted  
**Date:** 2026-06-08  
**Work item:** BUG-0013 (sub-defect **AL**)  
**Research:** [R-0076 §7](../docs/engineering/research.md#r-0076--omniflow-analytics-regression-hypotheses-post-us-0015)  
**Coordinate:** DEC-0073 / BUG-0011 — compare overlay delta is backend-only; Grafana `budgets.json` MTD panel is independent

## Context

Omniflow **MTD summary** panel (id **5**) sums `plan_daily_cashflow.planned_net` from `date_trunc('month', CURRENT_DATE)` forward **without** an upper bound. Dashboard `time.to` includes `now+30d`; active plan horizon spans **730** days → **−€150K** planned MTD with **€0** actual when no plan days have elapsed in the current month.

Discovery proof: broken sum **−150337.6** vs correct capped MTD **0** (plan starts 2026-06-07; probe date 2026-06-06).

## Decision

Freeze **month-to-date** semantics for Grafana budgets MTD summary:

| CTE | Filter (frozen) |
|-----|-----------------|
| `planned` | `pdc.ts >= date_trunc('month', CURRENT_DATE)` **AND** `pdc.ts::date <= CURRENT_DATE` |
| `actual` | unchanged — `t.date >= date_trunc('month', CURRENT_DATE)::date` (transactions are historical; no upper bound needed) |
| `Deviation MTD` | `actual − planned` using capped planned total |

**Scope:** Panel id **5** (`MTD summary`) only. Plan time-series and deviation chart panels use `$__timeFilter` — out of scope.

**Optional UX (AL1):** Panel description footnote when `MIN(pdc.ts::date)` for active plan is **after** `date_trunc('month', CURRENT_DATE)` — "Plan horizon starts mid-month; MTD planned reflects elapsed days only."

## Alternatives rejected

| Alternative | Why |
|-------------|-----|
| Cap via `$__timeFilter(pdc.ts)` on summary table | Dashboard range includes future days — same bug |
| Backend `plan_daily_cashflow` view with MTD column | Heavier than one-line SQL fix; Grafana-only symptom |
| Change plan engine horizon | Wrong layer — SQL semantics bug only |

## Impact

- **AL1** edits `grafana/provisioning/dashboards/analytics/budgets.json` panel **5** `rawSql`
- No backend migration or API change
- Grafana reprovision / container recreate required on deploy

## Risks

- Operator timezone vs `CURRENT_DATE` (DB UTC) — acceptable; matches existing panel `AT TIME ZONE 'UTC'` pattern on deviation chart
- Mid-month plan start shows **0** planned MTD until first plan day — footnote mitigates interpretation
