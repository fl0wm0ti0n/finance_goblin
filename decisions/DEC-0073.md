## Context

Compare tab **"Monthly delta sum"** aggregates full `plan_daily_cashflow.planned_net` (= baseline forecast + overlay delta). Zero-adjustment plans show illogical negatives (operator **-127489.44** monthly delta) because the column is mislabeled — it sums household forecast net, not scenario adjustments.

Research [R-0070](research.md#r-0070--bug-0011-planning-mode-compare-delta-empty-state-api-first-run-ux) §2 resolved the metric contract. Acceptance **AE** requires zero/neutral deltas on empty/minimal plans.

**ID note:** US-0090 caveman compression forward-refs formerly used DEC-0073 — renumbered to **DEC-0075** at BUG-0011 architecture (this record owns DEC-0073).

## Decision

Freeze **overlay-only** `monthly_delta_sum` at both compare call sites. **`projected_month_end_balance`** remains the full scenario month-end balance (baseline + overlay).

### 1. Metric formulas (frozen)

```text
monthly_delta_sum(version) =
  SUM( overlay_delta(d) for d in [month_start .. min(today, month_end)] )
  where overlay_delta = build_overlay_deltas(adjustments, confirmed_subs, month_start, month_end)[d]

projected_month_end_balance(version) =
  planned_balance at overlay_horizon_end(month_start) from full project_plan_series
  (unchanged — includes baseline forecast)
```

**Empty-plan guard:** when `adjustments.is_empty()` (and no subscription-removal overlay from Savings template), `monthly_delta_sum` **MUST** format **`0.00`** regardless of baseline magnitude. Projected balance may remain negative — that reflects household forecast, not a delta bug.

### 2. Implementation contract (frozen)

| Call site | File | Change |
|-----------|------|--------|
| DB-backed compare | `backend/src/plan/repository.rs` `version_metrics` | Replace `series.values().sum()` on `planned_net` with overlay-only sum via shared helper |
| In-memory compare | `backend/src/plan/service.rs` `project_adjustments_in_memory` | Same helper — identical semantics |
| Shared helper | `backend/src/plan/overlay.rs` or `project.rs` | `monthly_overlay_delta_sum(adjustments, confirmed_subs, month_start, month_end)` wrapping `build_overlay_deltas` |

**Endpoint scope:** `GET /api/v1/plans/{id}/compare` and React Compare tab only. No hypertable schema change; no Grafana panel SQL change.

### 3. UX contract (execute)

Optional compare-table footnote (not blocking): *"Monthly delta = scenario adjustments only; projected balance includes baseline forecast."*

### 4. Release note (mandatory)

Compare numbers shift for **all** non-empty plans — intentional alignment with R-0016 "net recurring impact €/month". Document in BUG-0011 release notes; not a DB migration.

## Rejected alternatives

| Alternative | Why rejected |
|-------------|--------------|
| Sum `planned_net` (status quo) | Fails acceptance **AE** on empty plans |
| Rename column only | Does not fix zero/neutral delta requirement |
| Delta vs Current (Ist) baseline version | Extra fetch; equivalent when overlay additive |
| Store overlay series in hypertable | Scope creep; compute at read time from adjustments |
| Change Grafana Dashboard 3 SQL | R-0020 — full `planned_net` correct for Plan/Ist/Abweichung panels |

## Risks

| Risk | Mitigation |
|------|------------|
| Operator surprise on non-empty compare numbers | Release note; overlay-only is R-0016 intent |
| Negative projected balance on zero-overlay plan | Help text; do not zero balance |
| Helper drift between repository and service | Single shared function; AE3 unit tests |
