# Decision DEC-0047

**Status:** Accepted  
**Date:** 2026-06-02  
**Story:** US-0008  
**Research:** R-0042, R-0035  
**Extends:** US-0006 Settings AI & Privacy, US-0007 exchange test-connection pattern

## Context

Operators need visibility into active AI provider without exposing secrets. US-0007 established Settings read-only display + `POST /api/v1/exchanges/{id}/test` pattern (R-0035). US-0008 extends this for AI provider reachability.

## Options

1. **Extend `GET /api/v1/settings` AI view + `POST /api/v1/ai/test`** тАФ recommended
2. **Dedicated `GET /api/v1/settings/ai` only** тАФ rejected (fragment settings API)
3. **Test endpoint runs full six-tool loop** тАФ rejected (slow, flaky, audit noise)
4. **Render API keys masked in Settings** тАФ rejected (secrets env-only; never in API)

## Recommendation

**Extend settings response** тАФ replace bare `openai_configured: bool` with structured AI status (keep backward-compatible alias during execute if needed):

```json
{
  "ai": {
    "provider": "ollama",
    "provider_label": "Local ┬╖ Ollama",
    "base_url": "http://ollama:11434/v1",
    "model": "qwen2.5:14b",
    "is_local": true,
    "provider_configured": true
  },
  "privacy": { "...": "unchanged" }
}
```

- `provider_configured`: factory `build_provider()` succeeds (API key present for OpenAI; valid config for local)
- Never return API key values

**Test endpoint:** `POST /api/v1/ai/test` (JWT required)

- Body optional: `{ "prompt": "ping" }` default `"Reply OK."`
- Executes **minimal non-streaming** `chat/completions` with **no tools**
- Response success: `{ "ok": true, "latency_ms": 842, "model": "...", "provider": "ollama", "sample": "OK" }`
- Response failure: `{ "ok": false, "error": "connection refused" }`
- Does **not** write audit row; optional in-memory `last_test_at` cache (exchange test pattern)

**React Settings AI & Privacy:**
- Provider table rows: Provider, Model, Base URL, Status badge
- **Test AI provider** button тАФ TanStack Query mutation (mirrors exchange test UX, R-0035)
- Read-only TOML note: restart required to change provider

**Chat UI provider badge:** map `provider_label` + `is_local` тЖТ Badge (`Local ┬╖ Ollama` / `Cloud ┬╖ OpenAI` / `Local ┬╖ Compatible`); disable input + Alert when `provider_configured=false`

## Impact

- `backend/src/api/mod.rs` or new `api/ai.rs` тАФ test handler
- React `SettingsPage` + `ChatPanel` header badges
- User guide documents test before first chat

## Risks

- Test success does not guarantee tool calling тАФ document separate tool smoke in user guide (R-0042)
- vLLM 400 tool-parser errors may appear only on tool requests тАФ test endpoint catches connectivity only
