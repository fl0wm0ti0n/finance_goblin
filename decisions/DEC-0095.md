# DEC-0095 — Goal account scope for balance projection (US-0019)

**Status:** Accepted  
**Date:** 2026-06-09  
**Story:** US-0019  
**Research:** [R-0084 §5](research.md#r-0084--us-0019-goal-plans-per-plan-stats-category-overlay--ai-savings)  
**Extends:** DEC-0009 (per-account forecast), **DEC-0091** (goal columns), **DEC-0092** (goal-stats)

## Context

`aggregate_daily_balances` sums all asset accounts in reporting currency — household total. Goal progress for "€10 000 in 5 months" is ambiguous across accounts (e.g. acct 114 vs 116 per BUG-0013).

## Decision

### 1. `goal_account_id` on `goal_balance` plans only

| State | Baseline / projection source |
|-------|------------------------------|
| `goal_account_id` set | `fetch_daily_series(computation_id, account_id)`; starting balance from that account |
| `NULL` on create | Default = asset account with **max positive balance** in `reporting_currency` (`ORDER BY balance DESC LIMIT 1` among `type='asset'`) |
| No qualifying asset account | Household sum (current behavior) + UI banner: *"Goal uses all accounts — select one account for precision"* |

### 2. Non-goal templates

Household aggregate unchanged — **no regression** to savings/leasing/house templates.

### 3. Operator edit

Changing `goal_account_id` on existing goal plan:

- Requires explicit save + **recompute**
- UI confirmation when balance baseline shifts materially

### 4. Multi-currency

Accounts outside `reporting_currency` excluded from default picker; document in goal create form.

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| Always household aggregate | Fails goal UX precision |
| Require account on create | Discovery: optional field |
| Per-version account scope | Plan-level field sufficient MVP |

## Risks

- Goal progress jumps on account change — confirmation + recompute
- Max-balance default picks wrong account on tie — show picker; operator can override
- Compare tab still household-level — document mismatch vs goal-stats account scope

## Implementation anchor

- `backend/src/plan/project.rs`, `service.rs`
- `backend/src/forecast/service.rs` (`fetch_daily_series`)
- `frontend/src/pages/PlanningPage.tsx` (account picker on goal create)
