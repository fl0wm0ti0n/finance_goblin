# Decision DEC-0029

**Status:** Accepted  
**Date:** 2026-05-31  
**Story:** US-0005  
**Research:** R-0025  
**Supersedes:** DEC-0012 (static €200 hardcode — partial; uids unchanged)

## Context

Acceptance requires configurable scarcity threshold and budget drift %. Dashboard 1 currently hardcodes €200 in `cashflow.json` refId B (DEC-0012). Grafana panel threshold UI cannot use variables; raw SQL can.

## Options

1. **TOML `[alerts]` → `alert_config` singleton DB mirror + Grafana `$scarcity_threshold` query variable** — per R-0025
2. **Env var in provisioning YAML only** — rejected (Alert Engine and Grafana diverge)
3. **Keep static €200 until US-0006** — rejected (acceptance requires centralization)
4. **Grafana Config from query results** — rejected for MVP (SQL variable simpler per R-0025)

## Recommendation

**TOML:**
```toml
[alerts]
scarcity_threshold_eur = 200.0
budget_drift_pct = 20.0
```

**DB mirror (migration 005):**
```sql
CREATE TABLE alert_config (
  id SMALLINT PRIMARY KEY DEFAULT 1 CHECK (id = 1),
  scarcity_threshold_eur NUMERIC(18,2) NOT NULL DEFAULT 200,
  budget_drift_pct NUMERIC(5,2) NOT NULL DEFAULT 20,
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

- App startup: UPSERT singleton from TOML into `alert_config`
- Alert Engine reads in-memory `AlertsConfig` (hot path)
- Grafana reads `alert_config` via template variable

**Dashboard 1 migration:** replace hardcoded `200 AS value` with `$scarcity_threshold` in refId B SQL. Variable query:
```sql
SELECT scarcity_threshold_eur AS __value, 'Scarcity threshold (€)' AS __text
FROM alert_config WHERE id = 1;
```

DEC-0012 stable uids (`cashflow`, `FlowFinancePostgreSQL`) remain unchanged.

## Impact

- Update `grafana/provisioning/dashboards/analytics/cashflow.json` during execute
- Config reload endpoint optional stretch; restart syncs DB mirror on startup

## Risks

- Grafana variable type coercion — use numeric cast in SQL if needed
- Config change requires app restart or reload to sync DB mirror
