# Decision DEC-0007

**Status:** Accepted  
**Date:** 2026-05-31  
**Story:** US-0002  
**Research:** R-0006

## Context

US-0002 needs a baseline forecast algorithm before the US-0003 subscription engine and US-0009 ML models. Discovery asked whether to use rolling averages only, full subscription detection, or a hybrid approach.

## Options

1. **Pure 3-month rolling average** — simplest; misses salary/rent cadence
2. **Full subscription engine first** — better accuracy but violates story split (US-0003)
3. **Hybrid rule-based** — lightweight recurring heuristics + rolling average for variable residual
4. **On-demand compute without persistence** — rejected by acceptance (hypertables required)

## Recommendation

**Hybrid rule-based model (R-0006):**

- Starting balance from synced Firefly asset account balance
- Deterministic layer: payee/description grouping, cadence stability, ±5% amount tolerance → project recurring flows
- Variable residual: 3-month rolling average (widen to all history if <90 days); cap outliers at 95th percentile
- Monthly buckets: income / fixed / variable via category config defaults
- Exclude internal transfers between own accounts from cashflow projection
- Expose `low_confidence` metadata when history is sparse

## Impact

- Forecast Engine module split: `recurring`, `rolling`, `project`, `repository`
- US-0003 replaces heuristics with confirmed subscriptions without API contract break
- False-positive recurring detection possible until US-0003 — mitigated via metadata flag

## Risks

- Heuristic false positives inflate certainty — document in API metadata; refine in US-0003
- Sparse accounts (<90 days) produce flat projections — `low_confidence` flag, do not fail recompute
- Category→bucket mapping needs sensible TOML defaults
