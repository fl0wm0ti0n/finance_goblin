# Decision DEC-0024

**Status:** Accepted  
**Date:** 2026-05-31  
**Story:** US-0004  
**Research:** R-0020, R-0008  
**Extends:** DEC-0012

## Context

US-0004 acceptance requires Grafana Dashboard 3 (Budgets: plan/ist/deviation) reflecting the active plan. Discovery asked household aggregate vs per-category panels and empty-state behavior.

## Options

1. **Single global active plan; Grafana uid `budgets`; household aggregate MVP** — per R-0020
2. **Per-category Grafana panels** — deferred to US-0005 budget-drift work
3. **Reuse Dashboard 1 with overlay** — rejected (uid/folder separation per Projectplan)
4. **Prometheus metrics exporter** — rejected (duplicate storage per R-0008)

## Recommendation

- **`POST /api/v1/plans/{id}/activate`** sets exactly one `plans.is_active=true` (transactional)
- Provision `grafana/provisioning/dashboards/analytics/budgets.json`:
  - uid **`budgets`**, folder **Analytics**
  - Variable `$active_plan_version` from active plan latest version
  - Panels: Plan, Ist, Abweichung time series; MTD summary table; active plan info stat
  - Empty state annotation when no active plan
- Datasource uid `FlowFinancePostgreSQL` (DEC-0012)

## Impact

- React active plan selector must call activate API
- Dashboard 3 SQL joins mirror transactions (Ist) with `plan_daily_cashflow` (planned)

## Risks

- Variable empty when no active plan — annotation fallback required
- Planned series extends to future days while Ist stops at today — visual distinction in UI/docs
