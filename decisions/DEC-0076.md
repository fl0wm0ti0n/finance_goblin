# DEC-0076 ŌĆö External profile ML sidecar compose contract (US-0013)

**Status:** Accepted  
**Date:** 2026-06-08  
**Story:** US-0013  
**Research:** [R-0071](../docs/engineering/research.md#r-0071--us-0013-production-ml-enablement-on-omniflow-external-profile)  
**Extends:** DEC-0049 (default-off sidecar), DEC-0052 (sync forecast_ml phase), DEC-0056 (omniflow external overlay), DEC-0066 (ML disabled metadata)

## Context

US-0009 ships a feature-complete ML stack (`stats-forecast` sidecar, sync `forecast_ml` phase, API `variant=ml_enhanced`, React Compare, Grafana `$forecast_variant`). On the US-0010 **external** omniflow profile, ML is **baseline-only by design today**: `stats-forecast` is gated to Compose profile `[full]` only; `docker-compose.external.yml` does not start the sidecar; `[forecast_ml] enabled=false` default (DEC-0049) preserves operator opt-in.

BUG-0010 closed baseline numbers (AA/AB/AC); AC3 production ML enablement was explicitly deferred to **US-0013**. Discovery and research (R-0071) resolved five open questions: overlay profile-merge (no duplicate service), traefik-only network co-attachment, runtime health SLO vs compose healthcheck, unchanged Ōēź12 monthly-point gate, and dual CI guard (compose assert + wiremock integration).

## Decision

Freeze the external-profile ML enablement contract as four elements below. **No new ML algorithms** ŌĆö wiring + operator opt-in + verification only.

### 1. Compose overlay contract (profile union)

| Element | Contract |
|---------|----------|
| **Base service** | Keep single `stats-forecast` definition in `docker-compose.yml` with `profiles: [full]` |
| **Overlay merge** | `docker-compose.external.yml` adds **additive** `profiles: [external]` on the **same** `stats-forecast` service key ŌĆö Compose merges profile arrays ŌåÆ `[full, external]` on one container definition |
| **Duplicate service** | **Reject** a second `stats-forecast:` block or full-only + external-only duplicate keys ŌĆö invalid merge |
| **Network** | Overlay sets `networks: [traefik]` on `stats-forecast` ŌĆö mirror `flow-finance-ai` / `grafana` external pattern; backend reaches `http://stats-forecast:8090` via embedded DNS |
| **Host port** | `${STATS_FORECAST_PORT:-8091}:8090` in external merge (omniflow host 8090 clash); internal DNS always uses container port **8090** |
| **Healthcheck** | Retain base `start_period: 30s`, `interval: 30s`, `retries: 3` ŌĆö **advisory only**; sync gates on runtime `health_ok()` |

**Rejected alternatives:**

| Alternative | Why rejected |
|-------------|--------------|
| `profiles: [full, external]` on base only | Breaks minimal/standard profiles starting sidecar unintentionally |
| Separate external-only sidecar service name | Duplicate keys / two containers on `full+external` merge |
| Dual-network (default + traefik) sidecar | Backend external merge is traefik-only; adds `_default` noise without benefit |
| `depends_on: service_healthy` for sync | Race on first cold start; DEC-0052 skip acceptable |

**Risks:** Cold-start race ŌĆö first post-deploy sync may record `sidecar_unavailable` until healthcheck green; runbook documents manual re-sync.

### 2. Backend env opt-in contract

| Variable | Scope | Default | Contract |
|----------|-------|---------|----------|
| `FORECAST_ML_ENABLED` | `flow-finance-ai` in external merge | unset ŌåÆ `false` (DEC-0049) | Operator sets `true` to enable ML phase |
| `STATS_FORECAST_URL` | `flow-finance-ai` in external merge | `http://stats-forecast:8090` | Document in `.env.example` omniflow block |
| `STATS_FORECAST_PORT` | host publish | `8091` on omniflow | Already in `.env.example`; overlay uses for host remap |

TOML `[forecast_ml] enabled` remains overridable via `FORECAST_ML_ENABLED` env merge in `backend/src/config/mod.rs` ŌĆö **no code change required** for enablement path if env passthrough is wired in compose overlay.

**Rejected:** Silent ML enable on external profile without explicit env ŌĆö violates DEC-0049 operator opt-in.

### 3. Failure semantics (unchanged)

| Outcome | Behavior | Ref |
|---------|----------|-----|
| Sidecar down / unhealthy | Sync **success**; `ml_skipped_reason=sidecar_unavailable` on baseline metadata | DEC-0052 |
| ML disabled | `sidecar_disabled` metadata; UI degraded copy per DEC-0066 | DEC-0049, DEC-0066 |
| Insufficient history (<12 mo) | `insufficient_history` skip; baseline authoritative | DEC-0051 |
| Sidecar HTTP error | `sidecar_error` skip; sync continues | DEC-0052 |

**Rejected:** Fail sync on ML error ŌĆö breaks omniflow availability contract.

### 4. CI guard contract

| Layer | Contract |
|-------|----------|
| **Compose assert** | `scripts/compose-config-check.sh`: external-only `config --services` = `flow-finance-ai`, `grafana`, **`stats-forecast`** (3 services); assert `stats-forecast` joins `traefik` network |
| **Integration** | Retain `cargo test --test forecast_ml_integration` in `tests/run-tests.sh` for wiremock sidecar + skip-metadata path |
| **Atomicity** | Compose-check update lands in **same PR** as overlay execute ŌĆö CI drift silently regresses US-0010 AC-1 |

**Rejected:** Wiremock integration only ŌĆö leaves external compose regression undetected (R-0071, R-0053 ┬¦7).

### Profile combination guard (unchanged)

`minimal+external` must **not** resurrect `firefly-iii` or `postgres` ŌĆö existing DEC-0056 / compose-config-check anti-combination rules unchanged. Adding `stats-forecast` to external merge does **not** relax profile-combination guards.

## Consequences

- Operators enable ML on omniflow with: external compose merge + `FORECAST_ML_ENABLED=true` + Full sync prerequisite + sidecar health smoke
- US-0009 full-profile dev path unchanged (`--profile full`)
- Grafana ML panels populate when `ml_enhanced` computations exist; BUG-0009 banner remains when ML off
- Runbook gains omniflow ML enablement section (execute S4)
