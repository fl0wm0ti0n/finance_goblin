# Decision DEC-0044

**Status:** Accepted  
**Date:** 2026-06-02  
**Story:** US-0008  
**Research:** R-0038, R-0039, R-0040  
**Extends:** US-0006 `[ai]` TOML section

## Context

Operators need three provider modes: cloud OpenAI, in-compose Ollama, and host-run OpenAI-compatible servers (LM Studio, LocalAI, vLLM). Config must follow existing TOML + env secret pattern (DEC-0040 exchange precedent, R-0035).

## Options

1. **Enum `openai | ollama | openai_compatible` + optional `base_url`** — recommended
2. **URL-only config (no provider enum)** — rejected (loses preset semantics and UI labels)
3. **Runtime provider switching via Settings UI** — rejected (restart required per backlog)
4. **Separate TOML sections per vendor** — rejected (YAGNI; same HTTP contract)

## Recommendation

Extend `[ai]` TOML:

```toml
[ai]
provider = "openai"              # openai | ollama | openai_compatible
base_url = ""                    # required for openai_compatible; default for ollama; ignored for openai
model = "gpt-4o-mini"            # or qwen2.5:14b / LM Studio loaded model id
api_key_env = "OPENAI_API_KEY"   # optional for local; required for openai
# existing max_tool_rounds, timeouts unchanged
```

**Factory preset resolution:**

| `provider` | `base_url` default | `api_key` | `is_local` | `display_label` |
|------------|-------------------|-----------|------------|-----------------|
| `openai` | `https://api.openai.com/v1` (fixed) | **required** from env | `false` | `Cloud · OpenAI` |
| `ollama` | `http://ollama:11434/v1` | optional | `true` | `Local · Ollama` |
| `openai_compatible` | from TOML (**required**) | optional | `true` | `Local · Compatible` |

Env override: `AI_PROVIDER`, `AI_BASE_URL`, `AI_MODEL` (optional operator convenience in Compose).

Startup validation: fail fast with clear log when `openai_compatible` missing `base_url` or `openai` missing API key.

## Impact

- `backend/src/config/mod.rs` — extend `AiConfig` + validation helpers
- `config/default.toml` — document local examples commented
- Compose operator docs — LM Studio via `host.docker.internal:1234/v1` (R-0039, R-0005)

## Risks

- Host-run LM Studio unreachable from container without `host.docker.internal:host-gateway` (R-0039)
- vLLM requires server flags (`--enable-auto-tool-choice`) — operator doc burden, surfaced via test endpoint (DEC-0047)
