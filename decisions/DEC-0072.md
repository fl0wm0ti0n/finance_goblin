**Coordinate:** [R-0065 § BUG-0008](research.md#bug-0008-coordination-do-not-merge-scope) — detection internals only; `list_patterns` REST unchanged

## Context

Live probe: **12** total patterns (**6 pending**) from **922+** transactions — operator expects materially higher recall. Root causes: SEPA memo strings fragment payee keys (`by_payee` / `extract_payee_source` prefers description); `RecurrenceGroup.category_ids` collected but unused; `detection_window_days = 365` misses annual subs needing 2+ cycles; `min_emit_confidence: 60` hardcoded (not TOML).

Research [R-0069](research.md#r-0069--bug-0008-detection-recall-levers-ai-path-boundary) phased recall levers. **Phase 1** adopted in this decision; **Phase 2** category grouping gated; threshold tuning and AI deferred.

## Decision

Freeze **X Phase 1 recall bundle**. Execute **after or with** DEC-0071 W slice in same sprint — never threshold-only tuning before dedup.

### Phase 1 — Adopt (execute in BUG-0008)

| Lever | Mechanism | Files (primary) |
|-------|-----------|-----------------|
| **Payee normalization** | Extend `payee_key()`: strip SEPA `SVWZ+`/reference tokens, card suffixes, collapse legal-entity suffixes (`GmbH`, `AB`); regex-guarded | `backend/src/recurrence/normalize.rs` |
| **Transfer-type counterparty priority** | For bank-transfer-shaped descriptions (`SVWZ\|UEBERWEISUNG\|Lastschrift`), prefer `counterparty_name` **before** full memo (invert DEC-0061 priority for transfer type only) | `backend/src/recurrence/group.rs`, `backend/src/subscriptions/detection.rs` |
| **Detection window** | `detection_window_days` **365 → 730** in `backend/config/default.toml` (config-only) | `default.toml`, config loader |

**Pass criteria (acceptance **X** probe):** pending + confirmed subscription-kind patterns **> 12** (live baseline) without `unread_new_detection > pending_patterns` (W invariant).

### Phase 2 — Gated (same sprint if capacity; else follow-up quick)

| Lever | Mechanism | Gate |
|-------|-----------|------|
| **Category-aware grouping** | When ≥**70%** txs in payee group share one `category_id`, add secondary key `cat:{category_id}` for weak payee keys | Execute after Phase 1 probe; integration tests for over-merge |
| **min_emit_confidence** | Wire to TOML; optional 60 → 55 | **Blocked** until W dedup + operator FP review on Phase 1 output |
| **Amount tolerance widen** | ±20% low-tier | **Defer** — try normalization first |

Acceptance **X** footer satisfied by Phase 1–2 rule improvements documented in architecture/release notes; AI path deferred.
