# Decision DEC-0030

**Status:** Accepted  
**Date:** 2026-05-31  
**Story:** US-0005  
**Research:** R-0023, R-0026, R-0011  
**Extends:** DEC-0012 (dashboard uid pattern)

## Context

US-0005 delivers React `/wealth`, unified `/alerts` inbox, and header notification bell (deferred from US-0003). Discovery asked unified vs separate subscription alert surfaces and Dashboard 4 partial scope.

## Options

1. **Separate unified inbox + header bell; subscription alerts unchanged; Dashboard 4 partial** — per R-0023, R-0026
2. **Migrate subscription alerts to unified inbox** — rejected (backlog boundary)
3. **Page-only alerts without header bell** — rejected (US-0003 explicitly deferred bell to US-0005)
4. **Full crypto panels in Dashboard 4** — rejected (US-0007)

## Recommendation

**React routes:**
- `/wealth` — enable Wealth nav; Overview tab with net worth stat, account breakdown, optional ECharts bar/line, mixed-currency banner, crypto placeholder row, link to Grafana Dashboard 4
- `/alerts` — full inbox table with type icon, message, timestamp; Acknowledge + Dismiss actions
- **Header bell** — unread badge from `GET /api/v1/alerts/unread-count`; Popover preview (latest 5); optional link "View subscription alerts (N)" when subscription unread > 0

**Grafana Dashboard 4 (`uid: portfolio`):**
- Total wealth stat (non-crypto asset sum)
- Account breakdown table; wealth-over-time from `net_worth_snapshots`
- Mixed-currency warning panel; crypto text placeholder
- Folder Analytics; `"id": null`

**Unread definition:** `status = 'active' AND acknowledged_at IS NULL` (dismissed excluded from bell).

## Impact

- Shared app header component gains bell + popover
- No changes to `/subscriptions` alert UX

## Risks

- Users may expect single inbox for all alert types — document cross-link
- Bell polling frequency vs sync interval — TanStack Query with reasonable staleTime
