# DEC-0091 — Goal balance plan schema and template (US-0019)

**Status:** Accepted  
**Date:** 2026-06-09  
**Story:** US-0019  
**Research:** [R-0084 §1](research.md#r-0084--us-0019-goal-plans-per-plan-stats-category-overlay--ai-savings)  
**Extends:** DEC-0019 (plan projection), DEC-0020 (version semantics), migration `004_plans.sql`

## Context

US-0019 AC-1 requires operators to create a plan with **target balance** and **target date** (e.g. €10 000 in 5 months). `plan_template` enum has no `goal_balance`; `PlanRow` has no goal columns.

## Decision

### 1. Enum + plan-level columns (frozen)

Migration adds:

| Column | Type | Rule |
|--------|------|------|
| `target_balance_eur` | `NUMERIC(18,2) NULL` | Required when `template = 'goal_balance'` |
| `target_date` | `DATE NULL` | Required when `template = 'goal_balance'`; must be ≥ today on create |
| `goal_account_id` | `TEXT NULL` | Optional Firefly `accounts.firefly_id`; see **DEC-0095** |

`ALTER TYPE plan_template ADD VALUE 'goal_balance'` in dedicated migration file after existing enum values.

### 2. API create contract

`POST /api/v1/plans` with `template: "goal_balance"` requires `target_balance_eur` and `target_date`; rejects missing fields with **422**.

Non-goal templates: columns remain NULL; no regression.

### 3. Version semantics

Goal fields live on **`plans`** row (not `plan_versions`). Editable until operator freezes version per DEC-0020; frozen versions expose goal fields read-only in UI only — no DB immutability trigger for MVP.

### 4. Scenarios template card

Planning Scenarios grid adds **Goal balance** card → same `POST` shape; extends US-0014 template discoverability (AC-6 regression guard).

## Alternatives rejected

| Alternative | Why rejected |
|-------------|--------------|
| JSON `metadata` blob | Weak typing; harder validation |
| Goal columns on `plan_versions` | Per-version goal drift rare; PO wants plan-level metadata |
| Separate `plan_goals` table | Over-normalized for three fields |

## Risks

- Enum migration ordering — dedicated migration; test rollback path
- Existing plans NULL-safe — validation only on `goal_balance` template
- Frozen version edit confusion — UI read-only copy only
